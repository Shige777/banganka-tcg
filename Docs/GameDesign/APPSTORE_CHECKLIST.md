## 6. 審査対策チェックリスト

### 6.1 必須対応（リジェクト回避）

- [ ] アカウント削除機能の実装（Guideline 5.1.1(v)）
- [ ] Apple Sign-in の提供（ソーシャルログイン提供時は必須）
- [ ] ガチャ確率の表示
- [ ] プライバシーポリシーURLの設定
- [ ] サポートURLの設定
- [ ] 全IAPの適切な登録（消費型）
- [ ] レシートのサーバーサイド検証
- [ ] IPv6対応（Firebase SDKは対応済み）
- [ ] Info.plist にカメラ・写真等の不要なパーミッション要求を含めない

### 6.2 推奨対応

- [ ] App Review用のデモアカウント準備（審査員が対戦をテストできるよう）
- [ ] オフラインモードorチュートリアルで基本動作の確認が可能
- [ ] 「復元」ボタン（非消費型IAPがある場合。消費型のみなら不要）
- [ ] 適切なエラーメッセージ（ネットワークエラー時にクラッシュしない）

### 6.3 よくあるリジェクト理由と対策

| リジェクト理由 | 対策 |
|--------------|------|
| メタデータのスクリーンショットと実際のUIが異なる | 実機キャプチャを使用 |
| アプリ内課金の説明が不十分 | 各IAP商品に詳しい説明を登録 |
| バグ（クラッシュ） | TestFlightで十分なテスト |
| アカウント削除がない | 設定画面に実装済み |
| 対戦相手がいないとプレイできない | チュートリアルまたはAI対戦を用意 |

## 7. TestFlight

### 7.1 内部テスト
- チーム内（最大100人）で配信
- App Store Connect のユーザーとして登録

### 7.2 外部テスト（MVP後）
- 最大10,000人のベータテスター
- Apple のベータ審査が必要（通常1〜2日）
- フィードバック収集用のリンクを設定
