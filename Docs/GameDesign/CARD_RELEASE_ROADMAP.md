# CARD_RELEASE_ROADMAP.md
> 万願果 カードリリースロードマップ v1.0

---

## 0. 目的

新カードセットの年間リリースサイクルを定義する。
「大弾＋ミニセット」の2段階構成により、プレイヤーの鮮度維持と1人開発の持続可能性を両立する。

---

## 1. リリースモデル：大弾＋ミニセット 2段階構成

### 1.1 基本サイクル

| 種別 | 頻度 | カード種類数 | 目的 |
|------|------|:-----------:|------|
| **大弾（Major Set）** | 年3回（約4ヶ月ごと） | 40〜50種 | 新キーワード・新シナジー・ストーリー新章の投入 |
| **ミニセット（Mini Set）** | 年3回（大弾の中間） | 10〜15種 | メタ補正・既存キーワードの派生・環境の揺さぶり |

- 年間の新カード総数: **150〜195種**
- コンテンツ投下回数: **年6回**（約2ヶ月に1回の新カード供給）

### 1.2 年間サイクルイメージ

```
月:  1月    2月半  3月   4月   5月    6月半  7月   8月   9月    10月半  11月   12月
     ┌──────┐      ┌─────┐      ┌──────┐      ┌─────┐      ┌──────┐      ┌─────┐
     │大弾①│      │    │      │大弾②│      │    │      │大弾③│      │    │
     └──────┘      │ミニ │      └──────┘      │ミニ │      └──────┘      │ミニ │
                   │セット│                   │セット│                   │セット│
                   │ ① │                   │ ② │                   │ ③ │
                   └─────┘                   └─────┘                   └─────┘
```

### 1.3 初年度の特別スケジュール

MVPカードプール（78〜162種）の薄さを補うため、初年度のみ大弾①を前倒しする。

| 時期 | 内容 | カード数 | 累計カードプール |
|------|------|:--------:|:--------------:|
| MVP リリース | 第1弾「交界の夜明け」 | 162種 | 162種 |
| **MVP後 2ヶ月** | **大弾①** 第2弾 | 45種 | 207種 |
| MVP後 3.5ヶ月 | ミニセット① | 12種 | 219種 |
| MVP後 6ヶ月 | 大弾② 第3弾 | 45種 | 264種 |
| MVP後 7.5ヶ月 | ミニセット② | 12種 | 276種 |
| MVP後 10ヶ月 | 大弾③ 第4弾 | 45種 | 321種 |

> 初年度終了時点で約333種。Hearthstone初年度（382種+ナーフ/バフ）に近い規模に到達する。

2年目以降は通常の4ヶ月サイクルに移行する。

## 2. 大弾（Major Set）の設計方針

### 2.1 構成比

CARD_SET_TEMPLATE.md §1 に準拠する。

| 項目 | 配分 | 枚数（45種の場合） |
|------|:----:|:-----------------:|
| 顕現（Manifestation） | 50〜60% | 23〜27種 |
| 詠術（Incantation） | 25〜30% | 11〜14種 |
| 界律（Algorithm） | 5〜15% | 3〜7種 |

| レアリティ | 配分 | 枚数（45種の場合） |
|-----------|:----:|:-----------------:|
| C | 40% | 18種 |
| R | 30% | 13〜14種 |
| SR | 20% | 9種 |
| SSR | 10% | 4〜5種 |

### 2.2 願相バランス

6願相に均等配分を原則とする。

| 願相 | 最低枚数 | 推奨枚数 |
|------|:--------:|:--------:|
| 各願相 | 6種 | 7〜8種 |
| 汎用（aspectTag: null） | 0種 | 0〜3種 |

### 2.3 大弾に含めるべき要素

- [ ] 新キーワード能力（1〜2種 / 大弾）
- [ ] 新アーキタイプまたは既存アーキタイプの新軸（1〜2種 / 大弾）
- [ ] ストーリー新章と連動したカード（フレーバーテキスト・イラスト）
- [ ] 各願相のSSR目玉カード（最低4枚、できれば全6願相に1枚ずつ）

### 2.4 制作スケジュール

CARD_SET_TEMPLATE.md §8 準拠。大弾は4ヶ月サイクル内で以下のように進行する。

```
             ← 4ヶ月 →
Week  1-2:  企画・テーマ策定（前弾のメタデータ分析を反映）
Week  3-6:  カード設計 + コストカーブ検証
Week  5-8:  イラスト制作（並行）
Week  7-9:  バランステスト（L5: 各組み合わせ3,000自動対戦）
Week 10-11: 実装・結合テスト
Week 12:    E2Eテスト・修正
Week 13-14: 告知期間（T-14日ティーザー → T-7日全カード公開）
Week 15:    リリース → 観察期間開始
Week 16-17: 新カード実装後データ収集（BALANCE_POLICY.md §2.2）
```

## 3. ミニセット（Mini Set）の設計方針

### 3.1 構成比

| 項目 | 配分 | 枚数（12種の場合） |
|------|:----:|:-----------------:|
| 顕現（Manifestation） | 50〜60% | 6〜7種 |
| 詠術（Incantation） | 25〜35% | 3〜4種 |
| 界律（Algorithm） | 10〜20% | 1〜2種 |

| レアリティ | 配分 | 枚数（12種の場合） |
|-----------|:----:|:-----------------:|
| C | 33% | 4種 |
| R | 33% | 4種 |
| SR | 25% | 3種 |
| SSR | 8% | 1種 |

### 3.2 ミニセットの役割

ミニセットは新しいメカニクスを導入する場ではない。以下の3つの役割に特化する。

**① メタ補正カード**: 大弾リリース後に支配的になったアーキタイプへのカウンターカードを投入する。BALANCE_POLICY.md §2.1 の監視指標に基づき、使用率60%超の願相/アーキタイプを抑制するカードを優先設計する。

**② 既存キーワードの派生**: 大弾で導入したキーワードを別の願相やコスト帯にも展開し、デッキ構築の選択肢を広げる。

**③ 弱い願相の底上げ**: 願相占有率が15%以下に落ちた願相に対し、バフ的なカード追加を行う。BALANCE_POLICY.md §3.2「バフ優先」の原則に沿い、ナーフの代わりにミニセットで周囲を引き上げる。

### 3.3 ミニセットに含めてはいけない要素

- 新キーワード能力（大弾で導入する）
- 環境を根本的に変えるSSR（大弾の目玉にする）
- ストーリー新章の解放（大弾と連動させる）

### 3.4 制作スケジュール

ミニセットは大弾リリース後のメタデータを見て設計するため、制作期間は短い。

```
             ← 約6週間 →
Week 1:     大弾リリース後2週間のメタデータ分析
Week 2-3:   カード設計 + コストカーブ検証
Week 3-4:   イラスト制作（並行）
Week 4-5:   バランステスト（簡易版: 影響デッキのみ1,000自動対戦）
Week 5:     実装・結合テスト
Week 6:     告知（T-5日） → リリース
```

### 3.5 ミニセットのバランステスト簡略化

BALANCE_POLICY.md の「各組み合わせ最低3,000自動対戦」は大弾に適用する。ミニセットは以下の簡略版を適用する。

| 項目 | 大弾 | ミニセット |
|------|:----:|:---------:|
| テスト対象 | 全アーキタイプ組み合わせ | 新カードが影響するデッキのみ |
| 自動対戦数 / 組み合わせ | 3,000 | 1,000 |
| 相性マトリクス | 全15通り | 影響のある組み合わせのみ |
| 先攻/後攻検証 | ±5%以内 | ±7%以内（緩和） |
| 平均決着ターン | 12〜20 | 12〜20（同一基準） |

## 4. リリースに伴うイベント連動

### 4.1 大弾リリース時のイベントスケジュール

```
T-14日: ティーザー公開（世界観・ストーリー予告）
T-7日:  全カードリスト公開 + パック予約開始（予約報酬: パック券×2）
T-0日:  大弾リリース
        ├─ 新パック販売開始
        ├─ ストーリー新章解放
        ├─ ログインボーナス（7日間: パック券×3 + ゴールド1,000）
        └─ 「新カード体験バトル」イベント（3日間: 新カードを試用貸出）
T+7日:  構築制限イベント（新シリーズ限定戦）
T+14日: 通常運営に移行
```

### 4.2 ミニセットリリース時のイベントスケジュール

```
T-5日: ミニセット告知（全カードリスト公開）
T-0日: ミニセットリリース
       ├─ 新カードは既存パックに混入（専用パックなし）
       ├─ ログインボーナス（3日間: パック券×1 + ゴールド500）
       └─ ミニセット記念ミッション（5勝で新SR確定チケット）
T+7日: 通常運営に移行
```

### 4.3 大弾間のイベント配置

大弾とミニセットの間に、EVENT_SYSTEM_SPEC.md のイベントを以下のように配置する。

```
月1: ── 大弾リリース ──────────────────────
       新カード体験バトル（3日）
       ログインボーナス（7日）
月1後半: 構築制限イベント（新シリーズ限定戦・7日）

月2: バランスパッチ（月次データレビュー）
     ドラフトイベント（3日）
     季節イベント or ランキングイベント

月2.5: ── ミニセットリリース ──────────────
       記念ミッション（7日）

月3: 構築制限イベント（レアリティカップ等・7日）
     バランスパッチ（月次データレビュー）

月4: ── 次の大弾リリース ──────────────────
```

## 6. ローテーションとの連動

### 6.1 スタンダード/エターナル導入タイミング

POST_MVP_ROADMAP.md §Phase 2 補足に準拠する。

| 時期 | 対応 |
|------|------|
| 第3弾（大弾②）リリース時 | スタンダード/エターナルの2フォーマット制導入 |
| 以降の大弾リリース時 | 最古のシリーズがスタンダード落ち |

### 6.2 スタンダード対象

```
スタンダード = ベーシック + 直近3シリーズ（大弾＋そのミニセット）
```

例:
- 大弾③リリース時: ベーシック + 第2弾 + 第3弾 + 第4弾（＋各ミニセット）
- 大弾④リリース時: ベーシック + 第3弾 + 第4弾 + 第5弾 → 第2弾がスタンダード落ち

### 6.3 ミニセットとローテーション

ミニセットは親の大弾と同一シリーズ扱いとする。大弾がスタンダード落ちすると、そのミニセットも同時にスタンダード落ちする。

## 8. KPI・成功指標

各リリース後に以下の指標を計測し、次弾の設計にフィードバックする。

| 指標 | 目標値 | 計測タイミング |
|------|--------|:------------:|
| リリース日DAU | 通常DAUの1.5倍以上 | T+0日 |
| 新カード採用率 | 新カードの50%以上がいずれかのデッキに採用 | T+14日 |
| メタ多様性 | 願相占有率が全願相15%〜25%の範囲内 | T+28日 |
| パック購入率 | アクティブユーザーの30%以上がパック購入 | T+7日 |
| プレイヤー復帰率 | 離脱ユーザーの10%以上がリリース週に復帰 | T+7日 |
