# AI_BOT_SPEC.md
> 万願果 AI対戦Bot設計 v1.0

---

## 0. 目的

DAU少数期やオフライン練習時にプレイ体験を維持するためのAI対戦Bot仕様を定義する。TUTORIAL_FLOW.md のダミーAIとは別に、通常対戦レベルのBot設計を行う。

---

## 1. Botの種類

| Bot名 | 難易度 | 対象プレイヤー | 思考深度 |
|-------|--------|-------------|---------|
| **見習い** | Easy | 初心者・ルール学習中 | 最善手を50%の確率で選択 |
| **挑戦者** | Normal | 中級者 | 最善手を75%の確率で選択 |
| **達人** | Hard | 上級者 | 最善手を95%の確率で選択 |

## 2. 意思決定ロジック

### 2.1 アーキテクチャ

ルールベース（スコアリング方式）を採用する。機械学習モデルはMVPでは不採用（サーバー負荷・開発コストの問題）。

```
各ターンの意思決定フロー:
  1. 手札の全プレイ候補を列挙
  2. 各候補にスコアを計算
  3. スコア順にソート
  4. 難易度に応じた確率で上位候補から選択
  5. 戦闘フェーズ: アタック対象の最適選択
```

### 2.2 スコアリング基準

| 評価要素 | 重み | 説明 |
|---------|------|------|
| **テンポ** | 高 | CPを効率よく使い切れるか |
| **盤面優位** | 高 | 顕現数 × 戦力合計の差 |
| **HP ダメージ評価** | 最高 | 直撃が通った場合の HP ダメージ量（固定%＝最大 HP の%, 現在%＝現在 HP の%） |
| **除去効率** | 中 | 相手の高コスト顕現を低コストで処理できるか |
| **願成進行** | 中 | 願主のKeyAspect一致カードの優先度 |
| **願主スキル価値** | 高 | スキル発動タイミングの最適化（Lv3必殺技はフィニッシュ盤面で使用） |
| **界律価値** | 中 | 界律の上書きによる全体効果の差分 |
| **ハンドアドバンテージ** | 低 | 手札枚数の維持 |

### 2.3 難易度別の挙動

| 行動 | Easy | Normal | Hard |
|------|------|--------|------|
| 最善手選択確率 | 50% | 75% | 95% |
| CPの使い切り | しない（1〜2枚残す） | 大体使う | 必ず使い切る |
| ブロック判断 | ランダム | 戦力比較して判断 | 最適ブロック |
| 界律の上書き判断 | 無視することが多い | 有利なら上書き | 常に最適判断 |
| 鯱鉾勝利認識 | 気づかないことがある（気づいても 50%の確率で狙う） | 80%気づいて狙う | 100%気づいて狙う |
| HP しきい値リスク認識 | 無視することが多い | 検討する（60%） | 常に最適判断 |
| マリガン | ランダム1〜2枚戻す | 高コストを戻す | 最適マリガン |

### 2.4 マリガン戦略

マリガンフェーズ（NETWORK_SPEC.md §7 準拠、30秒以内）でのBot判断ロジック。

**手札評価基準:**

各手札カードに「キープスコア」を算出し、低スコアのカードを入れ替え候補とする。

| 評価要素 | スコア影響 | 説明 |
|---------|----------|------|
| コストカーブ適合 | +30 | CP 1〜3 のカードはキープ優先（序盤展開に必須） |
| 願相一致 | +20 | デッキの主願相と一致するカードを優先 |
| キーカード | +25 | デッキコンセプトの中核カード（アーキタイプ依存） |
| 高コスト（CP 5+） | -20 | 序盤に使えないカードはキープ価値が低い |
| 重複カード | -15 | 同名カード2枚以上は1枚だけキープ |
| 界律 | -10 | 序盤に界律は不要なことが多い（例外あり） |

**難易度別マリガン挙動:**

| 行動 | Easy | Normal | Hard |
|------|------|--------|------|
| スコア計算 | しない | 簡易版（コストのみ） | 全要素 |
| 入替え判断 | ランダムに1〜2枚戻す | CP 4+を優先的に戻す | スコア下位カードを戻す |
| キープ枚数目安 | 3〜4枚（ランダム） | 2〜4枚（コスト依存） | 最適枚数（スコア依存） |
| 思考時間 | 2〜5秒 | 5〜10秒 | 10〜15秒 |

**アーキタイプ別キーカード優先度（6 人の願主）:**

| 願主 | アーキタイプ | キープ優先カード | 戻す優先カード |
|------|-----------|--------------|-------------|
| Aldric（曙） | アグロ | CP 1〜2 の Rush 持ち顕現 | CP 4+ 全般 |
| Vael（空） | コントロール | CP 1〜2 の妨害詠術 + Blocker | 高コスト顕現（後半ドローで引く前提） |
| 灯凪（穏） | ミッドレンジ | CP 2〜3 のバランス型顕現 | CP 1 の低パワー顕現 |
| Amara（妖） | コンボ | コンボパーツ（詠術 + 詠唱連鎖顕現） | 単体では機能しない高コストカード |
| Rahim（遊） | テンポ | CP 2〜3 の高コスパ顕現 | CP 5+ 全般 |
| 崔鋒（玄） | ディフェンス | Blocker 持ち顕現 + 封印詠術 | アグロ型カード |

### 2.5 思考時間（演出用）

Botの即座応答はプレイヤーに不自然な印象を与えるため、意図的な遅延を入れる。

| 難易度 | 思考時間 | 備考 |
|--------|---------|------|
| Easy | 1〜2秒 | 早めに返す |
| Normal | 2〜4秒 | 適度な間 |
| Hard | 3〜6秒 | 熟考している印象 |

## 3. Botデッキ

### 3.1 プリセットデッキ

各願相に1つずつ、計6種のBotデッキを用意する。

| デッキ名 | 願主 | コンセプト |
|---------|------|-----------|
| **燃え盛る意志** | Aldric（曙） | アグロ（低コスト+Rush主体） |
| **蒼天の策謀** | Vael（空） | コントロール（妨害詠術+界律） |
| **命脈の繁栄** | 灯凪（穏） | ミッドレンジ（展開+補助） |
| **幻惑の宴** | Amara（妖） | コンボ（詠術連鎖+HP 変動） |
| **機巧の遊戯** | Rahim（遊） | バランス型（柔軟対応） |
| **玄冥の防壁** | 崔鋒（玄） | ディフェンス（Blocker多数+耐久） |

### 3.2 デッキ選択ロジック

- Easy: 対戦相手のデッキ願主と異なる願主からランダム選択
- Normal: 6 種からランダム選択
- Hard: 対戦相手のデッキに有利な願主を優先選択（相性テーブル参照）

## 4. マッチメイキング連携

### 4.1 Bot提案トリガー

```
マッチメイキング開始
  → 30秒経過してもマッチングしない
  → トースト表示「対戦相手が見つかりません。AIと対戦しますか？」
  → [AIと対戦] [もう少し待つ]
  → 「もう少し待つ」→ 追加30秒 → 再提案
  → 「AIと対戦」→ 難易度選択 → Bot対戦開始
```

### 4.2 Bot対戦の表示

- 対戦画面で相手名に「[Bot]」を表示（例: `[Bot] 挑戦者`）
- Bot対戦であることを隠さない（透明性）

## 5. 報酬体系

### 5.1 Bot戦での報酬

| 報酬 | Bot戦 | 対人戦比 |
|------|-------|---------|
| ゴールド（勝利） | **25** | 50%（対人50） |
| ゴールド（敗北） | **10** | 50%（対人20） |
| バトルパスEXP | **通常の50%** | — |
| デイリーミッション「対戦N回」 | **カウントされる** | — |
| デイリーミッション「勝利N回」 | **カウントされない** | — |

### 5.2 日次上限

- Bot戦によるゴールド獲得: **1日10戦まで**（それ以降はゴールド0）
- バトルパスEXPも同じ10戦制限
- ミッションカウントは制限なし

## 6. Cloud Functions実装

Bot対戦はサーバーサイドで処理する（クライアントにBot AIを持たせない）。

### 6.1 フロー

```
1. クライアント: createBotMatch({ difficulty, playerDeckId })
2. Cloud Function:
   - Botデッキを選択
   - matchIdを生成
   - RTDB に /matches/{matchId}/ を初期化（Bot側のprivate/handも設定）
3. 対戦進行:
   - プレイヤーのコマンド → processCommand（通常通り）
   - Botのターン → scheduleBotTurn Cloud Function が2〜6秒後にコマンドを投入
4. 対戦終了:
   - onMatchEnd で報酬計算（Bot戦レートを適用）
```

### 6.2 コスト見積もり

- Cloud Functions呼び出し: Bot戦1回あたり約48回（24ターン×2コマンド/ターン）
- 追加コスト: 無料枠（月200万回）で DAU 1,000 × 5 Bot戦/日 = 24万回/月 → 無料枠内
