# MONETIZATION_DESIGN.md
> 万願果 マネタイゼーション総合設計 v2.0

---

## 0. 目的

ゲーム内経済・課金モデル・パック・バトルパス・広告・コスメティクスなど全収益設計を定義する。
プラットフォーム別の差異（iOS / Steam）も本書で管理する。

---

## 1. 設計方針

- **Pay-to-Win にしない**：カード入手速度の差のみ。レアリティ=強さにしない
- **無課金でも全カード入手可能**：時間はかかるが到達可能な設計
- **複数の課金導線**：パック・バトルパス・コスメ・スターターバンドルで多様な課金層に対応
- **プラットフォーム別最適化**：モバイルは広告あり、Steamは広告なし
- **リリース順序**：iOS先行 → 安定後にSteam展開 → 順次Android

## 4. パック設計（シリーズ制）

### 4.1 シリーズ（エキスパンション）構造

カードはシリーズ単位で追加される。各シリーズは独立したパックを持ち、そのシリーズのカードのみが排出される。

```
常設パック（シリーズ別）：
  第1弾「交界の目覚め」パック → 第1弾カードのみ排出
  第2弾「願主の試練」パック   → 第2弾カードのみ排出
  第3弾「...」パック          → 第3弾カードのみ排出
  （以降、シリーズ追加ごとに増加）

期間限定パック（願相ピックアップ）：
  1〜2週間ローテーションで6願相を巡回
  → 全シリーズ横断で特定願相の排出率UP
```

### 4.2 シリーズパックの種類

各シリーズに対して以下2種を提供：

| パック名 | 内容 | ゴールド | 願晶 |
|---------|------|---------|------|
| **通常パック** | 5枚（R以上1枚確定） | 500 | 50 |
| **プレミアムパック** | 10枚（SR以上1枚確定） | 900 | 90 |

### 4.3 願相ピックアップパック（期間限定）

| 項目 | 内容 |
|------|------|
| 名称 | 「曙の刻印」「空の囁き」など願相テーマ名 |
| ローテーション | 1〜2週間ごとに切替。6願相を順番に巡回 |
| 対象カード | **全シリーズ横断**で該当願相のカード |
| 確率変動 | 該当願相カードの排出率が通常の**2倍** |
| パック種類 | 5枚パック（500G / 50願晶）のみ |
| 実装時期 | Phase 3〜（カードプールが十分になった段階） |

**ピックアップ時の排出確率（5枚パック）**：

| スロット | ピックアップ願相 | 他の願相 |
|---------|---------------|---------|
| 1〜4枠目 | 通常確率の2倍 | 残りを均等配分 |
| 5枠目（R以上確定） | ピックアップ願相のR以上が60% | 他願相のR以上が40% |

### 4.4 排出確率（通常シリーズパック）

**通常パック（5枚）**：

| スロット | C | R | SR | SSR |
|---------|---|---|-----|------|
| 1〜4枠目 | 75% | 20% | 4% | 1% |
| 5枠目（R以上確定） | 0% | 80% | 16% | 4% |

**プレミアムパック（10枚）**：

| スロット | C | R | SR | SSR |
|---------|---|---|-----|------|
| 1〜9枠目 | 75% | 20% | 4% | 1% |
| 10枠目（SR以上確定） | 0% | 0% | 80% | 20% |

### 4.5 天井（Pity）システム
- **SSR天井：50パック**（通常パック基準）
- 50パック開封してもSSRが出ない場合、次のパックでSSR1枚確定
- カウントは**シリーズ別**に独立管理
- ピックアップパックのカウントは全シリーズ共有の別カウンター
- カウントは確認画面で表示

### 4.6 重複補償
- 既に3枚所持しているカードが排出された場合：
  - C → ゴールド30に変換
  - R → ゴールド80に変換
  - SR → ゴールド200に変換
  - SSR → ゴールド500 + 生成素材10に変換

## 6. リワード広告（モバイル限定）

### 6.1 概要
- **対象プラットフォーム**：iOS / Android のみ。**Steamには広告を一切入れない**
- **広告タイプ**：リワード動画広告（30秒・任意視聴）
- **広告SDK**：Google AdMob（Unity対応）

### 6.2 広告視聴ポイント

| 場面 | 報酬 | 回数上限 |
|------|------|---------|
| デイリーミッション完了後 | 報酬2倍（ゴールド + バトルパスEXP） | 各ミッション1回 |
| バトル終了後 | 報酬ゴールド2倍 | 1日5回 |
| パック開封後 | もう1枚ボーナスカード（C確定） | 1日3回 |

### 6.3 バトルパスとの関係
- バトルパス有料トラック購入者は、広告視聴なしで上記報酬2倍が常時適用
- パック開封ボーナスは広告視聴者のみ（パス購入者には別途パック券で還元）

### 6.4 Steam版の補填
- Steamには広告がないため、代わりにデイリーミッション基本報酬をモバイル版の1.3倍に設定
- またはSteam版バトルパスの価格を若干割引

## 9. カード生成（クラフト）システム

### 9.1 概要
重複カードや不要カードを「生成素材」に変換し、欲しいカードを直接生成する。
天井とは別の安全弁。「運が悪くても目当てのカードに確実に到達できる」設計。

### 9.2 分解（カード → 生成素材）

| レアリティ | 分解で得られる素材 |
|-----------|-----------------|
| C | 1 |
| R | 3 |
| SR | 10 |
| SSR | 30 |

### 9.3 生成（生成素材 → カード）

| レアリティ | 生成に必要な素材 |
|-----------|----------------|
| C | 5 |
| R | 15 |
| SR | 50 |
| SSR | 150 |

> SSR 1枚を生成するにはSSR 5枚分の分解が必要。重課金でないと即座には揃わないが、長期プレイで確実に到達可能。

### 9.4 実装時期
MVP（162種のカードプールで最初から利用可能）

## 10. ドラフトモード（アリーナ）

### 10.1 概要
ランダム提示されるカードからその場でデッキを構築し、連勝数を競うモード。

### 10.2 ルール

| 項目 | 内容 |
|------|------|
| 入場料 | ゴールド 150 |
| デッキ構築 | 3択×12回でカードを選択（12枚選択） + 22枚ランダム補填 = 34枚 |
| 対戦 | 7勝3敗制 |
| 中断・再開 | 可能 |

> ※ GAME_DESIGN.md v0.5.5に準拠（12勝3敗制から7勝3敗制に変更）

### 10.3 報酬

| 勝利数 | 報酬 |
|--------|------|
| 0勝 | パック1個 |
| 3勝 | パック1個 + ゴールド 50 |
| 5勝 | パック2個 + ゴールド 100 |
| 7勝 | パック3個 + ゴールド 200 + 確定SR |

> 7勝で入場料以上の価値を回収可能。実力に対する報酬。

### 10.4 実装時期
Phase 3（カードプール60種以上で成立）

## 15. App Store / Steam 審査対策

- パック購入画面に「提供割合」ボタンを設置

- 天井の仕組みも併記
- ピックアップパックはピックアップ対象カードの確率も個別表示

### 15.2 Apple ガイドライン準拠

| 項目 | 対応 |
|------|------|
| 3.1.1 In-App Purchase | すべての仮想通貨購入にIAP使用 |
| 3.1.2(a) 消費型IAP | 願晶は消費型IAP |
