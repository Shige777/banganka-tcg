# BALANCE_POLICY.md
> 万願果 カードバランス調整ポリシー v1.0

---

## 0. 目的

カードのナーフ（弱体化）・バフ（強化）・禁止/制限の判断基準、告知手順、プレイヤー補償ルールを定義する。運営後のバランス調整における信頼性と透明性を保証するための指針。

---

## 1. 設計原則

- **透明性**: 調整の理由をプレイヤーに必ず説明する
- **予測可能性**: 調整基準を事前に公開し、恣意的な変更を避ける
- **補償の公平性**: ナーフされたカードに投資したプレイヤーを補償する
- **慎重な介入**: データ不足の段階で急いで調整しない
- **バフ優先**: 可能な限りナーフより周囲のバフで環境を改善する

---

## 2. データ収集

### 2.1 監視指標

| 指標 | 計測方法 | 危険閾値 |
|------|---------|---------|
| **使用率** | デッキ採用率（全対戦のデッキに含まれる割合） | 単一カード: 60%以上 |
| **勝率** | そのカードを含むデッキの対戦勝率 | 58%以上（3,000試合以上のサンプル） |
| **初手キープ率** | マリガンで保持される割合 | 極端に高い場合は要注意 |
| **願相占有率** | ランク帯ごとの願相分布 | 単一願相が40%以上 |
| **ゲーム決定率** | そのカードがプレイされた試合の勝率変動 | プレイ時点で勝率+20%以上 |

### 2.2 観察期間

| フェーズ | 期間 | 対応 |
|---------|------|------|
| **新カード実装直後** | 1〜2週間 | データ収集のみ（緊急バグ除く） |
| **環境定着期** | 2〜4週間 | メタ分析、コミュニティ動向確認 |
| **調整判断期** | 4週間以降 | データが閾値を超えた場合に調整検討 |

### 2.3 緊急調整の条件

以下の場合に限り、観察期間を短縮して即時対応する。

- ゲームを壊すバグ（意図しない無限コンボ、0ターンキル等）
- 使用率80%以上かつ勝率65%以上
- サーバー負荷に影響するエクスプロイト

---

## 3. 調整の種類

### 3.1 ナーフ（弱体化）

カードのパラメータを下方修正する。

変更対象:
- CPコスト引き上げ
- 戦力（BattlePower）引き下げ
- 願撃（WishDamage）引き下げ
- **HP ダメージ % の引き下げ** (SPELL_HP_DAMAGE_FIXED / SPELL_HP_DAMAGE_CURRENT)
- **WishTrigger 効果の弱体化** (WT_DRAW → 枚数削減、WT_BOUNCE 効果範囲縮小等)
- 効果テキストの変更（対象縮小、条件追加等）
- キーワード削除

### 3.2 バフ（強化）

カードのパラメータを上方修正する。ナーフの前にまず弱い願相/アーキタイプをバフで引き上げることを検討する。

### 3.3 テキスト修正

バグ修正や表記揺れの統一。バランスに影響しない場合は補償なし。

### 3.4 禁止/制限（ランク戦導入後）

| 区分 | 内容 | 条件 |
|------|------|------|
| **制限** | デッキ内の採用枚数を1枚に制限 | ナーフでは解決困難な場合 |
| **禁止** | ランク戦での使用を禁止（カジュアル/フレンド戦では使用可） | ゲーム体験を根本的に損なう場合 |

## 4. 告知フロー

### 4.1 通常調整

```
T-7日: 調整予告（対象カード名 + 調整方向「弱体化予定」を公開）
        └─ アプリ内お知らせ + SNS
T-3日: 調整詳細（具体的な変更内容 + 理由 + データ根拠を公開）
        └─ アプリ内お知らせ + 公式サイト
T-0日: メンテナンス実施・調整適用
        └─ 補償期間開始
T+14日: 補償期間終了
```

### 4.2 緊急調整

```
T-0〜1日: 緊急告知 + 調整適用（同日 or 翌日メンテナンス）
T+14日: 補償期間終了
```

## 5. 補償ルール

### 5.1 ナーフ補償

ナーフされたカードは、**補償期間中（14日間）に限り、生成素材の全額返却レートで分解可能**にする。

| レアリティ | 通常分解 | ナーフ補償分解（14日間限定） |
|-----------|---------|--------------------------|
| **C** | 1 | **5**（＝生成コストと同額） |
| **R** | 3 | **15** |
| **SR** | 10 | **50** |
| **SSR** | 30 | **150** |

- Foil版は通常分解の1.5倍が補償レートに適用される
- 補償期間中は対象カードに「補償対象」バッジを表示
- 補償期間終了後は通常の分解レートに戻る

### 5.2 禁止補償

禁止カードも同様に14日間の全額返却分解を提供する。禁止カードはカジュアル/フレンド戦では使用可能なため、分解は任意。

### 5.3 バフ・テキスト修正

補償なし（プレイヤーにとって有利な変更のため）。

### 5.4 複数枚所持の扱い

同名カード3枚所持 → ナーフ → 3枚すべてを補償レートで分解可能。所持枚数の上限は問わない（パック開封で4枚以上持っている場合も全枚補償対象）。

## 6. バランスパッチスケジュール

### 6.1 定期パッチ

> カードリリースサイクル（大弾＋ミニセット 2段階構成）については `CARD_RELEASE_ROADMAP.md` を参照。

| 周期 | 内容 |
|------|------|
| **月次** | データレビュー（内部分析、調整の要否判断） |
| **大弾リリース後（4ヶ月ごと）** | 環境リセット候補の調整パッチ |
| **大弾実装2週後** | 新カードデータ収集完了。調整判断開始 |
| **ミニセット実装時** | メタ補正カード投入（ナーフ代替としてのバフ的追加） |
| **ミニセット実装2週後** | ミニセット影響評価。追加調整の要否判断 |

> **ミニセットとバランス調整の関係**: ミニセットのカード設計自体がバランス調整の一環として機能する。
> 使用率60%超のアーキタイプへのカウンター投入や、占有率15%以下の願相の底上げは、
> ナーフよりもミニセットでの新カード追加で対処することを優先する（§3.2「バフ優先」原則）。

### 6.2 パッチノート

調整実施時は以下の形式でパッチノートを公開する。

```
■ バランス調整 20XX.XX.XX

【弱体化】
▼ 暁門の剣士（曙 / 顕現 / SSR）
  - 戦力: 5000 → 4000
  - 理由: 使用率72%、勝率61%。曙アグロの支配的なフィニッシャーとなっており、
          他願相のミッドレンジ戦略を過度に抑圧していた。
  - 補償: 2週間の全額分解期間を設定

【強化】
▲ 静寂の守り手（玄 / 顕現 / R）
  - 戦力: 3000 → 4000
  - 理由: 玄デッキの採用率が全願相中最低（8%）。
          防御軸の選択肢として競争力を持たせる。
```

## 7. コミュニティ対応

### 7.1 フィードバック受付

- アプリ内「お問い合わせ」からバランス報告を受付
- SNS（Twitter/X）でのコミュニティ動向を監視
- 将来的にDiscordサーバーでのフィードバックチャンネルを検討

### 7.2 デザイナーズノート

シーズンごとに「デザイナーズノート」を公開し、以下を説明する。

- 現環境の評価（健全かどうか）
- 各願相の分布と目標値
- 今後の調整方針
- 次シリーズのデザイン意図（ネタバレしない範囲で）

---

## 8. MVP期間の特別ルール

MVP（ソフトローンチ〜正式リリース）期間中は、以下の緩和措置を適用する。

- 観察期間を2週間に短縮（データが少ないため素早く対応）
- 告知期間を3日に短縮（T-3日: 詳細 → T-0日: 適用）
- ナーフ補償は通常通り14日間を維持
- バフの頻度を上げ、積極的にカードプールの底上げを行う
