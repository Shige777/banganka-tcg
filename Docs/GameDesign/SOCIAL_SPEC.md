# SOCIAL_SPEC.md
> 万願果 ソーシャル機能設計 v1.0

---

## 0. 目的

フレンド機能・フレンド対戦・デッキ共有・エモート・プロフィールなど、プレイヤー間のソーシャルインタラクションを定義する。リテンションとバイラル拡散の基盤。

---

## 1. フレンドシステム

### 1.1 フレンドコード

- 各プレイヤーに**8桁の英数字フレンドコード**を自動付与（例: `BNG-A3K7-M2P9`）
- コードはアカウント作成時にランダム生成
- 変更不可（一意性保証）

### 1.2 フレンド追加フロー

```
方法1: コード検索
  フレンド画面 → 「コードで追加」→ 相手コード入力 → 相手プロフィール確認 → 「申請」

方法2: 対戦後追加
  リザルト画面 → 「フレンド申請」ボタン → 相手に通知

方法3: QRコード（MVP後）
  フレンドコードのQR表示 → 相手がカメラで読取
```

### 1.3 申請・承認

| 操作 | 動作 |
|------|------|
| 申請送信 | 相手の「申請一覧」に追加。プッシュ通知（有効時） |
| 承認 | 双方がフレンドリストに追加される |
| 拒否 | 申請が削除される。相手に通知しない |
| ブロック | 相手からの申請・対戦招待を今後受け付けない |

### 1.4 フレンドリスト

| 項目 | 表示内容 |
|------|---------|
| プレイヤー名 | 表示名 |
| オンライン状態 | 緑丸=オンライン / 灰丸=オフライン / 剣アイコン=対戦中 |
| 愛用願相 | 最も使用頻度の高い願相アイコン |
| 最終ログイン | 「5分前」「3時間前」「1日前」等 |

### 1.5 上限

| 項目 | 値 |
|------|-----|
| フレンド上限 | **200人**（競合水準: HS=200, MTGA=無制限。配信者・競技プレイヤーへの配慮） |
| 申請保留上限 | **50件**（超過時は古い申請から自動削除） |

## 2. フレンド対戦

### 2.1 招待フロー

```
フレンドリスト → 相手をタップ → 「対戦に招待」
  → ルーム自動作成（NETWORK_SPEC.md のルーム作成と同一基盤）
  → 相手にプッシュ通知 + アプリ内通知
  → 相手が「承諾」→ マッチ開始
  → 相手が「辞退」or 5分無応答 → ルーム自動削除
```

### 2.2 フレンド対戦の制約

- ランク戦には影響しない（カジュアル扱い）
- ミッション進捗には**カウントされる**（デイリーミッション「対戦N回」等）
- 報酬ゴールドは**通常の50%**（ファーミング防止）
- バトルパスEXPは**通常の50%**

### 2.3 ルーム設定（MVP後拡張）

将来的にフレンド対戦時にカスタムルールを設定可能にする。

- ターンタイマー変更（30s / 60s / 90s / 無制限）
- 界律禁止モード
- ランダムデッキモード
- 観戦許可 ON/OFF

## 3. デッキコード共有

DECK_BUILDER_SPEC.md §6 で定義したデッキコードのソーシャル側フローを補足する。

### 3.1 共有先

| 共有先 | 方法 | 実装時期 |
|--------|------|---------|
| クリップボード | コピーボタン | MVP |
| アプリ内フレンドチャット | 送信ボタン（MVP後） | Phase 2 |
| Twitter/X | シェアボタン（OGP画像付き） | Phase 2 |
| LINE | シェアボタン | Phase 2 |

### 3.2 インポート確認

受け取り側がデッキコードを開くと:
- デッキ内容のプレビュー表示
- 未所持カードの一覧 + 生成コスト合計
- 「このデッキを作成」→ デッキ枠に追加
- 未所持カードがある場合は使用不可状態で作成（後でクラフトで補完）

## 4. エモート/スタンプシステム

### 4.1 基本エモート（無料）

バトル中に使用可能な定型コミュニケーション。

| # | エモート | 表示 | 用途 |
|---|---------|------|------|
| 1 | よろしく | 願主がお辞儀 | 対戦開始 |
| 2 | ナイス | 願主が拍手 | 相手の良手を称える |
| 3 | すごい | 願主が驚く | 驚き |
| 4 | ありがとう | 願主が頭を下げる | 感謝 |
| 5 | 考え中… | 願主が腕を組む | 思考中 |
| 6 | お見事 | 願主が拍手（大きく） | 対戦終了時 |

### 4.2 プレミアムエモート（課金）

MONETIZATION_DESIGN.md §10 で販売するコスメティクスとして追加エモートを提供。

- キャラクター別エモート（ナル専用モーション等）
- シーズン限定エモート
- バトルパス報酬エモート

### 4.3 エモートUI

- バトル画面右下にエモートボタン（吹き出しアイコン）
- タップ → 6枠のエモートパレット表示
- エモート選択 → 自分の願主上にアニメーション表示（1.5s）
- 相手にも同時表示
- **エモートミュート**: 設定 or バトル中トグルで相手エモートを非表示にできる

### 4.4 スパム防止

- エモート送信後**5秒間のクールダウン**
- 1ターン中の最大送信回数: **3回**
- 相手がミュートしている場合、送信側にフィードバックなし（送れたように見える）

## 5. プレイヤープロフィール

### 5.1 表示項目

| 項目 | 公開範囲 | 編集可否 |
|------|---------|---------|
| プレイヤー名 | 全員 | 可（制限あり） |
| フレンドコード | 全員 | 不可 |
| 称号 | 全員 | 可（獲得済みから選択） |
| プロフィールアイコン | 全員 | 可（獲得済みから選択） |
| カードスリーブ | 全員 | 可 |
| 愛用願相 | 全員 | 自動（最多使用願相） |
| 通算戦績（勝/敗/引分） | フレンドのみ | 不可 |
| 最高ランク | フレンドのみ（ランク実装後） | 不可 |
| 登録日 | 自分のみ | 不可 |

### 5.2 プレイヤー名ルール

- 1〜12文字（日本語/英数字/一部記号）
- NGワードフィルタ（不適切な名前の禁止）
- 変更は**30日に1回**まで
- 初回設定はオンボーディングで実施

### 5.3 称号

バトルパス・ミッション・イベントで獲得。プレイヤー名の上に表示。

例:
- 「果求者」（初期称号）
- 「百戦の闘士」（通算100勝）
- 「願相マスター」（全6願相でデッキ構築）
- 「願道の探求者」（バトルパスLv30到達）

## 6. 対戦結果シェア

### 6.1 リザルト画面のシェア機能

対戦終了後に「結果をシェア」ボタンを表示。

生成画像の内容:
- 勝敗結果
- 自分の願主 + デッキ願相分布
- ターン数
- 万願果ロゴ + アプリリンク

### 6.2 テキストシェア（MVP）

```
【万願果】対戦結果
勝利！ ⚔️ Turn 15
願主: 灯凪（Lv3）
デッキ: 曙アグロ
#万願果 #Banngannka
```

## 7. Firestore設計

### 7.1 フレンドデータ

```
users/{uid}/friends/{friendUid}/
  ├── status: "pending" | "accepted" | "blocked"
  ├── requestedBy: string (uid)
  ├── createdAt: timestamp
  └── lastBattleAt: timestamp | null
```

### 7.2 プロフィールデータ

```
users/{uid}/
  ├── displayName: string
  ├── friendCode: string
  ├── title: string
  ├── iconId: string
  ├── sleeveId: string
  ├── favoriteAspect: string (auto)
  ├── stats: { wins: int, losses: int, draws: int }
  └── nameChangedAt: timestamp | null
```
