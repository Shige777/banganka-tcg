# PRODUCT_REQUIREMENTS.md

## 0. 目的

このファイルは、`万願果` のプロダクト要件を AI と人間の両方が同じ前提で扱えるように整理したものです。  
**ゲームルールの正本は `GAME_DESIGN.md`** とし、このファイルはプロダクト全体の要求事項を定義します。

## 1. プロダクト定義

- タイトル: `万願果`
- 読み: `ばんがんか`
- ジャンル: `1vs1 対戦型デジタルTCG`
- 世界観ジャンル: `異時代・異時空交錯ファンタジー`
- 提供形態: `iOS先行 → Steam → Android`（詳細は §4.3 参照）
- コアファンタジー:
  - プレイヤーは `果求者` として `願主` に力を貸し、`交界` で `万願果` を巡って戦う。

## 2. 目標

このプロダクトの目標は以下です。

- 初見プレイヤーが、迷わず対戦まで到達できること
- 対戦中に、盤面・攻撃・界律・願力の関係が読みやすいこと
- 世界観、カード、UI が一貫して `万願果` の体験として感じられること
- MVP段階でも「正式サービスに育てられる土台」があること

## 3. プレイヤー体験の柱

- ひと目で雰囲気が伝わる異時代・異時空交錯ファンタジー世界観
- 直撃の緊張感がある戦闘型TCG
- 願相による視覚的に分かりやすいデッキ個性
- 5画面構成で迷いにくいナビゲーション
- カードゲーム経験者にとって自然な構築・対戦・閲覧導線

## 4. MVPスコープ

### 4.1 MVPで必須とするもの

**画面・ナビゲーション:**
- ホーム / バトル / カード / ストーリー / ショップ の5画面
- 下部ナビゲーションによる5画面遷移
- デッキ選択・構築画面（DECK_BUILDER_SPEC.md）— **MVPから全機能実装**。プリセット選択 + カスタムデッキ構築（34枚・同名3枚上限・フィルタ/ソート付き）を提供する

**対戦:**
- 1vs1 オンライン対戦ループ（Firebase Realtime Database 同期）
- ランクマッチ（26段ELOレーティング・月次シーズン制）
- ドラフトモード（7勝3敗制・3択×12回ピック）
- AI Bot対戦（AI_BOT_SPEC.md）— DAU少数期の体験維持
- フレンド対戦（SOCIAL_SPEC.md）
- 願主 / 顕現 / 詠術 / 界律 の基本表現
- 願相の色 + アイコン二重識別（ACCESSIBILITY_SPEC.md）
- ロビーから対戦開始までの一連導線
- チュートリアル（TUTORIAL_FLOW.md 12ステップ）
- パック開封演出（レアリティ別アニメーション・予兆演出・天井カウンター）
- リプレイ記録（ローカル保存）
- データ引き継ぎ（Firebase Auth連携: Apple ID / Google）
- 後攻+1手札のバランス補正

**バックエンド:**
- Firebase Authentication（匿名 + Apple Sign-in）
- Cloud Firestore（ユーザーデータ・デッキ・カード所持）
- Cloud Functions（ゲームロジック検証）
- Firebase Analytics + Crashlytics

**マネタイゼーション:**
- パックシステム（シリーズ別パック + 願相ピックアップ）
- バトルパス（願道パス: 30レベル）
- リワード広告（AdMob、モバイルのみ）
- IAP（願晶5種 + スターターバンドル）
- カード生成/分解（クラフト）
- デイリー/ウィークリーミッション + ログインストリーク
- デイリーショップ

**ソーシャル:**
- フレンドシステム（コード追加・申請・承認）
- 基本エモート6種
- プレイヤープロフィール

### 4.2 MVPで扱わないもの

- リプレイ共有 / 観戦（リプレイ記録はMVPに昇格）
- 構築制限イベント
- Steam / Android 版
- 大量のPvEコンテンツ
- 高度なソーシャル機能（ギルド・チャット等）
- ローカライゼーション（日本語のみ）
- プッシュ通知の本格運用（基本実装のみ）

### 4.3 プラットフォーム展開計画

| 順序 | プラットフォーム | 時期 | 備考 |
|------|-------------|------|------|
| 1 | **iOS** | MVP | App Store先行リリース |
| 2 | **Steam** | MVP安定後（3〜6ヶ月） | 広告なし、1.3xミッション報酬 |
| 3 | **Android** | Steam後（6〜12ヶ月） | iOS同等の広告+IAP |

クロスプラットフォーム:
- アカウント連携（Apple ID / Steam ID のリンク）
- クロスプレイ（iOS ↔ Steam ↔ Android）
- 有償通貨（願晶）はプラットフォーム別管理（ストア規約準拠）
- 無償通貨（ゴールド）はクロスプラットフォーム共有

> ※ v0.5.5（2026-03-25）にて、バランスシミュレーション（108,000戦）の結果を受けてMVPスコープを更新。

## 5. 成功条件

MVP成功条件:

- 初回起動から対戦開始までの流れが成立している
- UI上で主要機能が視覚的に区別できる
- `GAME_DESIGN.md` のコア戦闘ルールに沿った対戦体験ができる
- ホーム / バトル / カード / ストーリー / ショップが最低限成立している

## 6. 非機能要件

- モバイル前提で視認性が高いこと
- WCAG 2.1 AA準拠のコントラスト比（`ACCESSIBILITY_SPEC.md §2.2`）
- タップターゲット最小44×44pt（`ACCESSIBILITY_SPEC.md §3.1`）
- 願相は色+アイコンの二重識別（`ACCESSIBILITY_SPEC.md §1.2`）
- 画面遷移が行き止まりにならないこと
- 重要状態は UI で明示されること
- バトル中60fps / メニュー30fps（`PERFORMANCE_SPEC.md §2`）
- iPhone SE 2 以上で動作（`PERFORMANCE_SPEC.md §1.1`）
- アプリ初期DLサイズ 200MB以下（`PERFORMANCE_SPEC.md §4`）
- コールドスタート 5秒以内（`PERFORMANCE_SPEC.md §6`）

## 7. 情報設計方針

- 最上位ナビは 5 画面固定
- 各画面には主目的を 1 つだけ置く
- 主CTAは常に明確にする
- テキストより視覚トークンで判別できる設計を優先する

## 8. AI実装ルール

AIはこのファイルを使って以下を守ること。

- `GAME_DESIGN.md` を上書きしない
- MVPスコープ外の大機能を勝手に追加しない
- 5画面構成を崩さない
- 願相・世界観・用語を統一する
- デバッグUIではなく正式UIを前提に構成する
