# PERFORMANCE_SPEC.md
> 万願果 パフォーマンス要件定義 v1.0

## 0. 目的

ターゲットFPS、メモリバジェット、アプリサイズ、最低動作端末、画面遷移速度などの非機能要件を数値で定義する。開発中の最適化判断基準として使用する。

## 2. フレームレート

| 場面 | ターゲット | 許容最低 | 備考 |
|------|-----------|---------|------|
| バトル中（通常） | **60fps** | 45fps | ドロップは1秒に3フレームまで許容 |
| バトル中（SSR演出） | **60fps** | 30fps | 一時的な低下は許容 |
| メニュー/一覧 | **30fps** | 24fps | GPU負荷軽減のため制限 |
| パック開封 | **60fps** | 30fps | 演出品質のため60推奨 |
| ストーリー再生 | **30fps** | 24fps | テキスト中心のため |

## 3. メモリバジェット

### 3.1 総メモリ上限

| 端末 | アプリ使用メモリ上限 | 備考 |
|------|-------------------|------|
| iPhone SE 2/3（3GB RAM） | **800MB** | OS・バックグラウンドに1.5GB確保 |
| iPhone 13〜（4GB+ RAM） | **1.2GB** | 余裕あり |

### 3.2 カテゴリ別バジェット（3GB端末基準）

| カテゴリ | 上限 | 備考 |
|---------|------|------|
| カードテクスチャキャッシュ | **200MB** | LRU方式で古いものから解放 |
| UIテクスチャ | **100MB** | アトラス化で圧縮 |
| バトルエフェクト | **80MB** | パーティクルプール管理 |
| オーディオ | **50MB** | BGM: ストリーミング、SE: メモリ常駐 |
| スクリプト・ロジック | **100MB** | Mono Heap |
| システム・バッファ | **150MB** | Unity内部・レンダリングバッファ |
| **予備** | **120MB** | 安全マージン |
| **合計** | **800MB** | |

### 3.3 カード画像管理

- カードイラスト: 512×512 PNG（圧縮後 約50〜100KB/枚）
- 同時メモリ保持: **最大200枚分**
- 画面外のカード画像は LRU キャッシュで自動解放
- サムネイル（一覧用）: 128×128（約10KB/枚）— 別途キャッシュ

## 4. アプリサイズ

| 項目 | 上限 | 備考 |
|------|------|------|
| **初期ダウンロード** | **200MB以下** | App Store WiFi不要DL上限に収める |
| **インストール後** | **500MB以下** | カード画像等のオンデマンドDL含む |
| **アセットバンドル（追加DL）** | 各50MB以下 | 新シリーズ追加時 |

### 4.1 サイズ削減戦略

- テクスチャ: ASTC圧縮（iOS標準）
- オーディオ: BGMはOgg Vorbis（128kbps）、SEはWAV 16bit
- カードイラスト: 初回起動時にスターターデッキ分のみ同梱、残りはオンデマンドDL
- Unity: IL2CPP + Strip Engine Code でバイナリ縮小

### 5.1 帯域要件

## 6. 起動・遷移速度

| 操作 | 目標 | 許容最大 |
|------|------|---------|
| **コールドスタート（起動→ホーム）** | **3秒** | 5秒 |
| **画面遷移（ナビタブ切替）** | **0.3秒** | 0.5秒 |
| **画面遷移（push/pop）** | **0.3秒** | 0.5秒 |
| **マッチ開始（ルーム参加→バトル画面）** | **2秒** | 4秒 |
| **パック開封（タップ→カード表示）** | **1秒** | 2秒 |
| **カード一覧読み込み（全カード）** | **0.5秒** | 1秒 |

## 7. バッテリー・発熱

### 7.1 目標

| 場面 | バッテリー消費率（1時間あたり） | 備考 |
|------|---------------------------|------|
| バトル連続プレイ | **≤20%** | iPhone 14基準 |
| メニュー操作 | **≤10%** | 30fps制限で軽減 |

### 7.2 対策

- メニュー画面は30fps固定（Application.targetFrameRate切替）
- バトル中のパーティクル同時数制限（ANIMATION_SPEC.md §9.2参照）
- バックグラウンド遷移時にレンダリング完全停止
- 端末温度検知API（ProcessInfo.thermalState）で発熱検知 → エフェクト品質自動低下

## 8. テスト基準

### 8.1 パフォーマンステスト項目

| テスト | 合格基準 | ツール |
|--------|---------|--------|
| バトル中FPS | 60fps中央値、45fps以上99th percentile | Unity Profiler |
| メモリピーク | iPhone SE3で800MB以下 | Xcode Instruments |
| 起動時間 | コールドスタート5秒以内 | Xcode Instruments |
| アプリサイズ | IPA 200MB以下 | Xcode Archive |
| 1時間バッテリー | 20%以下消費 | Xcode Energy Diagnostics |
| カード一覧スクロール | 60fps維持（200枚表示時） | Unity Profiler |

### 8.2 自動計測

- CI/CDパイプラインにパフォーマンスベンチマークを組み込み
- ビルドごとに起動時間・メモリピーク・IPAサイズを自動記録
- 閾値超過時にアラート
