# ACCESSIBILITY_SPEC.md
> 万願果 アクセシビリティ設計 v1.0

---

## 0. 目的

6つの願相をカラーで識別するゲームにおいて、色覚多様性をはじめとするアクセシビリティ対応を定義する。App Store審査基準・WCAG 2.1 AA準拠を目指し、すべてのプレイヤーが快適にプレイできる設計を保証する。

---

## 1. 色覚多様性対応

### 1.1 課題

万願果は6願相をカラーで区別する（空=#4DA3FF / 曙=#FF5A36 / 穏=#59C36A / 妖=#9A5BFF / 遊=#F4C542 / 玄=#3A3A46）。以下の色覚タイプで識別困難が生じうる。

| 色覚タイプ | 影響 | 混同しやすい願相ペア |
|-----------|------|-------------------|
| **P型（1型）** | 赤の感度低下 | 曙 ↔ 穏（赤 ↔ 緑） |
| **D型（2型）** | 緑の感度低下 | 曙 ↔ 穏（赤 ↔ 緑）、遊 ↔ 穏（黄 ↔ 緑） |
| **T型（3型）** | 青の感度低下 | 空 ↔ 妖（青 ↔ 紫） |
| **全色覚異常** | 全色の感度低下 | 全ペア |

### 1.2 二重識別システム（MVP必須）

カラーだけに依存せず、**色 + アイコン/パターン**の二重識別を必ず実装する。

| 願相 | HEX | アイコンモチーフ | パターン（カード枠） | Unicode候補 |
|------|-----|----------------|--------------------|----|
| **空** | #4DA3FF | 風の渦 / 羽 | 横波線ストライプ | 🌀 |
| **曙** | #FF5A36 | 炎 / 太陽 | 斜めストライプ | 🔥 |
| **穏** | #59C36A | 木の葉 / 芽 | ドット格子 | 🌿 |
| **妖** | #9A5BFF | 三日月 / 星 | ダイヤモンド格子 | 🌙 |
| **遊** | #F4C542 | 歯車 / 風車 | ジグザグ | ⚙️ |
| **玄** | #3A3A46 | 盾 / 鍵 | クロスハッチ | 🔒 |

### 1.3 アイコン適用箇所（MVP）

以下すべての箇所で、カラーに加えアイコンを**常時表示**する。

- カードの願相チップ（コスト横 or 名前帯）
- カード一覧のフィルタボタン
- デッキ構築画面の願相フィルタ
- バトル画面の顕現列（各顕現カードの願相チップ）
- 願主の願相表示
- 界律カードの願相表示
- ショップのパック表示（願相ピックアップ時）

### 1.4 カラーブラインドモード（設定画面）

デフォルトは通常カラー。設定画面から以下を切替可能にする。

| モード | 内容 |
|--------|------|
| **通常** | 標準カラー + アイコン |
| **P/D型サポート** | 赤緑の識別を強化（曙→オレンジ寄り、穏→シアン寄りに補正）+ アイコン拡大 |
| **T型サポート** | 青紫の識別を強化（空→シアン、妖→マゼンタに補正）+ アイコン拡大 |
| **ハイコントラスト** | 全願相を明度差の大きい6色に変更 + アイコン + パターン全表示 |

### 1.5 補正カラーパレット

| 願相 | 通常 | P/D型補正 | T型補正 | ハイコントラスト |
|------|------|----------|---------|----------------|
| 空 | #4DA3FF | #4DA3FF | #00CED1 | #FFFFFF |
| 曙 | #FF5A36 | #FF8C00 | #FF5A36 | #FF0000 |
| 穏 | #59C36A | #00BCD4 | #59C36A | #00FF00 |
| 妖 | #9A5BFF | #9A5BFF | #FF00FF | #FF00FF |
| 遊 | #F4C542 | #F4C542 | #F4C542 | #FFFF00 |
| 玄 | #3A3A46 | #3A3A46 | #3A3A46 | #808080 |

## 2. テキスト・タイポグラフィ

### 2.1 テキストサイズ設定

設定画面から3段階で変更可能にする。

| サイズ | 本文 | カード名 | 効果テキスト | ボタン |
|--------|------|---------|-------------|--------|
| **小** | 13pt | 14pt | 11pt | 15pt |
| **中（デフォルト）** | 15pt | 16pt | 13pt | 17pt |
| **大** | 18pt | 19pt | 15pt | 20pt |

### 2.2 コントラスト要件（WCAG 2.1 AA）

| 要素 | 前景色 | 背景色 | 最低コントラスト比 |
|------|--------|--------|------------------|
| 本文テキスト | 白系 (#FFFFFF / #F0F0F0) | 暗色パネル (#1A1A2E〜#2A2A3E) | **4.5:1 以上** |
| 大文字（見出し/ボタン） | 白 | 暗色パネル | **3:1 以上** |
| カードコスト数字 | 白 (#FFFFFF) | 願相カラー背景 | **3:1 以上** |
| 願相チップテキスト | 白 | 願相カラー | **3:1 以上** |

### 2.3 玄の特殊対応

玄 (#3A3A46) は暗色のため、暗背景上で見えにくい。以下の対策を適用する。

- 玄のチップ/アイコンには **白or明灰の縁取り（1.5px）** を付与
- 玄のカード枠は **明灰のインナーグロウ** で識別
- テキスト表示時は白文字 + 玄背景ではなく、**白背景 + 玄文字** or **玄背景 + 白文字 + 太縁** を使用

## 3. タッチ操作

### 3.1 タップターゲットサイズ

| 要素 | 最小サイズ | 推奨サイズ | 備考 |
|------|-----------|-----------|------|
| ボタン | 44×44pt | 48×48pt | Apple HIG準拠 |
| カード（一覧グリッド） | 60×84pt | 70×98pt | カード比率 5:7 |
| カード（手札バトル中） | 50×70pt | 60×84pt | 5枚並列を想定 |
| ナビゲーションタブ | 44×44pt | 48×48pt | 5タブ横並び |
| フィルタチップ | 36×36pt | 44×44pt | 横スクロール時 |

### 3.2 操作補助

- 長押し → カード詳細プレビュー（拡大表示、効果テキスト全文）
- ダブルタップ → カードプレイ確認（誤タップ防止）
- スワイプ → ターン終了確認（ボタン以外の代替操作）

## 4. 視覚エフェクトへの配慮

### 4.1 アニメーション低減モード

iOS の「視差効果を減らす」設定（UIAccessibility.isReduceMotionEnabled）を検知し、以下を適用する。

| 通常モード | 低減モード |
|-----------|-----------|
| カードプレイ時のスライド+回転アニメーション | フェードイン（0.2s） |
| 攻撃エフェクト（パーティクル+画面揺れ） | フラッシュ（0.1s）のみ |
| HPゲージの色変化演出 | 色変化のみ |
| 勝利/敗北の全画面エフェクト | テキスト表示のみ |
| パック開封演出（フルアニメーション） | 結果一覧即表示 |

### 4.2 画面フラッシュ制限

- 1秒間に3回以上の全画面フラッシュを禁止（光過敏性てんかん対策、WCAG 2.3.1）
- 赤色フラッシュの面積を画面の25%以下に制限

## 5. 音声・聴覚

### 5.1 VoiceOver対応（MVP後推奨）

iOS VoiceOverが有効な場合、以下の読み上げラベルを設定する。

| UI要素 | 読み上げ例 |
|--------|-----------|
| カード | 「曙の顕現、暁門の剣士、コスト3、戦力5000、願撃2、ラッシュ」 |
| HPゲージ | 「HP 75、相手 HP 50」（数値表示） |
| ターン表示 | 「ターン5、あなたのターン」 |
| 手札 | 「手札5枚。1枚目、穏の詠術、癒しの風…」 |
| 願主 | 「あなたの願主、レベル2、戦力6000、待機中。Lv2スキル『突撃令』使用可能」 |

### 5.2 字幕・テキスト表示

- チュートリアルのナルのセリフは全て**テキスト表示**（ボイスなしMVPでも対応済み）
- SE/BGM設定とは別に、**通知テキスト表示ON/OFF** を設けるon the HUD
- 対戦中のシステムメッセージ（「相手のターン」「直撃！」）は常にテキスト表示

## 6. 設定画面の構成

アカウント設定画面（ASSET_ACCOUNT_SETTINGS.md）に以下のアクセシビリティセクションを追加する。

```
設定 > アクセシビリティ
├── カラーモード: 通常 / P/D型サポート / T型サポート / ハイコントラスト
├── テキストサイズ: 小 / 中 / 大
├── アニメーション低減: システム設定に従う / 常にON / 常にOFF
├── カード長押し詳細: ON / OFF
└── HPゲージ数値表示: ON（デフォルト）/ OFF
```

## 7. テスト要件

### 7.1 色覚シミュレーションテスト

以下のツールで全画面をスクリーンショット検証する。

- **Sim Daltonism**（macOS）: P型/D型/T型シミュレーション
- **Color Oracle**（クロスプラットフォーム）: 全タイプ一括確認

検証基準:
- 6願相がすべてのモードで識別可能であること
- アイコンのみ（色なし）でも願相を特定できること
- カード一覧で隣接するカードの願相が混同されないこと

### 7.2 コントラスト比テスト

- 全テキスト要素で WCAG 2.1 AA 準拠を確認
- ツール: **Colour Contrast Analyser**（TPGi）

### 7.3 VoiceOver実機テスト（MVP後）

- iPhone SE3 + VoiceOver でホーム→バトル→勝利の一連フローを完走できること
- 全ボタンにaccessibilityLabelが設定されていること
