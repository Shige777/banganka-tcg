# NOTIFICATION_SPEC.md
> 万願果 通知設計 v1.0

---

## 0. 目的

プッシュ通知・ローカル通知・アプリ内通知の種類、送信タイミング、ディープリンク、ユーザー設定を定義する。

---

## 1. 通知チャネル

| チャネル | 技術 | 対象場面 |
|---------|------|---------|
| **プッシュ通知** | Firebase Cloud Messaging (FCM) | アプリ非起動時のリマインド |
| **ローカル通知** | iOS UNUserNotificationCenter | オフラインでも機能するリマインド |
| **アプリ内通知** | カスタムUI（トースト/バッジ） | アプリ起動中のリアルタイム通知 |

## 3. 送信ルール

### 3.1 クールダウン

| ルール | 値 |
|--------|-----|
| 同一ユーザーへの1日最大プッシュ通知数 | **5件** |
| 同一種類の通知の最小間隔 | **4時間** |
| 深夜帯（22:00〜8:00 ローカル時刻）の送信 | **禁止**（キューに入れて8:00に送信） |

### 3.2 優先度

通知が5件上限に達した場合、以下の優先度で送信する。

| 優先度 | 種類 |
|--------|------|
| 1（最高） | メンテナンス告知 |
| 2 | フレンド対戦招待 |
| 3 | イベント開始 |
| 4 | デイリーミッションリセット |
| 5（最低） | 長期離脱リカバリ |

## 4. ユーザー設定

設定 > 通知 から以下を個別にON/OFF可能にする。

```
通知設定
├── プッシュ通知全体: ON / OFF
├── 対戦招待: ON / OFF
├── フレンド申請: ON / OFF
├── ミッション・イベント: ON / OFF
├── メンテナンス: ON（変更不可）
├── 新コンテンツ: ON / OFF
└── ログインリマインド: ON / OFF
```

- メンテナンス通知は常にON（ユーザーが無効化できない）
- デフォルトはすべてON

## 5. 実装

### 5.1 FCMトークン管理

```
初回起動 / ログイン時:
  → FCMトークンを取得
  → Firestore: users/{uid}/fcmTokens/{deviceId} に保存
  → トークンリフレッシュ時に自動更新
```

### 5.2 Cloud Functions送信

```javascript
// 例: フレンド対戦招待通知
exports.sendBattleInviteNotification = functions.firestore
  .document('rooms/{roomId}')
  .onCreate(async (snap, context) => {
    const room = snap.data();
    const targetToken = await getFCMToken(room.invitedUid);
    await admin.messaging().send({
      token: targetToken,
      notification: {
        title: '対戦の招待',
        body: `${room.hostName}さんから対戦の招待が届いています`,
      },
      data: {
        deeplink: `banngannka://battle/join/${room.roomId}`,
      },
    });
  });
```

### 5.3 ディープリンク

| スキーム | パス | 遷移先 |
|---------|------|--------|
| `banngannka://` | `home` | ホーム画面 |
| | `battle/join/{roomId}` | ルーム参加画面 |
| | `friends` | フレンド画面 |
| | `shop` | ショップ画面 |
| | `event/{eventId}` | イベント詳細 |
| | `news` | お知らせ一覧 |
| | `battlepass` | バトルパス画面 |
