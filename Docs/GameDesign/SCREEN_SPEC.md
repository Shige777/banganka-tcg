# SCREEN_SPEC.md

## 0. 目的

このファイルは、各画面の目的・必須UI・表示情報・主要アクション・遷移先を定義します。  
AIはこのファイルをもとに、画面を追加・再構築・改善してください。

## 1. グローバル要件

- 下部ナビゲーションバーを常設する
- ナビ項目は以下の5つ:
  - `ホーム`
  - `バトル`
  - `カード`
  - `ストーリー`
  - `ショップ`
- どの画面からでも主要画面に1タップで戻れること
- 画面名、ボタン名、見出しは日本語を基本とする

## 2. ホーム

### 2.1 目的

- 最初に見る画面として世界観と現在の導線を示す
- 次にやるべき行動を明確にする

### 2.2 必須UI

- タイトル or ロゴ
- メイン背景
- 主CTA `バトルへ`
- サブCTA:
  - `カード`
  - `ストーリー`
  - `ショップ`
- 現在の注目バナー
- プレイヤー情報サマリー

### 2.3 表示すべき情報

- ゲームタイトル
- 現在のおすすめ導線
- 軽い世界観テキスト
- プレイヤー名またはプロフィール導線

### 2.4 遷移

- バトルボタン → `バトル`
- カードボタン → `カード`
- ストーリーボタン → `ストーリー`
- ショップボタン → `ショップ`

## 3. バトル

### 3.1 目的

- 対戦を始めるための入口をまとめる
- ルーム作成、参加、対戦開始の状態を分かりやすくする

### 3.2 必須UI

- 画面タイトル
- モード説明領域
- `ルーム作成`
- `ルーム参加`
- `対戦開始`
- 接続 / 同期状態表示
- ルームID表示欄

### 3.3 表示すべき情報

- 参加状態
- 相手接続状況
- ルーム識別情報
- ボタンの活性 / 非活性状態

### 3.4 遷移

- 対戦開始成功 → マッチ画面
- 戻る / ナビ操作 → 他画面

## 7b. バトル中操作UIフロー（v0.5.0追加）

### 7b.1 カードプレイフロー
1. 手札のカードを**タップ** → カードが拡大表示（詳細確認可能）
2. 拡大カードを**上方へドラッグ** → プレイ確定ゾーンにスナップ
3. CP不足の場合 → カードが赤く点滅して手札に戻る + ハプティック（Error）
4. 詠術でターゲットが必要な場合 → ターゲット選択モードに遷移（7b.3参照）

### 7b.2 アタック宣言フロー
1. 攻撃可能な顕現/願主を**タップ** → 赤い矢印が表示され、攻撃先選択モードに入る
2. 攻撃対象（相手顕現/願主）を**タップ** → 攻撃宣言確定
3. 空き領域をタップ → 攻撃キャンセル
4. 相手にBlocker持ちがいる場合 → 相手クライアントにブロック選択UIが表示（5秒タイムアウト）

### 7b.3 ブロック選択フロー（防御側）
1. 攻撃宣言を受けたとき → 画面上部に「ブロックしますか？」バナー表示
2. Blocker持ちの待機中カードが**ハイライト**される
3. ハイライトカードを**タップ** → ブロック宣言。タップされたカードが攻撃先を引き受ける
4. 「ブロックしない」ボタン or 5秒経過 → ブロックなし（攻撃がそのまま通る）

### 7b.4 ターゲット選択フロー（詠術・OnPlay効果）
1. 対象選択が必要な効果を発動 → ターゲット可能なカードが**ハイライト**
2. ハイライトカードを**タップ** → ターゲット確定
3. 対象が「敵味方問わず」の場合 → 全対象がハイライト
4. 画面下部に「キャンセル」ボタン表示 → タップで手札に戻す（CP返却）

### 7b.5 マリガンフロー
1. 初期手札5枚が表示される（30秒タイマー表示）
2. 交換したいカードを**タップ**で選択（トグル式、複数可）
3. 「交換」ボタンをタップ → 選択カードをデッキに戻してシャッフル後同数ドロー
4. 「このまま開始」ボタン → マリガンなしで開始
5. タイムアウト → 現在の手札で自動開始

## 7c. マッチリザルト画面（v0.5.3追加）

### 7c.1 目的
- 勝敗結果（鯱鉾勝利 / 塗り勝利）と獲得報酬を明確に伝える
- 次のアクションへの導線を提供する

### 7c.2 必須UI
- 勝利/敗北/引き分けテキスト（大）
- 勝利タイプ表示：
  - 鯱鉾勝利（対手HP=0の直接KO）
  - 塗り勝利（24ターン限界での HP比較）
- リプレイ用マッチID表示
- 最終HP表示（両プレイヤー）
- 報酬一覧（以下を縦に並べる）:
  - ゴールド獲得量（例: +50G）
  - バトルパスEXP獲得量（例: +30 BP EXP）
  - ミッション進捗更新（変化があったミッションのみ表示）
- 「もう一度」ボタン → マッチメイキングに戻る
- 「ホームへ」ボタン → ホーム画面
- 「リプレイを見る」ボタン（REPLAY_SPEC.md 連動）
- フレンド対戦時: 「フレンドに再戦を申し込む」ボタン

### 7c.3 表示アニメーション
- 勝利/敗北演出（ANIMATION_SPEC.md §4）の後にフェードインで表示
- 最終HP の視覚比較（バー表示）
- 報酬はカウントアップアニメーション（各項目0.3s）で表示
- レベルアップがあった場合はレベルアップジングル再生

### 7d.3 設定項目

```
│   ├── SE音量: [スライダー 0〜100]
│   └── バトル中BGM: 通常 / ダイナミック / OFF
├── 通知（NOTIFICATION_SPEC.md §6連動）
│   ├── プッシュ通知: ON/OFF
│   ├── バトル招待: ON/OFF
│   ├── フレンド申請: ON/OFF
│   ├── デイリーミッション: ON/OFF
│   ├── イベント開始: ON/OFF
│   └── 復帰促進: ON/OFF
├── 対戦
│   ├── 演出速度: 通常 / 高速
│   ├── エモート受信: ON/OFF
│   └── リプレイ公開: ON/OFF（REPLAY_SPEC.md連動）
├── アクセシビリティ（ACCESSIBILITY_SPEC.md連動）

│   └── お問い合わせ
├── その他
│   ├── 利用規約
│   ├── プライバシーポリシー

```

## 8. ローディング・遷移・エラー状態（v0.5.0追加）

### 8.1 ローディングインジケーター表示基準

| 待機時間 | 表示内容 |
|---------|---------|
| 0〜300ms | 何も表示しない（即時遷移に見せる） |
| 300ms〜2秒 | スピナーアイコン（画面中央、48×48pt） |
| 2秒〜5秒 | スピナー + 「読み込み中...」テキスト |
| 5秒以上 | スピナー + 「通信に時間がかかっています」+ キャンセルボタン |

### 8.2 スケルトンスクリーン

以下の画面は、データ読み込み中にスケルトンUI（灰色のプレースホルダー図形）を表示する。

| 画面 | スケルトン要素 |
|------|-------------|
| ホーム | バナー枠、ミッション進捗バー、バトルCTAボタン |
| カード一覧 | カードグリッド（6〜12枚分の灰色矩形） |
| ショップ | 商品グリッド（4〜6枚分の灰色矩形）、通貨表示枠 |
| デッキ構築 | デッキリスト（3〜5枠の灰色矩形） |
| ストーリー | 章ノードマップ（ノード位置は固定、内容が灰色） |

> バトル画面はスケルトン不要（マッチング待機画面が別途存在するため）。

### 8.3 画面遷移中のエラー処理

| 遷移シナリオ | エラー時の挙動 |
|------------|-------------|
| タブ切替中にネットワークエラー | 前の画面に留まる + トースト通知「通信エラー。再度お試しください」 |
| バトル開始中（マッチング→対戦画面） | マッチング画面に留まる + モーダル「対戦の開始に失敗しました」+ リトライボタン |
| パック購入→開封画面遷移中 | ショップ画面に留まる + モーダル（ERROR_HANDLING.md §2.1 パック購入中通信断の処理に従う） |
| ストーリーノード選択→バトル開始 | ストーリー画面に留まる + トースト通知 |
| 設定変更の保存中 | 設定画面に留まる + トースト「保存に失敗しました」+ 自動リトライ（3回） |

### 8.4 空状態（Empty State）

データが存在しない場合の表示。

| 画面 | 空状態メッセージ | CTA |
|------|--------------|------|
| カード一覧（フィルター結果0件） | 「条件に一致するカードがありません」 | 「フィルターをリセット」 |
| デッキ一覧（デッキ未作成） | 「デッキがまだありません」 | 「新しいデッキを作る」 |
| 対戦履歴（0件） | 「まだ対戦していません」 | 「バトルに行く」 |
| フレンドリスト（0人） | 「フレンドがいません」 | 「フレンドを追加」 |

## 9. AI実装ルール

- 各画面の主目的を崩さない
- 1画面に主導線を詰め込みすぎない
- モバイル縦持ち前提で設計する
- ダミーデータでも画面意図が伝わる状態にする
