## 2. 🔴 致命的な不足（競合全社が持つ基盤機能）

### 2.1 アクセシビリティ仕様書（ACCESSIBILITY_SPEC.md）

**影響度: 最高** — App Store審査にも直結

万願果は6つの願相をカラーで区別するゲーム。**色覚多様性への対応なし**は致命的。

競合他社の対応:
- **Hearthstone**: カードレアリティを色+形状で区別、テキストサイズ変更可能
- **MTG Arena**: カラーブラインドモード搭載（Mana色をアイコンで識別）
- **Shadowverse**: カードクラスをアイコン+色で二重識別
- **デュエプレ**: 文明をアイコン+テクスチャ+色で三重識別

必要な内容:
- 色覚シミュレーション（P型/D型/T型）で6願相が識別可能か検証
- 各願相にアイコン/パターンを併用した二重識別システム
- テキストサイズ設定（小/中/大）
- カードテキストの読み上げ対応（VoiceOver）
- コントラスト比のWCAG 2.1 AA準拠チェック
- 効果音/振動による視覚以外のフィードバック

---

### 2.2 カードバランス調整ポリシー（BALANCE_POLICY.md）

**影響度: 最高** — 運営開始後の信頼に直結

GAME_DESIGN.mdにバランス関連の数値はあるが、**運営開始後のナーフ/バフの手続き・告知・補償**の規定がない。

競合他社の対応:
- **Hearthstone**: ナーフ時にフルダスト返却（2週間限定）、バフは返却なし
- **MTG Arena**: Ban/制限リスト、ワイルドカード補償
- **Shadowverse**: ナーフ時にヴァイアル全額返却、事前告知+メンテナンス実施
- **Master Duel**: 禁止/制限リスト更新、CPポイント返却
- **デュエプレ**: 殿堂入り/プレミアム殿堂、分解時全額返却

必要な内容:
- ナーフ/バフの判断基準（使用率・勝率の閾値）
- 告知フロー（何日前に発表するか）
- 補償ルール（ナーフカードの生成素材全額返却期間）
- 禁止/制限リストの運用方針（将来のランク戦向け）
- バランスパッチの頻度（月次/シーズンごと）
- データ収集期間（新カード実装後の観察期間）

### 2.3 アニメーション・演出タイミング仕様（ANIMATION_SPEC.md）

**影響度: 高** — ゲームフィール（手触り）に直結

ASSET_CARD_EFFECTS.md にカード背景エフェクトはあるが、**ゲームプレイ中のアニメーション全体のタイミング設計**がない。TCGのゲームフィールの70%はアニメーション演出で決まる。

競合他社の対応:
- **Hearthstone**: カードプレイ→着地→能力発動が厳密にタイムライン管理（約0.5s+0.3s+0.5s）
- **Shadowverse**: 進化時フルスクリーンカットイン、攻撃エフェクト体系
- **デュエプレ**: 召喚→場に出る→能力テキスト表示のシーケンスが超高品質
- **Master Duel**: 召喚演出レベルが3段階（N/SR/UR）

必要な内容:
- カードプレイシーケンス（手札→盤面のアニメーション時間、イージング）
- 攻撃シーケンス（宣言→移動→衝突→退場/生存の各フレーム数）
- 願撃演出（ゲージ変動アニメーション、画面揺れ、SE同期）
- 願主レベルアップ演出（カットイン、パーティクル、SE）
- 界律設置/上書き演出（フルフィールドエフェクト）
- 勝利/敗北演出のタイムライン
- パック開封演出（レアリティ別のエフェクト段階）
- 「演出スキップ」オプション（上級者向け高速モード）
- 各アニメーションのスキップ可能タイミング

**影響度: 高** — リテンションとバイラルに直結

- **デュエプレ**: フレンド、ルームマッチ、デッキコピー
- **Master Duel**: フレンド、デッキ公開、リプレイ共有

必要な内容:
- フレンドシステム（フレンドコード、申請/承認、上限数）
- フレンド対戦（ルーム招待の簡略化）
- デッキコード（テキスト形式のデッキシェア/インポート）
