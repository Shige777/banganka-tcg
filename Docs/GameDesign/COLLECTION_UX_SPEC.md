# COLLECTION_UX_SPEC.md
> 万願果 カード一覧・コレクション画面UX設計 v1.0

---

## 0. 目的

SCREEN_SPEC.md §4 のカード画面をより詳細に設計し、コレクション体験を競合レベルまで引き上げる。カード図鑑・達成率・Foil表示・NEW表示など、所持体験を充実させる。

---

## 1. カード一覧画面

### 1.1 表示モード

| モード | 説明 | 切替 |
|--------|------|------|
| **所持カードのみ** | 所持しているカードだけ表示 | デフォルト |
| **図鑑モード** | 未所持カードもシルエットで表示 | トグルボタン |

### 1.2 カードグリッド

- 2列（大きめ表示）/ 3列（一覧性重視）の切替
- カードサイズ比: 5:7
- カード表示内容:
  - イラスト（サムネイル 128×179）
  - コスト（左上、願相カラー背景）
  - 願相アイコン（ACCESSIBILITY_SPEC.md §1.2準拠）
  - レアリティ帯（下部、色帯: C=灰 / R=青 / SR=金 / SSR=虹）
  - 所持枚数バッジ（右下: `×3`）
  - **NEW バッジ**（未確認カードの右上に赤丸）
  - **Foil マーク**（Foil版所持時、右上にキラキラアイコン）

### 1.3 仮想スクロール（パフォーマンス最適化）

カードプール拡張（MVP 162種→将来200種以上）に備え、カードグリッドは仮想スクロール（RecyclerView相当）を実装する。

| 項目 | 仕様 |
|------|------|
| 表示バッファ | 画面内 + 上下各2行分をプリレンダリング |
| オブジェクトプール | 最大40セルをプールし再利用。画面外セルはプールに返却 |
| サムネイルキャッシュ | LRU方式。メモリキャッシュ上限 50MB、ディスクキャッシュ上限 200MB |
| スクロール速度 | 60fps 維持。フレームドロップが検知された場合はサムネイル解像度を一時的に下げる |
| 適用基準 | 表示対象カード数が **50枚以上** の場合に仮想スクロールを有効化。50枚未満は通常レンダリング |

> `PERFORMANCE_SPEC.md` のメモリ使用量上限と連動して調整すること。

### 1.4 未所持カード（図鑑モード）

- シルエット表示（カード枠 + 暗いシルエット）
- 名前: `？？？`
- コスト/戦力/効果: 非表示
- 願相/タイプのみ表示（フィルタに使えるように）
- タップ → 「このカードはまだ入手していません」+ 入手方法の示唆（「{シリーズ名}パックから入手可能」）

## 2. カード詳細画面

### 2.1 レイアウト

```
┌──────────────────────────┐
│  ← 戻る                      │
├──────────────────────────┤
│                              │
│     [カードイラスト 512×512]     │  ← ピンチで拡大可能
│                              │
├──────────────────────────┤
│  カード名         レアリティ帯   │
│  願相アイコン + 名前  コスト表示  │
├──────────────────────────┤
│  タイプ: 顕現                  │
│  戦力: 5000   願撃: 2          │
│  キーワード: ⚡速攻             │  ← タップでツールチップ
├──────────────────────────┤
│  効果テキスト（フル版）          │
│  CARD_TEXT_GUIDELINES準拠      │
├──────────────────────────┤
│  フレーバーテキスト（イタリック）   │
├──────────────────────────┤
│  所持: 3/3  Foil: 1           │
│  [デッキに追加] [生成] [分解]   │
├──────────────────────────┤
│  シリーズ: 第1弾「交界の夜明け」  │
│  カードID: MAN_EXAMPLE_01      │
└──────────────────────────┘
```

### 2.2 カードイラスト操作

| 操作 | 動作 |
|------|------|
| ピンチイン/アウト | 拡大/縮小（最大3倍） |
| スワイプ左右 | 前後のカードに切替（一覧の並び順） |
| ダブルタップ | フルスクリーンイラスト表示（UI非表示） |
| 長押し | カードを回転表示（3D風、Foil効果確認用）※MVP後 |

### 2.3 Foilカード表示

Foil版を所持している場合、詳細画面で以下の切替が可能。

- 「通常」/ 「Foil」タブ切替
- Foil版: カード全面にホログラフィック効果（Shader Graphで実装）
- 効果種類: 角度に応じて虹色の光沢が移動するパララックス効果

## 4. NEW バッジシステム

### 4.1 ルール

- パック開封・クラフト・報酬で新しいカードを入手したとき、`NEW` バッジを付与
- カード詳細画面を開いたら `NEW` バッジを消す
- ナビバーの「カード」タブに未確認カード数の赤丸バッジを表示

### 4.2 表示

- NEW バッジ: カードサムネイルの右上に赤い「N」アイコン（12pt）
- タブバッジ: 「カード」タブの右上に赤丸 + 数字（例: `3`）

## 5. フィルタ・ソート強化

DECK_BUILDER_SPEC.md §4 と共通のフィルタ基盤を使用しつつ、コレクション固有の追加を行う。

### 5.1 追加フィルタ

| フィルタ | 選択肢 | 用途 |
|---------|--------|------|
| シリーズ | 第1弾 / 第2弾 / … | シリーズ別閲覧 |
| 所持状態 | 所持のみ / 未所持のみ / 全て | 図鑑用 |
| Foil | Foil所持のみ | Foilコレクション |
| NEW | NEWのみ | 新規入手確認 |

### 5.2 追加ソート

| ソート | 用途 |
|--------|------|
| 入手日時（新しい順） | 最近入手したカード確認 |
| カードID順 | 図鑑番号順 |

## 6. クラフト/分解の導線

### 6.1 カード詳細からの操作

| ボタン | 条件 | 動作 |
|--------|------|------|
| 「生成」 | 未所持 or 3枚未満 | 生成確認ダイアログ（素材コスト表示）→ 生成 |
| 「分解」 | 1枚以上所持 | 分解確認ダイアログ（獲得素材表示）→ 分解 |
| 「デッキに追加」 | 1枚以上所持 | デッキ選択 → 追加 |

### 6.2 一括分解

カード一覧画面から「一括分解」モードに入れる。

```
「一括分解」ボタン → 分解対象選択モード
  → カードをタップで選択/解除（複数選択可）
  → 画面下部にリアルタイムで獲得素材合計を表示
  → 「分解する」→ 確認ダイアログ → 実行
```
