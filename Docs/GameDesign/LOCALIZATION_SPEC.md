## 2. テキスト管理方式

### 2.1 構造
Unity の `Localization` パッケージ（com.unity.localization）を使用。

```
Assets/Localization/
├── StringTables/
│   ├── UI.asset           (ボタン・ラベル・メニュー)
│   ├── Cards.asset        (カード名・効果テキスト・フレーバーテキスト)
│   ├── Story.asset        (ストーリーテキスト・セリフ)
│   ├── Tutorial.asset     (チュートリアルセリフ)
│   ├── System.asset       (エラーメッセージ・通知)
│   └── Shop.asset         (商品名・説明)
└── Locales/
    ├── ja.asset
    └── en.asset (MVP後)
```

### 2.2 キー命名規則
```
{カテゴリ}_{画面}_{要素}

例：
UI_HOME_BTN_BATTLE        → 「バトルへ」
UI_BATTLE_BTN_END_TURN    → 「ターン終了」
CARD_MAN_CON_01_NAME      → 「断章の剣士」
CARD_MAN_CON_01_FLAVOR    → フレーバーテキスト
STORY_CH1_SCENE01_LINE01  → ストーリーテキスト
SYS_ERR_NETWORK           → 「通信エラーが発生しました」
TUTO_STEP02_NARU_01       → ナルのセリフ
```

## 3. 翻訳方針

### 3.1 カード名
- **意訳優先**：日本語の雰囲気を保ちつつ、対象言語で自然な名前にする
- 例：「断章の剣士」→ "Swordsman of Broken Chapters" ではなく "Severed Chronicle" のように簡潔に

### 3.2 世界観用語
固有名詞は音訳（ローマ字）を基本とし、初出時にツールチップで意味を補足する。

| 日本語 | 英語表記 | 備考 |
|--------|---------|------|
| 万願果 | Banngannka | タイトル。翻訳しない |
| 交界 | Kōkai | 固有名詞として扱う |
| 願主 | Ganshu | または "Wish Bearer" |
| 果求者 | Kakkyūsha | または "Seeker" |
| 願力 | Ganriki | または "Wish Force" |
| 顕現 | Kengen | または "Manifestation" |
| 詠術 | Eijutsu | または "Chant" |
| 界律 | Kairitsu | または "World Law" |
| 願相 | Gansō | または "Aspect" |
| ナル | Naru | 固有名詞 |

### 3.3 UIテキスト
- 直訳で問題ない場合はそのまま翻訳
- 文字数制限があるボタンは各言語で収まるよう調整

## 4. フォント要件

| 言語 | フォント | フォールバック |
|------|---------|-------------|
| 日本語 | Noto Sans JP | 游ゴシック |
| 英語 | Noto Sans | Roboto |

## 5. テキスト以外のローカライゼーション

| 対象 | 方針 |
|------|------|
| 日付/時刻 | 端末のロケール設定に従う |
| 通貨表示 | 金額はApp Storeのロケール設定に従う |
| 画像内テキスト | テキストは画像に焼き込まず、UIオーバーレイで表示 |
| 音声（MVP後） | 日本語のみ。字幕で多言語対応 |
| 縦書き | 不要（横書きのみ） |

### 5.1 RTL言語対応方針（将来検討）

MVPは日本語のみだが、将来のアラビア語・ヘブライ語等のRTL（右から左）言語への対応コストを最小化するため、以下の方針を**MVP設計段階から**守る。

| 方針 | 具体策 |
|------|--------|
| レイアウトミラーリング | UI要素の左右配置をハードコードしない。`leading` / `trailing` でアンカーする（Unity の RectTransform 設定で対応可能） |
| テキスト方向 | テキスト表示コンポーネントに `textDirection` プロパティを持たせ、ロケールに応じて切替可能にする |
| アイコン方向 | 矢印・進む/戻るアイコンはRTL時に自動反転対象にする。ただし願相アイコン（剣、目等）は反転しない |
| カード枠 | コスト（左上）と戦力（右下）の位置はRTL時にミラーリングする |
| 導入時期 | POST_MVP Phase 6（ローカライゼーション拡張）で実施。MVP時はコード構造だけ準備しておく |
