# PACK_OPENING_SPEC.md
> 万願果 パック開封体験設計 v1.0

---

## 0. 目的

パック開封は万願果の課金導線における生命線であり、プレイヤーの欲望と喜悦を最大化するための仕様書である。本書では、パック選択から結果確認までの全工程における視覚・聴覚・触覚の演出設計、ならびに実装仕様を統一管理する。

参考実例：シャドウバース Shadowverse、ハースストーン Hearthstone 等のデジタルTCGにおけるパック開封演出設計を参考にしながら、万願果独自の願相システム・レアリティ表現をもって実装する。

---

## 1. 設計原則

### 1.1 脳汁全盛り（Maximum Dopamine）の要素

- **予兆演出**：開封前にレアリティを暗示し、期待値を高める
- **漸進的興奮**：通常カード → レアカード → SSRの段階的な演出強度上昇
- **触覚・視覚・聴覚の同期**：複数感覚の統合により心理的充実感を最大化
- **スクリーンショット共有誘発**：特に高レアリティ獲得時、SNS共有を促すデザイン

### 1.2 プレイヤー配慮

- **全演出スキップ可能**：設定メニューから演出簡略化モードを選択可能
- **ヘビーユーザー対応**：一括開封（10パック同時）オプションで時間効率化
- **アクセシビリティ**：色覚多様性対応・光感受性対応・ハプティクス無効化オプション（§13）

### 1.3 実装方針

- **テンポ重視**：1アクションが2秒を超えないこと（ANIMATION_SPEC.md §1に準拠）
- **パフォーマンス**：60fps 維持を目標（iPhone 12以上）、最低30fps保証（iPhone SE 3rd gen）
- **通貨・天井管理**：MONETIZATION_DESIGN.md §4に厳密に準拠

### 2.2 パック3D表示

**パック種別ごとの視覚差異：**
- **第1弾「交界の目覚め」**: 青・紫ベースのミニマル幾何学模様
- **プレミアムパック**: ゴールド枠＋煌めきエフェクト
- **期間限定（願相ピックアップ）**: 該当願相の色で染まった背景 + 願相シンボル

**表示内容：**
```

- 現在の連続ハズレ数を視覚化するプログレスバー
- **テキスト**: `body-md` で配置

**脈動アニメーション（残り5以下時）：**
- バーが +10% スケール → -10% スケール を 0.8s サイクルで繰り返し（pulse）
- **SE**: 軽い「ポン」音が毎サイクルで再生

**ゴール演出（残り0になった瞬間）：**
- カウンターが金色（`accent-gold`）に一瞬フラッシュ（0.2s）
- SE：キラーン音
- ハプティクス：`.medium` の振動

## 3. パック予兆演出 — 「光るパック」

プレイヤーがパック開封アクションを実行する前に、パック外観でそのパック内のレアリティを暗示する演出。期待値を高め、開封時の驚きをコントロールする。

### 3.1 予兆レベル定義

| 予兆レベル | 内容 | パック外観 | SE | ハプティクス | 画面揺れ |
|----------|------|---------|-----|----------|-------|
| **通常** | C～Rが混在 | 白い光がふんわり脈動（opacity: 0.4→0.8） | 軽い「スゥ」 | なし | なし |
| **R確定** | R確定枠目出現 | パック上部が青く輝く（#4DA3FF） | 「シャキーン」 | `.light` 1回 | なし |
| **SR確定** | SR以上確定 | パック全体が金色に脈動（#D4A843） | 「キラーン」 + エコー | `.medium` 連続（3回） | 軽度（2px） |
| **SSR確定** | SSR確定 | パック全体が虹色に発光 + 高速脈動 | 「ドゴォォン」 + ビブラート | `.heavy` × 4連打 | 中度（4px, 0.3s） |

### 3.2 予兆演出の時間設定

**予兆演出は パック選択画面から開封アクション直前までに 0.8秒間 継続する。**

- **開始タイミング**: ユーザーがパック画像をタップ → パック開封確認画面へ遷移した瞬間
- **演出フェーズ**:
  1. 0.0～0.1s: 予兆フラッシュ（フルスクリーン、不透明度 20%）
  2. 0.1～0.8s: パックの外観変化 + SE + ハプティクス
  3. 0.8s以降: 開封アクションの入力受付

## 4. パック開封アクション

### 4.1 開封トリガー

**ユーザー操作：** パック画像を上方向へスワイプ

- **スワイプ距離閾値**: パック高さの 30% 以上
- **スワイプ速度**: 最小 200 pt/s （素早く）

### 4.2 開封エフェクト

**時系列：**

| ステップ | 演出 | 時間 | 備考 |
|---------|------|------|------|
| パック割れ開始 | パック表面に亀裂パーティクル | 0.3s | FX_SUMMON シリーズを流用 |
| 光の散乱 | 光の粒子がパック中央から放射 | 0.25s | 粒子数: 50～100個 |
| カード出現 | 5枚のカードが扇形に広がる | 0.4s | 3D回転アニメーション |
| **合計** | | **≤ 1.0s** | |

**ハプティクス：** 開封時に `.light` の短い振動（「パリッ」感）

**SE：** 「バシャッ」という破裂音（効果音素材 SE_PACK_OPEN）

### 4.3 カード展開アニメーション

**5枚のカードが以下のような扇形で展開される：**

```
       ← 左端: -45°
    ↙ ↓ ↘
  ◀─ ■ ─▶  ← 中央: 0°（正面向き）
    ↖ ↑ ↗
       → 右端: +45°
```

- 各カード間の角度差: 22.5°
- **Z軸回転**: 各カード（-45°～+45° の範囲内で均等）
- **イージング**: EaseOutBack（弾むような着地感）
- **カードサイズ**: 手札表示サイズ（64×90pt）の 1.2倍（開封時は拡大表示）

## 5. カードめくり演出 — 「1枚ずつめくる」

### 5.1 基本フロー

ユーザーが左から順に扇形のカードをタップしていく。めくるたびに異なるレアリティ演出が発生。

**操作方法：** カードをタップ → 裏向き状態から表向きへ 180° 回転（0.4s）

### 5.2 レアリティ別演出

#### 5.2.1 C（コモン）

**演出内容：**
- カード回転時に白い光がふんわり広がる（最大 opacity: 0.3）
- パーティクル数: 少数（5～10個）
- SE: 軽い「ポン」音（SE_CARD_FLIP）
- **演出時間**: 0.3s （カード回転の 0.4s に含む）
- ハプティクス: なし

**ビジュアルイメージ：** 淡いマナライト、控えめな流光

#### 5.2.2 R（レア）

**演出内容：**
- カード回転時に青い光柱が立つ（#4DA3FF, 高さ: カード高さの 1.5倍）
- パーティクル数: 中程度（20～30個）
- SE: 「シャキーン」という鋭い音
- **演出時間**: 0.5s
- ハプティクス: `.light` × 1回

**ビジュアルイメージ：** 切り立つ光、知的な輝き

#### 5.2.3 SR（スーパーレア）

**演出内容：**
- カード回転時に金の光柱が爆発（#D4A843）
- パーティクル爆発: 50～80個の金色粒子が半径 200pt へ放射
- フレーム外縁がフラッシュ（黄金色、opacity: 0.5）
- SE: 「ドゥルルル…キラーン！」（音声エフェクト 0.8s）
- **演出時間**: 1.0s
- ハプティクス: `.medium` × 1回

**ビジュアルイメージ：** 豪華な金粉爆発、勝利の瞬間

**NEW!バッジ：** 未所持カードの場合、カード上部に「NEW!」バッジが脈動（pulse）表示される

#### 5.2.4 SSR（スーパースーパーレア）

**演出内容：** §6「神引き演出」へ移行（即座に画面遷移）

### 5.3 フォイルカードの追加演出

既に 3 枚所持しているカード（フォイル化可能なカード）がめくられた場合：

**追加演出：** カード上部に「FOIL」バッジが光り、カード表面に UVスクロール効果（虹色グラデーション）が 1.5s 間継続

## 6. 神引き演出 (SSR専用)

SSRカードをめくった時のみ発動。プレイヤーの高揚感を最大化するための一連の演出。

### 6.1 フェーズ分解

**フェーズ1: 画面暗転（0.3s）**
- 現在の画面が 0.3s で暗転（フェードアウト）
- **SE**: なし

**フェーズ2: 虹色オーラ爆発（0.6s）**
- 画面中央から虹色の光が爆発的に広がる（拡大スケール: 0→1.5, EaseOutBack）
- パーティクル：虹色の大粒子（50～100個）が放射
- **SE**: 「ドゴォォン」という深くて豊かな音色
- ハプティクス: `.heavy` × 4連打（0.1s 間隔）
- 画面揺れ: 強度 (ShakeX: 8px, ShakeY: 8px, 0.4s)

**フェーズ3: カード開示＆Live2D/Spine演出（2.0s）**
- SSRカードのイラストが画面中央に拡大表示
- 各SSRカード固有の アニメーション（Live2D or Spine）が再生
  - 例：願主Aldric → 剣を振り下ろすモーション（0.8s）
  - 例：願主灯凪 → 木々が芽吹き、光が満ちるモーション（1.2s）
- 背景: アニメーションに応じた幻想的な背景（願相のカラーをベース）

**フェーズ4: 専用BGM・ジングル（全体2.5s）**
- 通常BGMから専用ジングル（SOUND_DESIGN_SPEC.md §8 JINGLE_GACHA_SSR）に切替
  - 願相別に異なるジングル（各8～15秒）を用意
  - 例：曙専用ジングル → 戦いの鐘が鳴り響く
  - 例：穏専用ジングル → 木の根音と光が響く

**フェーズ5: スクリーンショットタイム（2.0s）**
- 演出終了後、カードが定位置（やや大きめのサイズ）にゆっくり着地（EaseOutCubic, 0.8s）
- **2秒間の静止画面状態** を設ける（SNS共有を促す意図）
- UI要素は非表示（スクリーンショット時の見栄えを優先）
- **SE**: 深く静寂的な音（BGM ボリューム絞り）
- ハプティクス: `.success` × 3連打（やや長めの間隔）

**フェーズ6: 結果画面への遷移（0.5s）**
- スクリーンショットタイム終了後、自動的にフェードアウト
- 結果画面へ遷移

### 6.2 カットイン（Live2D/Spine）制作基準

**対象カード：** 全 SSR 12 種

**制作内容：**
- キャラクターイラストの一部（顔・上半身）を Live2D or Spine でアニメーション化
- 動作パターン: 各キャラクターの代表的なアクション 1 パターン（0.8～1.5s）
- 背景: 願相テーマのカラー＋パーティクルで装飾

**制作優先度：**
1. 願主 6 体（灯凪・Aldric・崔鋒・Rahim・Amara・Vael）
2. 他の強力 SSR（2023 年版の設計に準じて決定）

## 7. フォイルタッチ演出

フォイルカード（既に 3 枚所持している高レアリティカード）が出た場合、長押しまたは詳細表示で以下の特別な触覚体験が実現される。

### 7.1 フォイル拡大表示

**トリガー：** カード詳細画面でカードを長押し → 全画面拡大表示

**表示モード：**
```
┌─────────────────────────────┐
│         [×] (閉じる)         │
├─────────────────────────────┤
│                             │
│      [フォイルカード]       │（拡大表示, 高さ 600pt）
│                             │
│   「本物のフォイルを          │
│    触っている」感覚           │
│                             │
└─────────────────────────────┘
```

### 7.2 ジャイロスコープ連動

**実装方式：** `Input.gyro` を使用（iOS のみ）

**動作：**
- ユーザーが端末を傾けると、カード上の ホログラム効果（光の反射点）が連動して動く
- 反射光の色: 虹色グラデーション（赤→青→緑→黄）
- 反射点の大きさ: 10～30pt、複数点表示（3～5個）
- **レスポンス**: ジャイロ入力から 0.1s 以内に反応（リアルタイムティッパック）

### 7.3 指でなぞると光が反応

**実装方式：** タッチ入力の `Touch.position` をリアルタイムで追跡

**動作：**
- ユーザーが指でカード上をなぞると、なぞった軌跡に沿って光が追従
- 軌跡光の色: 白～薄紫（opacity: 0.6）
- 軌跡の幅: 15～30pt、尾部の長さ（トレイル）: 100pt で減衰
- パーティクル: なぞった軌跡に沿って発生（10～20個/フレーム）

### 7.4 UVスクロール レインボーシフト

**実装方式：** シェーダーベースの UVスクロール アニメーション

**動作：**
- カード表面全体に虹色の光が 2.0s サイクルで横スクロール
- 色相循環: 赤(0°) → 黄 → 緑 → 青 → 紫 → 赤
- スクロール速度: 0.5 unit/s（ゆったり目）
- **透過度**: 10～40%（情報を完全に隠さないよう注意）
- **重ね合わせモード**: Overlay or Screen（乗算で光を加算）

## 8. 願相オーラ演出

パック内で同じ願相のカードが **3 枚以上** 出た場合、背景がその願相の色に染まる。これにより、同じ色でデッキを組める喜びを視覚化する。

### 8.1 願相別オーラ表現

| 願相 | 色 HEX | オーラ表現 | SE |
|------|--------|----------|-----|
| **曙** | #FF5A36 | 赤い炎が背景全体を覆う。粒子が上昇 | 「ゴゥゥ」（低めの轟音） |
| **空** | #4DA3FF | 青い水が背景に流れる。光が下降 | 「ザァァ」（風音） |
| **穏** | #59C36A | 緑の木漏れ日。光が斜めから差す | 「スウ」（穏やかな音） |
| **妖** | #9A5BFF | 紫の魔法陣がゆっくり回転 | 「シーン」（高めの音） |
| **遊** | #F4C542 | 黄色い稲妻が背景を駆ける | 「ビリビリ」（電光音） |
| **玄** | #3A3A46 / #6E6E8A | 白い霧が背景を満たす | 「ドゥン」（深い沈黙音） |

### 8.2 フェーズ

**開始条件：** 3 枚目のカード をめくり終わった瞬間（その願相が 3 枚以上であると確定した時点）

**フェーズ:**
1. **背景フェードイン** (0.5s): 現在の背景から願相カラーのオーラへ段階的に遷移
2. **SE + ハプティクス** (0.3s): 願相テーマのジングル再生、`.medium` 振動
3. **背景パーティクル** (1.5s～継続): 願相別パーティクルが背景を満たす

**終了条件：** パック開封が完全に終了した時（結果画面に遷移）

│ [この結果を共有]               │
└────────────────────────────────┘
```

### 10.2 カード表示

**ソート順：** レアリティ降順（SSR → SR → R → C）

**カード枠：**
- サイズ: コレクション一覧サイズ（80×112pt）
- 各カード間ギャップ: `space-md` (16pt)
- レアリティ別装飾: UI_STYLE_GUIDE.md §8.3 に準拠

**NEW!マーク：** 未所持カードの右上に表示（脈動アニメーション）

**フォイルバッジ：** フォイル化可能カードの場合、右下に「FOIL」バッジ

### 10.3 重複カード変換表示

**重複カード（既に 3 枚所持）の処理：**
- カードの右下に「+◯G」と変換ゴールド額を表示
- C: 30G / R: 80G / SR: 200G / SSR: 500G
- テキストカラー: `accent-gold`
- フォント: `card-power` サイズ

**変換演出（アニメーション）：**
1. カード表示時に該当カードが点滅（opacity: 0.5→1.0, 0.2s × 3回）
2. 「+◯G」テキストが下から上へスライドイン＆フェードアウト（0.8s）
3. 画面下部のゴールド表示が加算値分だけ数字カウントアップ（EaseOutQuad, 0.5s）

### 10.4 ボタンレイアウト

**ボタン群：**

## 12. 技術仕様

### 12.1 アニメーション実装

**フレームワーク：** DOTween（既存導入済み）

**タイムライン管理：**
- 各演出シーケンスを `DOTween.Sequence()` で構築
- 演出の開始→完了→次演出へ の非ブロッキング実装（async/await パターン）

**例：SR カードめくり演出**
```csharp
var flipSequence = DOTween.Sequence()
    .Append(card.transform.DORotate(new Vector3(0, 180, 0), 0.4f, RotateMode.FastBeyond360))
    .Join(lightColumn.DOFade(1f, 0.2f))
    .Append(lightColumn.DOFade(0f, 0.3f))
    .OnComplete(() => PlaySE("SE_CARD_FLIP_SR"));
```

**Particle System：** Unity の標準 ParticleSystem を使用（VFX Graph は最低端末で重いため MVP では不採用）

### 12.2 ハプティクスフィードバック

**API：** iOS UIFeedbackGenerator（Taptic Engine）

**実装例：**
```csharp
if (targetCard.rarity == Rarity.SSR)
{
    for (int i = 0; i < 4; i++)
    {
        UINotificationFeedbackGenerator.Feedback(.success);
        await Task.Delay(100);
    }
}
```

### 12.3 ジャイロスコープ連動

**実装：** `Input.gyro` （iOS のみ対応）

**検出内容：**
- Gyro.gravity (Y軸: 上下傾け)
- Gyro.rotationRate (Z軸: 左右傾け)

**ホログラム反射点の計算：**
```csharp
Vector3 gyroRot = Input.gyro.attitude.eulerAngles;
hologramReflection.position = cardCenter + new Vector3(gyroRot.z * sensitivity, -gyroRot.x * sensitivity, 0);
```

### 12.4 音声システム統合

**BGM 遷移：** SOUND_DESIGN_SPEC.md に準拠

- パック選択画面: `BGM_PACK` （110 BPM, Am）
- パック開封～結果: 同上、ただしボリュームダッピング(-10dB)
- SSR 演出時: `BGM_PACK` → JINGLE_GACHA_SSR への切替

**SE 同時再生上限：** SOUND_DESIGN_SPEC.md §4.2 に準拠（最大 8 チャンネル）

## 13. アクセシビリティ

### 13.1 全演出スキップ オプション

**設定画面：** 設定 > グラフィック > パック開封演出

**設定項目：**

| 設定項目 | ON/OFF | デフォルト | 効果 |
|---------|--------|----------|------|
| 演出スキップ | オフ→オン | OFF | 全パック演出を即座に結果表示 |
| 光感受性対応 | オフ→オン | OFF | フラッシュ演出を削除・減光 |
| ハプティクス | オフ→オン | ON | 振動フィードバック 無効化 |
| アニメーション低減 | オフ→オン | OFF | スライド→フェード置換、パーティクル削除（§13.4） |

### 13.2 色覚多様性対応

**原則：** レアリティ判定は **色 + テキスト + アイコン** の三重表示

**実装例：**
```
┌────────────────┐
│ [金] SR        │  ← 色 + テキスト + 星アイコン
│ ★ SR          │
└────────────────┘
```

**レアリティアイコン定義：**
- C: 円（白）
- R: ダイヤ形（銀）
- SR: 星 5 個（金）
- SSR: 星 + 宝玉（虹）

### 13.3 Haptic Touch 無効化

**対象：** バイブレーションの種類すべて

**実装：**
```csharp
if (accessibilitySettings.hapticEnabled == false)
{
    // ハプティクス処理をスキップ
    return;
}
```

### 13.4 光感受性対応

**基準：** ACCESSIBILITY_SPEC.md §4 に準拠

**対応内容：**

| 演出箇所 | 光感受性 ON 時 | 処理 |
|---------|----------------|------|
| 画面フラッシュ（SSR） | 禁止 | グラデーションフェードに置換 |
| パーティクル爆発 | 減数 | パーティクル数を 50% 削減 |
| 画面揺れ | 禁止 | 画面揺れをオフ |
| テキストフラッシュ | 禁止 | フェード動作のみ |

### 13.5 スクリーンリーダー対応

**対象デバイス：** iOS VoiceOver

**読み上げ内容（パック選択画面）：**
- 「通常パック。5 枚、R 以上 1 枚確定」
- 「天井カウンター。SSR 確定まで あと 50 パック」
- 「開封ボタン」

**読み上げ内容（結果画面）：**
- 「SR カード <カード名>」
- 「ゴールド 350 獲得」
- 「あと 45 パックで SR/SSR 確定」
