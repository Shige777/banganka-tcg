# TEST_PLAN.md
> 万願果 テスト計画 v2.0

---

## 0. 目的

MVPリリースに必要なテスト方針・範囲・手法を定義する。
`GAME_DESIGN.md` のルール正確性と、オンライン対戦の安定性を保証することが最優先。

## 1. テスト階層

| 階層 | 対象 | 手法 | 自動化 |
|------|------|------|--------|
| L1: ユニット | バトルロジック関数 | C# NUnit / Unity Test Framework | ◎ 必須 |
| L2: 統合 | 画面遷移・UI挙動 | Unity Test Runner (PlayMode) | ○ 推奨 |
| L3: ネットワーク | Firebase通信・同期 | Cloud Functions エミュレータ | ○ 推奨 |
| L4: E2E | 対戦フロー全体 | 手動 + TestFlight | △ 手動 |
| L5: バランス | カードバランス検証 | シミュレーション（自動対戦Bot） | ○ 推奨 |

## 4. L3: ネットワークテスト

### 4.1 Firebase エミュレータ利用
- `firebase emulators:start` でローカルにAuth/Firestore/RTDB/Functionsを起動
- 実際のFirebaseに接続せずにテスト可能

### 4.2 テストケース

| テストケース | 検証内容 |
|-------------|---------|
| コマンド送信→反映 | PlayManifest送信 → GameState更新 → 両クライアント受信 |
| 不正コマンド拒否 | 相手ターン中にコマンド → ERR_NOT_YOUR_TURN |
| CP不足拒否 | CP不足のカードプレイ → ERR_INSUFFICIENT_CP |
| ターンタイムアウト | 90秒無操作 → 自動EndTurn |
| 3回連続タイムアウト | 自動敗北判定 |
| 切断→再接続 | アプリをバックグラウンド → 復帰 → GameState復元 |
| 切断タイムアウト | 60秒以内に再接続しない → 自動敗北 |
| 両者切断 | 両方が60秒超過 → 引き分け |
| マリガン同期 | 両者のMulligan完了待ち → ゲーム開始 |
| マリガンタイムアウト | 30秒未送信 → 引き直しなしで進行 |
| ルーム有効期限 | 5分で参加なし → ルーム自動削除 |

## 6. L5: バランステスト

### 6.1 自動対戦シミュレーション
- 同一デッキ同士の対戦を各組み合わせにつき最低3,000回実行（§10.1準拠）
- 先攻勝率が45%〜55%の範囲に収まるか確認
- 平均決着ターン数を測定（目標：12〜20ターン）

### 6.2 カード使用率分析
- 各カードのピック率・勝率貢献を集計
- 採用率が極端に低い/高いカードはバランス調整対象

### 6.3 決着パターン分布
- 願力端到達 vs 24ターン判定 vs 引き分け の比率を測定

## 10. L5強化: バランステスト詳細（v2.0追加）

### 10.1 サンプルサイズ要件
- 統計的有意性のため、各デッキ組み合わせにつき **最低3,000試合** のシミュレーションを実施
- 信頼区間95%で勝率差±2%以内の精度を目標とする

### 10.2 アーキタイプ別勝率検証

| 検証項目 | 健全範囲 | 異常時アクション |
|---------|---------|---------------|
| 各アーキタイプ勝率 | 45%〜55% | BALANCE_POLICY.md §2 の調整プロセスへ |
| 先攻/後攻勝率差 | ±5%以内 | ゲージ初期値またはドロー枚数の調整検討 |
| 最強デッキ勝率 | 60%以下 | トップカードのナーフ検討 |
| 最弱デッキ勝率 | 40%以上 | ボトムカードのバフ検討 |

### 10.3 アーキタイプ相性マトリクス

6アーキタイプ全組み合わせ（15通り）の対戦を検証し、以下を確認する：

| 検証項目 | 基準 |
|---------|------|
| 相性差の最大値 | 勝率差35%以下（65:35が上限） |
| 三すくみ構造 | 完全な1方向支配がないこと |
| コントロール vs アグロ | 決着ターンが想定通り（アグロ=10前後、コントロール=18前後） |

### 10.4 カード個別バランス指標

| 指標 | 健全範囲 | 異常時 |
|------|---------|--------|
| カード採用率 | 5%〜50%（34枚中の出現率） | 0%=弱すぎ、80%以上=強すぎ |
| カードプレイ率 | ドロー時の30%以上がプレイされる | 低すぎる=コスト高すぎ |
| カード勝利貢献度 | プレイした試合の勝率が全体平均±10%以内 | 大幅乖離=バランス問題 |

### 10.5 ゲームテンポ検証

| 検証項目 | 目標値 | 異常時 |
|---------|--------|--------|
| 平均決着ターン | 12〜20ターン | 短すぎ=アグロ優勢、長すぎ=膠着問題 |
| 24ターン到達率 | 10%以下 | 高い=ゲージ変動量不足 |
| 降参率 | 15%以下 | 高い=逆転要素不足 |
| 平均試合時間 | 8〜15分 | 長すぎ=モバイルに不適切 |
