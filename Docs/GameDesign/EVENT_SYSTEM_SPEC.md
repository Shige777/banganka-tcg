# EVENT_SYSTEM_SPEC.md
> 万願果 ライブオプス・イベントシステム設計 v1.0

---

## 0. 目的

期間限定イベント・特殊ルール対戦・イベント報酬配布など、運営フェーズで必要なライブオプス基盤を定義する。

---

## 1. イベント種類

| 種類 | 概要 | 頻度 | 実装時期 |
|------|------|------|---------|
| **構築制限イベント** | 特殊ルールでの対戦（例: Rレア以下限定戦） | 月1〜2回 | Phase 2 |
| **ランキングイベント** | 期間中の勝利数/ポイントでランキング | シーズンごと | Phase 3 |
| **ドラフトイベント** | MONETIZATION_DESIGN.md §14のドラフトモードの期間限定版 | 月1回 | Phase 2 |
| **ストーリーイベント** | 期間限定ストーリー + 報酬 | 2〜3ヶ月に1回 | Phase 3 |
| **コラボイベント** | 外部IP連携（将来） | 不定期 | Phase 4+ |
| **季節イベント** | 正月・ハロウィン等の装飾+限定報酬 | 年4〜6回 | Phase 2 |

## 2. イベントデータ構造

### 2.1 Firebase Remote Config

```json
{
  "active_events": [
    {
      "eventId": "evt_2026_04_rarity_cup",
      "type": "restricted_battle",
      "title": "レアリティカップ",
      "subtitle": "R以下限定で腕を試せ！",
      "startAt": "2026-04-01T00:00:00+09:00",
      "endAt": "2026-04-07T23:59:59+09:00",
      "bannerImageUrl": "events/rarity_cup_banner.png",
      "rules": {
        "maxRarity": "R",
        "deckSize": 34,
        "specialRules": []
      },
      "rewards": [
        { "threshold": 3, "type": "gold", "amount": 500 },
        { "threshold": 7, "type": "pack_ticket", "amount": 1 },
        { "threshold": 15, "type": "cosmetic", "itemId": "sleeve_rarity_cup" }
      ]
    }
  ]
}
```

### 2.2 Firestore（プレイヤー進捗）

```
users/{uid}/events/{eventId}/
  ├── progress: int (累積勝利数 or ポイント)
  ├── claimedRewards: string[] (受取済み報酬threshold)
  ├── entries: int (参加回数)
  └── updatedAt: timestamp
```

## 3. 構築制限イベント

### 3.1 制限ルール例

| イベント名 | 制限 |
|-----------|------|
| レアリティカップ | R以下のカードのみ使用可 |
| 単願相杯 | 1つの願相のカードのみ使用可 |
| 低コスト戦 | コスト5以下のカードのみ |
| 界律なし杯 | 界律カード使用禁止 |
| ランダムデッキ杯 | サーバーがランダムデッキを配布 |

### 3.2 デッキバリデーション

- イベントマッチ開始時にCloud Functionでデッキを検証
- 制限違反 → エラーコード返却 + 「デッキがイベントルールに適合しません」表示
- デッキ構築画面でイベントルール適用フィルタを提供

## 4. ホーム画面バナー管理

### 4.1 バナーローテーション

- ホーム画面上部にバナー枠（横スクロール / 自動切替5秒）
- Remote Config の `active_events` 配列から動的表示
- バナータップ → イベント詳細画面へ遷移

### 4.2 バナー優先度

| 優先度 | 内容 |
|--------|------|
| 1（最高） | メンテナンス告知 |
| 2 | 進行中イベント（残り時間が短い順） |
| 3 | 新シリーズ/パック告知 |
| 4 | バトルパスシーズン告知 |
| 5 | 常設コンテンツ案内 |

## 5. イベント報酬配布

### 5.1 配布フロー

```
イベント画面で進捗確認
  → threshold 到達 → 「受取」ボタン活性化
  → タップ → Cloud Function: claimEventReward
    - 進捗とthresholdを検証
    - 報酬をユーザーに付与（gold/pack_ticket/cosmetic）
    - claimedRewards に追加
  → 演出（ゴールド獲得 / アイテム獲得アニメーション）
```

### 5.2 イベント終了後

- イベント終了後**48時間**は報酬受取可能（猶予期間）
- 猶予期間終了後の未受取報酬は受取不可（ただしお知らせで事前に告知）

## 6. イベントスケジュールテンプレート

### 6.1 大弾リリース月のスケジュール

> 大弾＋ミニセットの年間サイクルについては `CARD_RELEASE_ROADMAP.md` を参照。

```
Week 1: ── 大弾リリース ──
        新カード体験バトル（3日間・新カード試用貸出）
        ログインボーナス（7日間・パック券×3 + ゴールド1,000）
Week 2: 構築制限イベント（新シリーズ限定戦・7日間）
Week 3: ドラフトイベント（3日間）/ バトルパスシーズン切替
Week 4: 季節イベント or ストーリーイベント or 休息期間
```

### 6.2 ミニセットリリース月のスケジュール

```
Week 1: バランスパッチ（月次データレビュー）
Week 2: ── ミニセットリリース ──
        記念ミッション（7日間・5勝でSR確定チケット）
        ログインボーナス（3日間・パック券×1 + ゴールド500）
Week 3: 構築制限イベント（レアリティカップ等・7日間）
Week 4: ランキングイベント or ドラフトイベント
```

### 6.3 通常月のスケジュール（大弾/ミニセットなし）

```
Week 1: 構築制限イベント開始（7日間）
Week 2: イベント後半 → 終了。次弾ティーザー告知
Week 3: ドラフトイベント（3日間）/ バトルパスシーズン切替
Week 4: 季節イベント or ストーリーイベント or 休息期間
```

## 7. イベント同時開催ルール

### 7.1 同時開催上限

| 項目 | 値 |
|------|-----|
| 同時開催イベント上限 | **3件** |
| 同一種別の同時開催 | **不可**（構築制限×2などは禁止） |

### 7.2 優先表示ルール

複数イベント同時開催時のホーム画面バナー表示優先度:

| 優先度 | 条件 |
|--------|------|
| 1（最高） | 残り24時間以内に終了するイベント |
| 2 | 開始から24時間以内の新規イベント |
| 3 | プレイヤーが未参加のイベント |
| 4（最低） | それ以外（進行中イベント） |

### 7.3 報酬計算

- 複数イベントの報酬は **独立して計算** される（合算や相殺はしない）
- 対戦1回で複数イベントの進捗が同時に進むことを許可する（例: 構築制限イベントの対戦がバトルパス進捗にもカウントされる）
- デイリーミッション・ウィークリーミッションとイベントミッションは **並行して進行** する

## 8. 緊急イベント配信

Remote Config のリアルタイム更新を利用して、アプリアップデートなしにイベントを配信可能。

- 新イベント追加: Remote Config に `active_events` を追加 → アプリ側で自動検知（12時間ごとのfetch + アプリ起動時fetch）
- イベント緊急停止: `active_events` から該当イベントを削除
- fetch間隔: 開発中12時間、ローンチ後は最短1時間（Remote Config のスロットリング考慮）
