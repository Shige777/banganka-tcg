- **ジャンル**：1vs1 対戦型デジタルTCG（e-sports志向）

- **プラットフォーム**：iOS（MVP固定。v0.5.3では他プラットフォーム展開を扱わない）

## 1. 概要

### 1.1 コンセプト
各時代、各時空、各運命から現れた者たちが、たったひとつ願いを叶える `万願果` を求めて争う。  
相手の願主へ攻撃を通し、**自分の願いこそが、この瞬間もっとも強く世界を揺らしたことを証明する**。

### 1.2 観戦性・競技性の設計原則（v0.5.0）
- **有利不利が残り願力カード枚数で分かる**（各プレイヤーの願力ゾーン残枚数を常時表示）
- **毎ターンの“流れ”が見える**（願主 / 顕現の攻防、直撃時のゲージ変動）
- **読み合いが1枚で変わる**（界律の上書き）
- **攻撃先・ブロック・上書き・リーサル読みの四択が毎ターン発生する**
- **盤面要素を絞り、テンポと可読性を最優先**（不要なメーター/持続盤面を廃止）

## 2. 世界観

### 2.1 交界
この世界には、本来交わるはずのない時代と時空が重なる領域 `交界` が存在する。  
古代の王、滅びた未来の兵、名もなき放浪者、失われた時代を生きた魔術師。  
異なる歴史と運命を背負う者たちは、ただひとつの奇跡 `万願果` を求めてこの地に現れる。

### 2.2 万願果
`万願果` は、あらゆる願いをひとつだけ叶えるとされる存在。  
滅びた国を取り戻したい者。  
失った誰かを蘇らせたい者。  
赦されぬ過去を消したい者。  
変えられなかった未来を書き換えたい者。  
それぞれの願いは交わらず、やがて衝突する。

### 2.3 プレイヤー
プレイヤーは **果求者**。  
果求者は願札を放ち、顕現を率い、界律を書き換え、願主を成長させながら、自らの願いを `交界` に刻み込もうとする。

### 2.4 願いの衝突
戦う理由はそれぞれ。  
守れなかったものを取り戻すため。  
失った未来に手を伸ばすため。  
自分だけの願いを、この世界に認めさせるため。  
`交界` に集った者たちは、願いの重さそのものを武器にして激突する。

## 3. 盤面（v0.5.0）

### 3.1 上段：HPゲージ＋願力カード（各プレイヤー個別）
- 各プレイヤーが独立した **HPゲージ（初期値 100）** を持つ
- HPゲージ上の **15%ごとの閾値**（85/70/55/40/25/10）に **願力カード** を表向きで配置（計6枚）
- HPゲージと残り願力カード枚数は **両プレイヤーに公開情報** として常時表示する
- ダメージを受けてHPが閾値を下回ったとき、その位置の願力カードが **発動→墓地**（詳細は §4.1・§5.5）
- HP = 0 になると **瀕死（Final）** 状態になる
- Final 状態で次の直撃が通ると **即時敗北**（鯱鉾勝利）

```
HP 100━━[■85]━━[■70]━━[■55]━━[■40]━━[■25]━━[■10]━━0
         15%     15%     15%     15%     15%     10%
```

> 観戦時はHPゲージの残量と「次の願力カードまで何%か」が見える。閾値に近づく緊張感。

### 3.2 中段：顕現列（各プレイヤー専用・前列3 / 後列3）
- `顕現` を配置する戦闘列
- **前列3体 / 後列3体** の合計6体まで配置できる
- 各顕現は **待機 / 消耗** の状態を持つ
- 待機中の顕現は **アタック**、`ブロッカー` を持つ場合は **ブロック** に参加できる
- 消耗中の顕現も攻撃対象にできる

### 3.3 下段：願主（各プレイヤー1体・同一願主で開始）
- 各プレイヤーが **同じ願主** を持つ（MVP）
- 願主は **願成**と**レベル**を持つ
- 待機中なら **アタック** できる
- 効果で `ブロッカー` を得ていれば **ブロック** できる
- `願主` への直撃が通ると `願撃` 分だけ願力を動かす

### 3.4 下段：界律（共有1枠）
- 盤面に **共有の界律枠を1つ**持つ（両者に影響）
- 新しい界律が出たら **上書き**（古い界律は破棄）
- **永続**：Durationなし（上書きされるまで有効）

## 4. メーター & リソース

### 4.1 願力（HP＋閾値カード）
- 各プレイヤーが **HP 100** を持つ（内部値は整数管理、小数点以下は切り上げ）
- HPゲージ上の **15%ごとの閾値** に願力カードを配置（6枚：85/70/55/40/25/10）
- ダメージを受けてHPが閾値を **下回った** とき、その位置の願力カードが発動→墓地
- 1回のダメージで複数の閾値を超えた場合、**超えた全ての願力カードが順に発動**する
- HP = 0 で **Final（瀕死）** 状態。この状態で直撃が通ると即敗北（詳細は§8）
- ターンリミット時は **残りHP** を比較して勝者を決定（詳細は§8）
- HPは0未満にならない（0でクランプ）

> 旧「共有ゲージ」はv0.5.2で廃止。旧「願力カード枚数制」はv0.5.3でHP+閾値制に移行。

### 4.2 CP（Creative Point）
- カードプレイ用リソース
- ターン開始時：
  - **最大CP +1（上限10）**
  - **現在CP = 最大CP（全回復）**
- MVPの狙い：顕現展開・詠術使用・界律上書きのテンポを制御する

### 4.3 行動状態
- **待機**：アタック / ブロック可能
- **消耗**：そのターンはアタック / ブロック不可
- 自分のターン開始時、自分の願主と顕現は **待機** に戻る
- **召喚酔い**：そのターンに新しく出た顕現はアタック / ブロック不可

### 4.4 戦力
- `願主` と `顕現` はそれぞれ **戦力** を持つ
- 戦闘では **攻撃側の戦力** と **防御側の戦力** を比較する
- 攻撃側戦力 > 防御側戦力 なら、防御側は **退場** する
- 防御側戦力 > 攻撃側戦力 なら、攻撃側は **退場** する
- 攻撃側戦力 = 防御側戦力 なら、**両者退場** する

### 4.5 願撃（%ダメージ）
- `願主` と `顕現` はそれぞれ **願撃** を持つ
- 願撃は **「固定X%」** または **「現在Y%」** のいずれかで定義される
  - **固定X%**：maxHP（=100）のX%のダメージ。常に一定。終盤ほど頼れる
  - **現在Y%**：現在HPのY%のダメージ。HP高いほど痛く、低いほど弱い。単独で0にはできない
- `願主` へのアタックが **ブロックされずに通った** とき、攻撃側の願撃に応じたHPダメージを相手に与える
- ダメージにより閾値を超えた場合、該当する願力カードが発動する
- 小数点以下のダメージは **切り上げ**（最低1ダメージ保証）
- カードごとに「固定%型」か「現在%型」かが決まっている

```
例：
  固定5%  → 常に5ダメージ（HP100でも HP20でも）
  現在8%  → HP100なら8ダメージ、HP50なら4、HP10なら1（切り上げ）
```

> 固定%と現在%の使い分けはデッキ構築の戦略要素。アーキタイプによる傾向はあるが固定ではない。

### 4.6 願主願成（Evolution Gauge）
- 願主固有の成長リソース（詳細は7章）
- **願札プレイに反応して増える**
- 満タンでLvアップ

> v0.5.0では Heat/Trust 等の追加メーターは存在しない。

### 5.1.2 願相の視覚ルール（MVP）
- カード枠の形状や装飾は **共通枠** を使い、スキン差分やレアリティ差分で変化させる
- 願相は、共通枠の上に乗る **色トークン** として表現する
- 最低限の適用箇所:
  - コスト背景色
  - 願相チップ / アイコン色
  - 名前帯の差し色
  - イラスト外周の薄いオーラ
- コスト表示ルール:
  - コスト背景は対応する願相カラーを使う
  - コスト数字は **全属性共通で白文字**
  - コスト数字には **黒の縁取り** を付ける
- MVPでは、願相がひと目で分かることを優先し、全文字を読まなくても色で識別できる見え方を目標にする

### 5.2 顕現
- 盤面に残る戦闘カード（ユニット相当）
- 顕現列に配置される
- 主な役割：
  - 相手 `願主` へのアタック
  - 相手 `顕現` へのアタック
  - `ブロッカー` による防御
- プロパティ（MVP）：
  - `CPCost`
  - `BattlePower`
  - `WishDamage` / 表示名 `願撃`
  - `Keywords`（`Blocker`, `Rush`, `GuardBreak` などの許可制）
  - `AspectTag` / 表示名 `願相`（`空` / `曙` / `穏` / `妖` / `遊` / `玄`）
  - `EmotionTag` / 表示名 `情相`（`渇望` / `悔恨` / `慈愛` / `追慕` / `諦観` / `矜持` / `None`）— 願いの動機を表す第2軸タグ。界律の設置者ボーナス条件・種族シナジーに使用（§5.4b参照）
  - `EffectKey`（カテゴリ：unknownゼロ運用）
  - `OnPlayEffect`（任意）

### 5.3 詠術
- 即時発動して墓地に送られる術カード（呪文相当）
- 主な役割：
  - 相手HPに直接ダメージ（`SPELL_HP_DAMAGE_FIXED` / `SPELL_HP_DAMAGE_CURRENT`）
  - 顕現 / 願主の戦力補正
  - 願撃補正
  - 除去 / 待機状態操作 / ドロー補助
- プロパティ（MVP）：
  - `CPCost`
  - `AspectTag`
  - `EffectKey`
  - `damageType`（`fixed` or `current`）+ `percent`（ダメージ%）

### 5.4 界律（アルゴリズムカード：共有）
- **共有1枠**に設置されるカード（上書き・永続）
- 目的：読み合い・環境変化を観戦者に分かりやすく提供する
- 構造（MVP）：
  - **全員効果（GlobalModifier）**：両者の顕現 / 詠術 / 戦闘に影響
  - **設置者ボーナス（OwnerBonus）**：設置者のみ追加恩恵

> MVPでは効果適用ポイントを固定する：**詠術解決時の願力変動** または **戦闘時の戦力比較** に補正をかける。

#### 5.4.1 界律の裏返し（セット／オープン）

界律は **表向き（即時発動）** または **裏向き（セット状態）** で設置できる。

- **表出し（通常プレイ）** — CPコストを払い、界律を表向きで共有枠に設置する。即座にGlobalModifierとOwnerBonusが発動する。既存界律がある場合は上書き（旧界律は墓地）。
- **裏出し（セット）** — CPコストを払い、界律を **裏向き** で共有枠に設置する。この時点では効果は一切発動しない。相手には「裏向きの界律がセットされた」ことのみ公開される（カード名・効果は非公開）。
- **裏返し（オープン）** — 自分のメインフェイズ中に、裏向きの界律を **表向き** にできる（追加コスト不要）。この瞬間にGlobalModifierとOwnerBonusが発動する。

**ルール詳細：**

- 裏向き界律は「共有枠を占有している」扱い。裏向き界律がある状態で相手が新たな界律を表出しした場合、裏向き界律は **上書きされて墓地送り**（効果未発動のまま退場）。
- 裏向き界律がある状態で自分が新たな界律を出す場合も上書き扱い（自分の裏向きは墓地送り）。
- 1ターンに界律のプレイは1回まで（表出し・裏出し問わず）。裏返し（オープン）はプレイ回数にカウントしない。
- 裏向き界律は最大1ターンしか裏のまま維持できない。**次の自分のメインフェイズ開始時に自動的に表返し**される（忘れ防止＆長期待ち伏せ防止）。

**戦略的意図：**

- **即座に効果が欲しい** → 表出し。テンポ重視。
- **次ターンにシナジーカードと同時起動したい** → 裏出し → 次ターンにシナジー顕現を展開 → 裏返して同時発動。
- **相手の界律上書きを牽制したい** → 裏出しで「何かセットした」プレッシャーを与える。相手は中身がわからないまま上書き判断を迫られる。

> 裏返しはバトスピの伏せカードやデュエマのシールドトリガー的な読み合いを生むが、「最大1ターン裏のまま」制限により遊戯王のセットカード長期待機のような複雑さは避ける。

### 5.4b 情相（EmotionTag）のゲームメカニクス

情相は願札が持つ **第2のタグ** であり、願相（AspectTag）が「何を願うか」を表すのに対し、情相は **「どんな動機で願うか」** を表す。デュエルマスターズの種族に相当する横断的シナジー軸。

#### 5.4b.1 情相一覧（5種＋None）

| 情相 | 読み | 英語キー | 意味 | 願いの動機 |
|---|---|---|---|---|
| 渇望 | かつぼう | Thirst | 渇望 | 足りないものを求めて願う。飢餓、野心、向上心 |
| 悔恨 | かいこん | Remorse | 悔恨 | 過去の過ちを悔いて願う。後悔、責任、償い |
| 慈愛 | じあい | Grace | 慈愛 | 誰かを守るために願う。愛情、庇護、自己犠牲 |
| 追慕 | ついぼ | Longing | 追慕 | 失ったものへの想いで願う。喪失、未練、再会 |
| 諦観 | ていかん | Resign | 諦観 | 全てを受け入れた先に願う。悟り、虚無、達観 |
| 矜持 | きょうじ | Dignity | 矜持 | 消えることを拒み願う。誇り、意志、不屈 |
| None | — | None | 無相 | 情相を持たない（バニラ顕現やシステムカード用） |

#### 5.4b.2 情相の使用ルール

**① 界律の設置者ボーナス条件**

一部の界律は設置者ボーナスに **情相条件** を持つ。条件を満たさない場合、GlobalModifierは発動するがOwnerBonusは発動しない。

例：「界律：執念の戦場」
- GlobalModifier：全顕現の戦力+500
- OwnerBonus（情相条件：追慕）：場にEmotionTag=追慕の顕現が1体以上いれば、設置者の顕現はさらに戦力+1000

**② 情相シナジー効果**

一部のカード効果が情相を参照する。

- `SUMMON_ON_PLAY_EMOTION_BUFF` — 場に出たとき、同じ情相の味方顕現全体に戦力+500
- `SPELL_EMOTION_DRAW` — 場に指定情相の顕現がN体以上いればドロー追加
- `ALGO_EMOTION_POWER` — 指定情相の顕現の戦力+1000（界律EffectKey）

**③ デッキ構築への影響**

- 願相は願主との一致で願成を進める「縦軸」
- 情相はカード間シナジーの「横軸」
- 曙（Contest）の渇望顕現も妖（Verse）の渇望顕現もいるため、願相を跨いだ「渇望デッキ」が成立しうる
- 混色デッキ（複数願相）の場合、情相を統一することでシナジーを確保する戦略が生まれる

#### 5.4b.4 MVP実装範囲

- 162種の全カードに情相を割り当てる（CARD_SCHEMA.md参照）
- MVP界律24種のうち **6種以上** に情相条件付き設置者ボーナスを設定
- 情相シナジーEffectKeyは3種（§11.2参照）

### 5.4c 枠ロック（スロット封印）

**概要**：Vael（空）の固有メカニクスとして、相手の場のスロット1つを封印マーカーで使用不可にする。

**基本ルール**：
- 封印されたスロットには顕現を配置できない（実質的に場が1枠狭くなる）
- 封印マーカーは特定のカード効果またはVaelのスキルで設置
- 同時に封印できるスロットは最大1つ（MVP制限）
- 解除条件：相手が自分のメインフェイズに2CPを支払うことで解除可能
- 封印中のスロットに既に顕現がいる場合：既存の顕現はそのまま残るが、退場後は再配置不可

**戦略的意図**：
- 相手の展開数を制限し、空のコントロール戦略を支える
- 解除にCPを要求することで、相手のテンポを奪う（展開とロック解除のジレンマ）
- 2弾以降で「自分の場の上限拡張」メカニクスへ発展予定（Vaelの成長を表現）

**MVP制限**：
- 枠ロック可能カード：空の一部の顕現・詠術カード＋Vaelスキル
- 最大同時ロック数：1

### 5.5 願力トリガー（WishTrigger）
- カードが **願力ゾーンから削られるとき**、そのカードに `WishTrigger: true` が設定されていれば効果を即時発動する
- 効果発動後、そのカードは墓地に送られる
- `WishTrigger` のないカードは何も発動せずそのまま墓地へ
- 発動する効果は `WishTriggerEffectKey` で定義する（EffectKeyと同じカテゴリ体系を使用）
- MVPでは以下のWishTrigger効果を実装する：
  - `WT_DRAW` — カードを1枚引く
  - `WT_BOUNCE` — 相手の顕現1体を手札に戻す
  - `WT_POWER_PLUS` — 味方顕現1体の戦力+1000（このターン中）
  - `WT_BLOCKER` — 味方顕現1体に Blocker を付与（このターン中）

> 観戦・プレイ体験上の「トリガーが発動するか？」の緊張感を意図した設計。
> 願力ゾーンに置かれたカードは表向き公開なので、両者とも「何が削られるか」は事前に分かる。

### 5.6 墓地（Graveyard）
- 各プレイヤーは自分の **墓地ゾーン** を持つ
- 以下のカードが墓地に送られる：
  - 戦闘で退場した `顕現`
  - 効果で退場した `顕現`
  - 発動後の `詠術`
  - 上書きされた `界律`
  - 手札上限（MaxHandSize=10）を超えたためドロー時に溢れたカード
- 墓地の枚数は **両プレイヤーに公開情報** として表示する
- MVPでは墓地との相互作用（墓地からの回収・墓地参照効果）は実装しない
- 将来拡張候補：`SPELL_REVIVE`（墓地から顕現を手札に戻す）、`SUMMON_GRAVEYARD_COUNT`（墓地枚数参照）

### 5.7 奇襲（Ambush）— 限定インスタントスピード（v0.5.4）

> 設計思想：遊戯王のような常時オープンなチェーン/罠システムは**導入しない**。
> 代わりに、デュエマの「革命チェンジ」「侍流ジェネレート」「ニンジャ・ストライク」に近い
> **特定条件下でのみ手札から割り込める顕現**を限定的に実装する。
> ターン構造を複雑にせず、「読み」と「裏目」の緊張感だけを追加する。

#### 5.7.1 奇襲の定義

- `Ambush` は顕現専用のキーワード（詠術・界律には付与不可）
- Ambush持ちの顕現は、**相手ターンの特定タイミングでのみ**手札からプレイできる
- 通常のCPコスト支払いは不要（CP0でプレイ）。代わりに**奇襲コスト**を支払う
- 奇襲コスト = 手札を1枚捨てる（Ambush持ちのカード自身 + 手札1枚 = 手札2枚消費）

#### 5.7.2 発動タイミング（MVP: 2種類のみ）

| タイミング | 条件 | 割り込みウィンドウ |
|-----------|------|-----------------|
| **奇襲・防衛** | 相手の顕現が自分の願主に直撃を宣言したとき | 直撃ダメージ解決の**直前** |
| **奇襲・報復** | 自分の顕現が戦闘で退場したとき | OnDeath効果解決の**直前** |

#### 5.7.3 処理順序

**奇襲・防衛の場合：**
1. 相手が願主への直撃を宣言
2. 防御側に奇襲ウィンドウが開く（**3秒**のタイマー）
3. 防御側がAmbush持ちの顕現を手札から選択 + 手札1枚を追加コストとして捨てる
4. Ambush顕現が**前衛に即座に配置**される
5. Ambush顕現は **Blocker として直撃を受ける**（戦力比較で戦闘処理）
6. 奇襲ウィンドウをパスした場合 → 通常どおり直撃ダメージ解決

**奇襲・報復の場合：**
1. 自分の顕現が戦闘で退場
2. 退場側に奇襲ウィンドウが開く（**3秒**のタイマー）
3. Ambush持ちの顕現を手札から選択 + 手札1枚を追加コストとして捨てる
4. Ambush顕現が**退場した顕現のスロットに即座に配置**される
5. Ambush顕現のOnPlayEffectがあれば発動
6. **このターン中、Ambush顕現は攻撃できない**（次の自ターンから行動可能）

#### 5.7.4 制約

- 盤面がフル（6体）の場合、奇襲は発動できない
- 1ターンに奇襲できる回数は **1回まで**（連続奇襲の防止）
- Ambush顕現は奇襲で出した場合、**そのターン中Rushを持っていても攻撃不可**
- Ambush顕現を通常ターンに手札からプレイすることも可能（その場合は通常CPコスト）
- CP0でプレイできるのは奇襲発動時のみ

| ID | 名称 | 願相 | 戦力 | 願撃 | 通常CP | 奇襲タイプ | 効果 |
|---|---|---|---:|---|---:|---|---|
| MAN_CON_15 | 伏兵の剣鬼 | 曙 | 3000 | 固定3% | 3 | 防衛 | 奇襲で出た時、戦力+2000（このターン中） |

| MAN_WEA_15 | 盾樹の番人 | 穏 | 4000 | 固定3% | 4 | 防衛 | 奇襲で出た時、味方顕現全体の戦力+500（このターン中） |

## 7. 願主（願成仕様）

### 7.1 初期状態（MVP）
- `Level = 1`（上限3）
- `EvoGauge = 0`
- `EvoGaugeMax`：固定（Lv1→2 は **3**、Lv2→3 は **4**）
- `KeyAspect`：願主カードごとに固定（付録A6参照。6願相に各1体）
- `BattlePower`：願主カードごとに固定（基本 **5000**、崔鋒のみ **5500**）
- `WishDamage`：願主カードごとに固定（基本 **固定3%**、Amaraのみ **現在5%**、崔鋒のみ **固定2%**）
- 初期キーワード：なし

### 7.2 願成獲得（固定）
- **自分が願札をプレイしたとき**、
  - `願札.AspectTag == KeyAspect` なら **EvoGauge +1**

> 条件が“1行で説明でき”、カード側にアイコン等で可視化しやすいことを優先する。

### 7.3 レベルアップ
- `EvoGauge >= EvoGaugeMax` で **Level +1**（上限3）
- レベルアップ時：
  - `EvoGauge = 0` にリセット
  - `BattlePower` を +1000
  - `WishDamage` を +1 してよい
  - 必要に応じて許可済みキーワードを付与できる（例：Lv3で `Blocker`）
  - **願主スキルが解放される**（§7.4 参照）

### 7.4 願主スキル（v0.5.4）

> 設計思想：願主を「ただのステータス台座」から「プレイヤーの分身」に昇格させる。
> バトスピの「バースト」のような伏せカード発動ではなく、
> WIXOSSのコイン技のように**プレイヤーが能動的にタイミングを選んで起動する**方式を採用。
> 「いつ使うか」の判断がゲームの勝敗を分ける。

#### 7.4.1 スキルの基本ルール

- 各願主は **2つのスキル** を持つ：`Lv2スキル`（小技）と `Lv3スキル`（必殺技）
- Lv2スキルは願主がLv2に到達した時点で解放。Lv3スキルはLv3到達で解放
- 各スキルは **1ゲームに1回のみ** 使用可能（使用済みになったら復活しない）
- 起動タイミング：**自分のメインフェイズ中**、カードプレイや攻撃宣言とは独立したアクション
- スキル使用に CP コストは不要（レベル到達が唯一のコスト）
- 1ターンにスキルは **1つまで**（Lv2とLv3を同一ターンに両方使うことは不可）
- スキルの効果解決後、通常どおりカードプレイ・攻撃宣言を行える

#### 7.4.2 スキルの UI 表現

- 願主パネルにスキルアイコンが2つ表示（未解放=暗転、解放=点灯、使用済み=灰色）
- スキル発動時：願主のカットインアニメーション + 固有SE
- 相手にはスキルが「解放済み/使用済み」かどうかが見える（公開情報）
  → 「相手のLv3スキルがまだ残っている」という心理的プレッシャーが生まれる

## 8. 勝利条件・決着（v0.5.3）

### 8.1 勝利条件の概要（2層構造）
万願果の勝利条件はスプラトゥーンの勝敗設計に準拠した2層構造を採用する。

| 種別 | 名称 | 発生タイミング | 条件 |
|------|------|------------|------|
| ① | **鯱鉾勝利（即時勝利）** | ゲーム中いつでも | 相手がFinal（HP=0）のときに直撃が通った瞬間 |
| ② | **塗り勝利（ターンリミット）** | 24ターン終了時 | 残りHPが低い方が敗北 |

### 8.2 鯱鉾勝利（即時勝利）
1. 相手のHPが **0（Final状態）** になる
2. その後、さらに **願主への直撃が通る** と即座に勝利確定
3. Final状態の相手は次の直撃を防ぐために全力でブロックしなければならない

```
例：相手HP 12 → 固定5%の直撃（5ダメージ）→ HP 7（10%閾値の願力カード発動）
    → さらに固定5%の直撃 → HP 2
    → さらに固定5%の直撃 → HP 0 = Final状態
    → 次の直撃が通ると → 鯱鉾達成・即時勝利
```

> Final状態でも顕現を守りに使えるため、逆転の余地が残る。

### 8.3 GuardBreak と願力トリガー
- **GuardBreak** を持つ攻撃がブロックされた場合でも、HPダメージは発生しない（ブロックは成立）。GuardBreak はブロック不可化ではなく、戦闘時の追加効果として定義する（MVPでは許可制）
- HPが閾値を下回った際、その位置の願力カードに `WishTrigger: true` があれば効果を即時発動する（§5.5 参照）

### 8.4 塗り勝利（ターンリミット）
- 合計ターン数が **24ターン** に到達した時点で試合を強制終了（延長なし）
- 24ターン終了時の判定順（固定）：
  1. まず8.2の鯱鉾勝利判定を実行（最終ターンに即時勝利が成立していれば適用）
  2. 未確定の場合：**残りHP** を比較する
     - Player1のHPが高い → **Player1勝利**
     - Player2のHPが高い → **Player2勝利**
     - 同値 → **引き分け確定**

```
例：24ターン終了時
  Player1 HP = 35
  Player2 HP = 22
  → Player1勝利（HPがより多く残っている）
```

> 「どれだけ削ったか（= 相手の残りHPが低いか）」が勝敗を決めるため、攻撃を諦めない動機が生まれる。

### 8.5 ゲーム終了
- 終了したら `isGameOver = true`
- 以降のターン進行／カードプレイを受け付けない（入力はリスタート等を除き無効）

## 9. ゲーム開始フロー（Setup） & マリガン（v0.5.0）

### 9.1 用語
- **Player1**：先攻（ランダムで決定）
- **Player2**：後攻
- 各プレイヤーは独立した願力ゾーン（`LifeZone`）を持つ。残枚数が少ない側が不利。

### 9.2 Setup手順
1. **デッキ選択**（MVPはプリセット固定）
2. **先攻/後攻決定**（ランダム）
3. **盤面初期化**
   - 顕現列は空（各プレイヤー 前列3 / 後列3）
   - 界律は空

4. **デッキ/手札**

   - 先攻プレイヤー: 初期手札 **5枚**
   - 後攻プレイヤー: 初期手札 **6枚**（後攻補正）
5. **HP＋願力カード初期化**
   - 各プレイヤーの `HP = MaxHP（100）` を設定
   - マリガン前に、各プレイヤーのデッキ上から **6枚** を表向きで願力ゾーンの各閾値（85/70/55/40/25/10）に配置
6. **マリガン**（9.3）
7. **Player1ターン開始**

> CPはターン開始処理で更新する（初期最大CP=0 → 先攻1ターン目開始時に最大CP=1）。

> CPはターン開始処理で更新する（初期最大CP=0 → 先攻1ターン目開始時に最大CP=1、現CP=1 が典型）。

### 9.3 マリガン（固定：任意枚数引き直し）
- 各プレイヤー **1回のみ**
- 手順（両者同時）：
  1) 引き直したいカードを任意枚数選択
  2) 選択カードをデッキへ戻す
  3) デッキをシャッフル
  4) 戻した枚数と同じ枚数をドロー
- マリガン後の追加補填（+1枚など）は行わない

## 10. ターン進行（v0.5.0）

### 10.1 ターン構造（MVP）
1) **ターン開始**
   - 自分の願主 / 顕現をすべて待機に戻す
   - 最大CP +1（上限10）
   - 現在CP = 最大CP（全回復）
   - **ターンタイマー開始（60秒）**
2) **ドロー**
   - 1枚ドロー（先攻1ターン目も実施）
3) **メイン**
   - 顕現 / 詠術 / 界律 をプレイ
4) **戦闘**
   - 待機中の願主 / 顕現でアタック
   - 相手は `ブロッカー` を持つ待機中の願主 / 顕現でブロック可能
   - **ブロック選択タイムアウト：5秒**（5秒以内にブロック宣言がなければブロックなし。SCREEN_SPEC.md §7b.3 準拠）
5) **ターン終了**
   - 勝利判定（8.1）
   - 勝者未確定の場合に限り、合計24ターン判定（8.3）
   - 次プレイヤーへ

### 10.1a ターンタイマー（v0.5.0追加、v0.5.5短縮）
- 各ターンの制限時間は **60秒**
- 制限時間内にEndTurnを宣言しない場合、サーバーが自動でEndTurnを実行する
- **連続タイムアウト**: 3回連続タイムアウトで自動敗北（`MATCH_END(result=opponent_win, reason=timeout)`）
- 詳細は `NETWORK_SPEC.md §5` を参照

### 10.1b 先攻/後攻バランス（v0.5.0 MVP方針）
- **先攻1ターン目もドロー実施**（v0.5.0固定）
- **先攻補正は導入しない**（MVP固定）
- 理由：CPが1から漸増する仕組み自体が先攻不利の緩和に寄与し、先攻はテンポ優位、後攻はレスポンス優位の非対称性が自然に生まれる
- バランステスト結果を踏まえ、MVP後に以下の選択肢を検討する：
  - 後攻の初期CPを+1する（後攻1ターン目だけ最大CP=2）
  - 後攻の初期手札を+1する（6枚スタート）
  - 何も変更しない

### 10.2 顕現プレイ時の処理順（固定）
顕現カードをプレイした場合、以下の順で処理する。

1) CP支払い、カード選択確定
2) 顕現列に空きがあるか確認
3) 顕現を **消耗状態かつ召喚酔い状態** で配置
4) OnPlayEffect があれば解決
5) **願主の願成判定**（願相一致なら+1、満タンでLvアップ）
6) ログ出力（`PLAY_SHOW`, `LEADER_EVO_GAIN` 等）

> 前衛・後衛の合計がFieldFrontSize + FieldBackSize（= 6）に達している場合、新たな顕現をプレイすることはできない（カード選択自体がUIでブロックされる）。

### 10.3 詠術プレイ時の処理順（固定）
1) CP支払い、カード選択確定
2) 詠術の効果を解決
3) 界律の補正を適用（全員効果 → 設置者ボーナスの順）
4) 願力変動があれば反映（-10..+10でクランプ）
5) **願主の願成判定**（願相一致なら+1、満タンでLvアップ）
6) ログ出力（`PLAY_SHOW`, `HP_DAMAGE`（該当する場合）, `LEADER_EVO_GAIN` 等）

> OnPlay/OnDeath/WishTriggerによるドロー効果で手札がMaxHandSize（10）を超える場合、超過分のカードは引いた直後に墓地へ送られる（手札には加わらない）。複数枚ドローの場合、1枚ずつ引いて上限チェックを行う。

### 10.4 界律プレイ時の処理順（固定）
1) CP支払い、カード選択確定  
2) **界律枠に設置**（上書き、設置者を記録）  
3) UI更新（カード名＋設置者）  
4) ログ出力（`SET_SHARED_ALGO`）

### 10.5 戦闘処理（固定）
1. 攻撃側は待機中の `願主` または `顕現` を選ぶ
2. 攻撃対象を選ぶ
   - 相手 `願主`
   - 相手の `顕現`（待機 / 消耗を問わない）
3. 攻撃側は **消耗** になる
4. 相手は、`ブロッカー` を持つ待機中の `願主` または `顕現` を1体選び、攻撃先をそのカードへ変更してよい
   > 防御側にBlocker持ちの顕現が複数いる場合、防御側プレイヤーがどのBlockerで受けるかを選択する。選択制限時間はTurnTimeLimit内で処理する（別途タイマーは設けない）。
5. 戦力比較を行う
   - 攻撃側戦力 > 防御側戦力 なら、防御側は退場
   - 防御側戦力 > 攻撃側戦力 なら、攻撃側は退場
   - 攻撃側戦力 = 防御側戦力 なら、両者退場
6. 攻撃対象が **相手 `願主`** でブロックされずに通った場合（直撃）：
   a. 攻撃側の願撃に基づきダメージを計算する
      - `固定X%` の場合：ダメージ = `MaxHP × X / 100`（切り上げ、最低1）
      - `現在Y%` の場合：ダメージ = `防御側の現在HP × Y / 100`（切り上げ、最低1）
   b. 防御側の `HP -= ダメージ`（0でクランプ）
   c. HPが下回った閾値（85/70/55/40/25/10）の願力カードを **上から順に** 発動処理する
      - `WishTrigger: true` なら効果発動 → 墓地
      - なければそのまま墓地
   d. HP = 0 になったとき：**Final状態フラグ** を立てる
   e. 防御側が既に Final 状態だった場合：**鯱鉾勝利** = `MATCH_END(result=attacker_win, reason=final_blow)`
7. ログ出力（`HP_DAMAGE`, `WISH_TRIGGER`, `FINAL_STATE`, `MATCH_END` など必要なイベント）

> OnDeath効果は退場した顕現ごとに即座に解決する。複数体が同時に退場した場合、ターンプレイヤー側 → 非ターンプレイヤー側の順に、各プレイヤー内では左スロットから順に解決する。OnDeath効果が新たな退場を引き起こした場合、その退場のOnDeath効果も同様のルールで即座に解決する（再帰チェイン許可）。

### 11.1c 願主スキル EffectKey（v0.5.4）

願主スキルは EffectKey システムで管理する。各スキルは願主固有だが、カテゴリ処理で統一する。

#### Lv2スキル（小技）
- `LEADER_SKILL_RUSH_ALL` — 自軍顕現全体にRush付与（ターン終了まで）（Aldric専用）
- `LEADER_SKILL_ATTACK_SEAL` — 相手顕現1体を攻撃封じ（ターン終了まで）（params: `target=enemy, count=1`）（Vael専用）
- `LEADER_SKILL_DRAW` — カードをN枚引く（params: `drawCount=2`）（灯凪専用）
- `LEADER_SKILL_DISMANTLE_HALVE_DRAW` — 自軍顕現1体を解体＋相手顕現1体の戦力を永続半減＋1ドロー（params: `dismantleCount=1, target=enemy, count=1, powerMultiplier=0.5, drawCount=1`）（Amara専用）
- `LEADER_SKILL_GRAVE_TO_HAND` — 墓地の顕現1体を手札に戻す（params: `source=graveyard, destination=hand, count=1`）（Rahim専用）
- `LEADER_SKILL_DAMAGE_HALVE` — 自軍全体が受けるダメージを半減（1ターン）（params: `damageMultiplier=0.5, duration=1`）（崔鋒専用）

#### Lv3スキル（必殺技）
- `LEADER_SKILL_POWER_BUFF_GUARDBREAK` — 自軍顕現全体に戦力+固定3%相当＋GuardBreak付与（ターン終了まで）（params: `powerDelta=+3000, grantKeyword=GuardBreak`）（Aldric専用）
- `LEADER_SKILL_HAND_DISCARD` — 相手の手札からランダムに2枚を墓地送り（params: `target=enemy, count=2, random=true`）（Vael専用）
- `LEADER_SKILL_MASS_BUFF_HEAL` — 自軍顕現全体の戦力+2000（永続）＋自HP10回復（params: `powerDelta=+2000, permanent=true, healHP=10`）（灯凪専用）
- `LEADER_SKILL_DISMANTLE_ALL_RUIN_DAMAGE` — 自軍顕現を全て解体し、墓地枚数×固定2%ダメージを相手HPに与える（params: `dismantleTarget=all, damageFormula=ruinCount*2, damageType=fixed`）（Amara専用）
- `LEADER_SKILL_GRAVE_TO_FIELD` — 墓地の顕現1体を場に直接召喚（params: `source=graveyard, destination=field, count=1`）（Rahim専用）
- `LEADER_SKILL_SACRIFICE_DAMAGE` — 自軍顕現1体を退場させ、その戦力の10%を固定%ダメージとして相手HPに与える（params: `sacrificeCount=1, damageFormula=power*0.10`）（崔鋒専用）

> 願主スキルの EffectKey は `LEADER_SKILL_` プレフィックスで統一。
> 各スキルは1ゲーム1回制限のため、使用済みフラグ `leaderSkillUsed[skillLevel]` をゲームステートに持つ。

### 11.2 界律 EffectKey（MVPカテゴリ）
界律は少数カテゴリで始め、戦闘か詠術のどちらかに明確に作用させる。

#### 基本（v0.5.0 初期）
- `ALGO_SPELL_PLUS`：全員の詠術による願力変動 +5、設置者はさらに +5
- `ALGO_CONTEST_POWER`：Aspect=Contest の顕現 / 願主の戦力 +1000、設置者はさらに +1000
- `ALGO_DIRECT_HIT_PLUS`：願主へ通した攻撃の願撃 +1、設置者はさらに +1
- `ALGO_LOWCP_RUSH`：CPCost<=2 の顕現に `Rush` を付与、設置者の該当顕現は願撃 +1
- `ALGO_BLOCKER_PLUS`：`Blocker` を持つ顕現 / 願主の戦力 +1000、設置者はさらに +1000

#### 拡張（v0.5.0 カードプール拡張用）
- `ALGO_DRAW_PLUS`：ターン開始時のドロー +1（設置者のみ）
- `ALGO_ASPECT_POWER`：指定願相の顕現の戦力 +1000（params: `aspect=AspectTag`）、設置者はさらに +500
- `ALGO_SPELL_COST_DOWN`：両者の詠術CPコスト -1（最低1）（設置者のみ -2）
- `ALGO_SUMMON_POWER_DOWN`：両者の顕現が召喚時に戦力 -500（設置者は影響なし）

#### 情相シナジー（v0.5.4）
- `ALGO_EMOTION_POWER`：指定情相の顕現の戦力 +1000（params: `emotionTag=EmotionTag`）、設置者はさらに +500。**設置者ボーナス条件：場に指定情相の顕現が1体以上**
- `ALGO_EMOTION_GUARD`：指定情相の顕現に Blocker 付与（params: `emotionTag=EmotionTag`）、設置者の該当顕現はさらに戦力+500。**設置者ボーナス条件：場に指定情相の顕現が2体以上**
- `ALGO_EMOTION_DRAW`：ターン開始時、場に指定情相の顕現がN体以上いれば追加ドロー+1（params: `emotionTag=EmotionTag, threshold=2`）。設置者のみ

> 情相条件付き界律の設置者ボーナスは、条件未達の場合 GlobalModifier のみ発動し OwnerBonus は不発。
> 界律のセット/オープン仕様は §5.4.1 を参照。裏向き界律は effectKey を持つが、表返しまで効果は発動しない。

#### 枠ロック（Vael専用、v0.5.4）
- `ALGO_SLOT_LOCK`：相手の場のスロット1つに封印マーカーを設置（params: `lockCount=1`）。§5.4cを参照
- `SLOT_LOCK` — 相手の場のスロット1つに封印マーカーを設置（params: `lockCount=1`）

### 11.1d 情相参照 顕現/詠術 EffectKey（v0.5.4）

- `SUMMON_ON_PLAY_EMOTION_BUFF` — 場に出たとき、自身と同じ情相の味方顕現全体に戦力+500（params: `emotionMatch=self, powerDelta=+500`）
- `SUMMON_ON_PLAY_EMOTION_DRAW` — 場に出たとき、場に同じ情相の味方顕現がN体以上いれば1ドロー（params: `emotionMatch=self, threshold=2, drawCount=1`）
- `SPELL_EMOTION_DESTROY` — 指定情相の相手顕現1体を退場（params: `emotionTag=EmotionTag, target=enemy, count=1`）

> 情相参照EffectKeyは情相の「横軸シナジー」を体現する。願相を跨いだデッキでも情相を揃えればカード間連携が成立する。

### 11.1e 枠ロック（スロット封印）顕現/詠術 EffectKey（v0.5.4）

- `SUMMON_ON_PLAY_SLOT_LOCK` — 場出し時に相手のスロット1つを封印（params: `lockCount=1`）
- `SPELL_SLOT_LOCK` — 対象：相手のスロット1つ。封印マーカーを設置（params: `lockCount=1`）

> 枠ロックEffectKeyはVael（空）アーキタイプの展開制限戦略を具現化。詳細は §5.4c を参照。

### 11.1f 自壊設計（Dismantle Architect）顕現/詠術 EffectKey（v0.5.4）

#### 解体（Dismantle）誘発
- `SUMMON_DISMANTLE_TRIGGER_DRAW` — 自軍の他の顕現が解体されたとき、カードを1枚引く（params: `triggerSource=allyDismantle, drawCount=1`）
- `SUMMON_DISMANTLE_TRIGGER_BUFF` — 自軍の他の顕現が解体されたとき、自身の戦力+1000（params: `triggerSource=allyDismantle, selfPowerDelta=+1000`）
- `SPELL_DISMANTLE` — 自軍顕現1体を解体する（params: `dismantleCount=1, target=ally`）。解体された顕現の残響を誘発する
- `SPELL_DISMANTLE_DAMAGE` — 自軍顕現1体を解体し、相手HPに固定N%ダメージ（params: `dismantleCount=1, target=ally, damageType=fixed, percent=N`）

#### 残響（Echo）— 解体時の死亡時効果
- `SUMMON_ECHO_DRAW` — この顕現が解体（または退場）したとき、カードを1枚引く（params: `echoType=draw, drawCount=1`）
- `SUMMON_ECHO_HP_DAMAGE` — この顕現が解体（または退場）したとき、相手HPにN%ダメージ（params: `echoType=damage, damageType=fixed, percent=N`）
- `SUMMON_ECHO_SUMMON` — この顕現が解体（または退場）したとき、手札からCP2以下の顕現を1体場に出す（params: `echoType=summon, costLimit=2, count=1`）
- `SUMMON_ECHO_DEBUFF` — この顕現が解体（または退場）したとき、相手顕現1体の戦力-1000（params: `echoType=debuff, target=enemy, powerDelta=-1000`）

#### 墓地参照（Ruin Count）— 墓地枚数スケール
- `SUMMON_RUIN_POWER` — 場にいる間、墓地の枚数×N の戦力ボーナスを得る（params: `ruinScale=power, bonusPerCard=+200`）
- `SPELL_RUIN_DAMAGE` — 墓地の枚数×固定N%ダメージを相手HPに与える（params: `ruinScale=damage, percentPerCard=1`）
- `SPELL_RUIN_DRAW` — 墓地の枚数が閾値以上なら追加ドロー（params: `ruinThreshold=N, drawCount=1`）

> 解体は自軍顕現を墓地に送る行為。通常の退場と同様にON_DEATH系効果も発動する。
> 残響（Echo）は ON_DEATH の妖特化版。命名で差別化し、妖デッキのアイデンティティとする。
> 墓地参照（Ruin Count）はデュエマ墓地ソースからの着想。墓地が肥えるほど妖は強くなるため、序盤の弱さを終盤の爆発力で補う設計。

### 11.3 unknown時の挙動（固定）
- 未定義の `EffectKey` が入力された場合は **ハード失敗**（例外/処理停止）とする。
- unknownをno-opで継続しない。

## A0. アーキタイプ設計方針

> 各アーキタイプは「傾向」であり、デッキ構築の制約ではない。
> 固定%/現在%の比率、勝利パターンはあくまで設計指針。
> プレイヤーは願相を跨いだ混色デッキも構築可能。

### A0.1 総覧テーブル

| 願相 | AspectTag | 通称 | 速度 | 主な勝利パターン | ダメージ傾向 |
|------|-----------|------|------|-----------------|-------------|
| 曙 | Contest | **波及連ドラ型** | 速攻 | 塗り勝利（序盤で大差をつける） | 固定%主体 |
| 空 | Whisper | **征服七武海型** | 中低速 | 塗り勝利（盤面制圧で安全に削る） | 固定%+現在%混合 |
| 穏 | Weave | **共栄全体バフ型** | 中速 | 塗り勝利（厚い盤面で安定削り） | 固定%主体 |
| 妖 | Verse | **呪縛封印型** | 変則 | 鯱鉾勝利狙い（連鎖で一気にHP0） | 現在%+固定%コンボ |
| 遊 | Manifest | **再演トリックスター型** | 中速 | 塗り勝利（リソース差で圧殺） | 固定%主体 |
| 玄 | Hush | **殉陣ブライゼシュート型** | 低速 | 鯱鉾勝利狙い（退場時誘爆で逆転） | 固定%（退場時バースト） |

### A0.2 曙（Contest）— 波及連ドラ型

**コンセプト**：低コスト顕現を連続展開し、序盤から固定%ダメージを波状攻撃で叩き込む。
TCG参考：デュエマの速攻（連ドラ系）、遊戯王のビートダウン。

- **キーメカニクス**
  - Rush持ち多数 → 出したターンに即攻撃、固定%で確実にHPを削る
  - 展開補助詠術 → 手札を減らさず横並びを維持
  - GuardBreak → 終盤のBlocker突破用フィニッシャー
- **ダメージ設計**
  - 願撃は大半が**固定%**（3〜5%）。序盤でも終盤でも安定した削り量
  - 現在%はほぼ不採用（序盤にHP100から削ると固定%より効率が悪い場面が多い）
- **勝ちパターン**
  - 理想：ターン1〜8で相手HPを50以下に → 塗り勝利で逃げ切り
  - 鯱鉾は狙わない（フィニッシャー1枚のみ、保険程度）
- **弱点**
  - 手札切れ。中盤以降にリソースが枯渇すると失速
  - Blockerの壁に弱い（GuardBreak以外では突破困難）
- **閾値カードとの相性**
  - 相手の閾値カードを早期に踏ませる → WT_DRAW等で相手にリソースを与えるリスクあり
  - 「踏ませてもリソース差で押し切る」速度が求められる

### A0.3 空（Whisper）— 征服七武海型

**コンセプト**：妨害・除去・バウンスで相手の盤面を支配し、安全になってから中〜大型顕現で削る。
TCG参考：ワンピースカードの七武海型、MTGのコントロール。

- **キーメカニクス**
  - SPELL_BOUNCE / SPELL_DESTROY → 相手の展開を繰り返し崩す
  - SPELL_REST → 攻撃を封じて時間を稼ぐ
  - SLOT_LOCK → 相手の場スロットを1つ封印し展開を制限
  - 高戦力のフィニッシャー（CP5〜6）→ 盤面が安全な状態から一方的に殴る
- **ダメージ設計**
  - 願撃は**固定%と現在%の混合**
  - 序盤の軽量顕現：現在%（小さめ）で牽制。相手HP100なら十分なダメージ
  - 終盤のフィニッシャー：固定%（大きめ）で確実に削る
- **勝ちパターン**
  - 序盤は防御・妨害に徹し、ターン10以降に盤面優位から塗り勝利
  - 稀に相手の自爆的展開を許さず、HP差で圧倒 → 鯱鉾も視野に入る
- **弱点**
  - 序盤の打点が低い。速攻型（曙）に序盤で大差をつけられると巻き返せない
  - 除去が間に合わないと盤面崩壊
- **閾値カードとの相性**
  - 相手の閾値カードを踏ませるタイミングを選べる（攻撃を遅らせる選択肢がある）
  - 自分の閾値カードは防御的なWT_BOUNCE・WT_BLOCKERが有効

### A0.4 穏（Weave）— 共栄全体バフ型

**コンセプト**：複数の顕現を並べて全体バフで底上げし、分厚い盤面から安定して削り続ける。
TCG参考：デュエマのメタリカ系、MTGのトークン＋アンセム。

- **キーメカニクス**
  - SUMMON_ON_PLAY_BUFF_ALLY → 登場時に味方全体の戦力を底上げ
  - 界律との相性◎ → ALGO_ASPECT_POWER で穏顕現をさらに強化
  - 横並び展開 → 1体除去されても盤面が崩壊しにくい
- **ダメージ設計**
  - 願撃は**固定%主体**（3〜5%）
  - 横並びの頭数 × 固定%で「薄く広く」削る
  - 現在%はほぼ不採用（複数体で殴るなら固定%の方が総ダメージが安定）
- **勝ちパターン**
  - ターン5〜15で盤面を3〜4体維持し、毎ターン合計10〜15%のHP削り → 塗り勝利
  - 盤面が完成すれば相手は1体ずつ除去しても追いつかない
- **弱点**
  - 全体除去（SPELL_DESTROY_ALL）に壊滅的に弱い
  - バフが乗る前に各個撃破されると機能しない
- **閾値カードとの相性**
  - 横並び攻撃で閾値を少しずつ踏ませる → 相手のWishTrigger発動タイミングを分散させられる
  - 自分の閾値カードにWT_POWER_PLUSを仕込むと、削られた時にさらにバフが乗る好循環

### A0.5 妖（Verse）— 自壊設計型（Dismantle Architect）

**コンセプト**：自軍の顕現を意図的に「解体」し、その死亡時効果（残響）と墓地枚数参照（墓地参照）でアドバンテージとダメージを蓄積する。構築→解体→残骸で爆発。
TCG参考：MTG Aristocrats + Shadowverse Shadowcraft（Last Words）+ デュエマ墓地ソース。

- **キーメカニクス**
  - **解体（Dismantle）**：自分の顕現を能動的に破壊して効果を発動するアクション。詠術やスキルで自軍顕現を退場させることで誘発する。解体された顕現は通常の退場と同様に墓地に送られる
  - **残響（Echo）**：解体された顕現が持つ死亡時誘発効果。通常の退場でも発動するが、解体を前提とした強力な残響持ち顕現が妖デッキの核。Shadowverse Last Wordsに相当
  - **墓地参照（Ruin Count）**：墓地のカード枚数を参照して効果が強化されるメカニクス。デュエマ墓地ソースに相当。墓地が溜まるほど詠術・顕現・願主スキルが強くなる
  - DISMANTLE_TRIGGER → 解体実行時に誘発する効果群
  - ECHO_ON_DISMANTLE → 解体された顕現の残響効果
  - RUIN_COUNT_SCALE → 墓地枚数でスケールする効果
- **ダメージ設計**
  - 願撃は**現在%主体**（序盤〜中盤の削り）+ **墓地参照固定%**（終盤の爆発）
  - 戦略：序盤は顕現を展開→中盤で解体して残響+墓地蓄積→終盤に墓地参照で大ダメージ
  - 例：墓地10枚の状態で世界消去宣言（墓地枚数×固定2%）= 20ダメージ。場の顕現3体を追加解体→墓地13枚×2%= 26ダメージ
- **勝ちパターン**
  - **墓地蓄積→終盤爆発型**
  - 中盤まで顕現の展開と解体を繰り返し、残響でリソース回収しつつ墓地を肥やす
  - 墓地が十分溜まったら（10枚以上推奨）、世界消去宣言で一気にHPを削り鯱鉾勝利を狙う
  - 墓地参照の詠術・顕現効果で塗り勝利も十分狙える
- **弱点**
  - 墓地が溜まるまで爆発力が低い（序盤が弱い）
  - 解体前提のため盤面が薄くなりやすい → 直接攻撃を通されやすい
  - 墓地除外や墓地リセットに極端に弱い（v1では墓地妨害カードは少数に留める）
- **閾値カードとの相性**
  - 自分の閾値カードにWT_DRAW → 解体で減った手札を補充する保険
  - 残響持ち顕現を閾値カードに仕込む → 相手に削られたときも残響が発動してリソース回収

### A0.6 遊（Manifest）— 再演トリックスター型

**コンセプト**：効果付き顕現を使い回し、場・手札・墓地のリソースを循環させて総合的に有利を築く。
TCG参考：デュエマのループ系、MTGの瞬速テンポ。

- **キーメカニクス**
  - SUMMON_ON_PLAY_DRAW / SUMMON_ON_PLAY_BOUNCE → 登場時に手札補充＋妨害
  - バウンス→再展開 → 自分の顕現を手札に戻して再び場に出し、OnPlay効果を再利用
  - リソース循環 → 手札が枯れにくく、常に選択肢が豊富
- **ダメージ設計**
  - 願撃は**固定%主体**（3〜5%）
  - 1体の大火力より、多数の小ヒットを繰り返す
  - 「再演」で同じ顕現の願撃を2回以上使う → 見た目以上の総ダメージ
- **勝ちパターン**
  - リソース差で圧殺 → 相手の手札・盤面が枯れた状態で塗り勝利
  - 常に手札があるため、あらゆる状況に対応できる柔軟性
- **弱点**
  - 1体あたりの戦力・願撃が控えめ → 爆発力に欠ける
  - バウンス→再展開のテンポロス → 空（除去型）に展開を潰され続けると失速
- **閾値カードとの相性**
  - 自分の閾値カードにWT_BOUNCE → 顕現を手札に戻してOnPlay再利用のシナジー
  - 相手の閾値を少しずつ踏ませる → リスク分散

### A0.7 玄（Hush）— 殉陣ブライゼシュート型

**コンセプト**：高耐久Blocker壁で守りながら、退場時効果（殉陣）で逆に相手HPを大量に削る逆転型。
TCG参考：デュエマのブライゼシュート、MTGの死亡誘発コントロール。

- **キーメカニクス**
  - Blocker多数 → 相手の直撃を壁で受け止め、HP減少を最小限に
  - SUMMON_ON_DEATH_HP_DAMAGE → 退場時に固定%ダメージ。壁が倒されるほど相手HPが減る
  - SUMMON_ON_DEATH_DRAW → 退場時にドロー。リソースが途切れない
  - 自爆系効果（将来拡張）→ 自分の顕現を能動的に退場させて殉陣を誘発
- **ダメージ設計**
  - 願撃は**固定%**が中心（退場時バーストに全力を注ぐ設計）
  - 通常の願撃は控えめ（固定3%）→ 退場時の固定5〜8%がメインダメージ源
  - 「守って殉陣で削る」→ 攻撃的に見えて実は受動的なダメージ生産
- **勝ちパターン**
  - **鯱鉾勝利のもう一つの候補**
  - 相手が壁を突破しようとして殉陣が連鎖 → 一気にHP0付近へ → 鯱鉾
  - 壁が突破されなければ自分のHPは高いまま → 塗り勝利も堅い
  - 「相手にとって殴っても殴らなくても不利」なジレンマを作る
- **弱点**
  - バウンス（手札に戻す）に極端に弱い → 退場時効果が発動しない
  - 自発的な攻め手が少ない → 相手が攻めてこないと殉陣が機能しない
  - GuardBreakで壁を貫通されると防御計画が崩壊
- **閾値カードとの相性**
  - 自分の閾値カードにWT_BLOCKER → 削られた瞬間に追加の壁が立つ
  - WT_DRAW → 殉陣要員を引き込む
  - HP減少を受けても「殉陣ダメージで取り返す」構造のため、閾値到達を恐れない

### A0.8 アーキタイプ間相性チャート

```
        曙(速攻)
       ↗ 有利   ↘ 不利
  玄(殉陣)          妖(呪縛)
    ↑ 有利            ↓ 有利
  遊(再演)          空(征服)
       ↖ 有利   ↙ 不利
        穏(共栄)
```

> 簡易相性（じゃんけん的な傾向、絶対ではない）：
> - 曙 → 妖に有利（コンボが揃う前に削り切る）、玄に不利（壁＋殉陣で逆に削られる）
> - 空 → 曙に有利（序盤の展開を除去で崩す）、穏に不利（横並びを1体ずつ除去しきれない）
> - 穏 → 空に有利（除去を頭数で凌ぐ）、妖に不利（全体除去系コンボで壊滅）
> - 妖 → 穏に有利（チェインで盤面ごと焼く）、曙に不利（揃う前に押し切られる）
> - 遊 → 妖に有利（リソース循環でコンボを捌ける）、空に不利（除去されるたびテンポ喪失）
> - 玄 → 曙に有利（壁で速攻を止める）、遊に不利（バウンスで殉陣が機能しない）

- `詠術` でのドロー・サーチを入れると安定性が増す

### A6.1 願主レベルアップ仕様（共通）

| レベル遷移 | 必要願成ゲージ | 効果 |

### A6.2 願主別設計意図

**Aldric（曙）** — 攻撃特化。Lv3でGuardBreakを獲得し、Blocker壁を無視して直撃を通せるフィニッシャーになる。曙の速攻戦略と噛み合い、序盤に願成を積めば中盤に制圧力が跳ね上がる。

**Vael（空）** — 妨害支援型。Lv3キーワードなし（素のステータスで勝負）。空の除去・バウンス戦略でゲームを長引かせ、高い戦力で殴る堅実型。「特殊能力がない」こと自体がVaelの空白のキャラクター性を反映。

**灯凪（穏）** — 展開防御型。Lv3でBlocker獲得。穏の横並び戦略と組み合わせ、願主自身が壁になることで盤面を安定させる。最も初心者向けの安定型願主。

**Amara（妖）** — 唯一の現在%願撃持ち願主。Lv1から現在5%（HP100なら5ダメージ）で、序盤の削り効率が高い。自壊設計型（Dismantle Architect）のリーダーとして、自軍を解体して残響効果を誘発し、墓地を肥やして終盤に墓地参照で爆発する。Lv2スキル「呪縛の糸」は自軍解体+妨害+ドロー、Lv3スキル「世界消去宣言」は全自軍解体+墓地枚数×固定2%の大ダメージ。

**Rahim（遊）** — テンポ型。Lv3でRush獲得。レベルアップ後に即攻撃可能になり、遊のバウンス→再展開→即殴りの流れと噛み合う。攻防を切り替えるトリックスター的運用。

**崔鋒（玄）** — 超防御型。初期戦力5500（最高）だが願撃は固定2%（最低）。Lv3で戦力7500＋Blockerになり、突破困難な壁として機能。「攻めない願主」という唯一無二のポジションで、玄の殉陣戦略を支える盾となる。

### A6.3 願主バランス設計

> 願主間のパワー差は意図的に均一化している（Lv3合計ステータスの差は±5%以内）。
> 差別化は **キーワード** と **ダメージタイプ**（固定% vs 現在%）で行う。
> 将来拡張で各願相に2体目以降の願主を追加し、同じ願相でも異なるプレイスタイルを選択可能にする。

- デッキ: カスタム構築のみ（プリセットデッキは使用不可）

- 対戦履歴: 直近20戦を保存・閲覧可能

## 15. ドラフトモード（v0.5.4 MVP昇格）

> ドラフトは構築済みデッキを持たないプレイヤーにも対戦の面白さを提供し、
> カードプール全体への理解を深める教育的効果もある。
> また、F2P層のエンゲージメント維持に重要な役割を果たす。

### 15.1 ドラフト概要

- **参加費**: 300G（ゴールド）または 150Gem
- **形式**: カード選択 → 即席デッキ構築 → 対戦
- **対戦数**: 最大7勝 or 3敗で終了（BO1）
- **報酬**: 勝利数に応じたゴールド + パックチケット

### 15.2 ドラフトのカード選択

1. **30枚選択フェーズ**（デッキ34枚のうち30枚をドラフトで選ぶ）
   - 10ラウンド × 3択（毎ラウンド3枚提示 → 1枚選択）= 10枚
   - 次の10ラウンド × 3択 = 10枚
   - 最後の10ラウンド × 3択 = 10枚
2. **自動補完**: 残り4枚はシステムが願相バランスを見て自動追加（バニラ顕現）
3. **願主選択**: ドラフト開始時に3体の願主から1体を選択

### 15.3 カード提示アルゴリズム

- レアリティ分布: 各3択のうち、少なくとも1枚はR以上を保証
- 願相バランス: 選択済みカードの願相を追跡し、偏りすぎないよう提示を調整
- アーキタイプ誘導: 序盤（ラウンド1〜10）は多様な願相を提示、中盤以降は選択傾向に合わせる
- 重複制限: 同名カード3枚以上は提示しない

### 15.4 報酬テーブル

| 勝利数 | ゴールド | パックチケット | 備考 |
|--------|---------|-------------|------|
| 0勝 | 50G | 0 | 参加賞 |
| 1勝 | 100G | 0 | |
| 2勝 | 150G | 0 | |
| 3勝 | 200G | 1 | 参加費回収ライン |
| 4勝 | 300G | 1 | 参加費＋α |
| 5勝 | 400G | 2 | |
| 6勝 | 500G | 2 | |
| 7勝 | 700G | 3 | 最大報酬 |

> 3勝で参加費(300G)相当を回収できる設計。
> 勝率50%のプレイヤーが長期的に微損〜均衡になるバランス。

### 15.5 ドラフト固有ルール

- 界律はドラフトプールから除外（即席デッキでの界律運用は複雑すぎるため）
- ドラフトデッキは顕現 + 詠術のみで構成
- Ambush（奇襲）カードはドラフトプールに含む（SR以上の出現率に従う）
- 対戦はランクマッチのレーティングに影響しない
- ドラフト中の切断は現在の勝敗状態を保持（再接続で続行可能）
