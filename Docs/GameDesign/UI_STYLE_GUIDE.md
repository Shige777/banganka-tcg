# UI_STYLE_GUIDE.md
> 万願果 UIスタイルガイド・デザインシステム v2.0

---

## 0. 目的

このファイルは、`万願果` のUI見た目ルールを統一するためのスタイルガイドです。
AIは新しい画面やUI要素を作る際、このファイルに従ってください。
実装者はこのファイルをデザインシステムの正本として使用してください。

> **v1.0 → v2.0 変更点**: フォントサイズ/ウェイト、8ptスペーシンググリッド、Safe Area対応、画面サイズ適応、ダークモード方針、ハプティクスフィードバック、デザイントークンを追加。

---

## 1. 全体トーン

- 異時代・異時空交錯ファンタジー
- 黒、灰、深い青、鈍い金を基調
- 願相カラーでアクセントを付ける
- 明るいカジュアルUIではなく、重厚かつ視認性の高いUIを目指す
- 競合参考: Master Duel の重厚感 + Shadowverse の華やかさの中間

## 4. スペーシングシステム（8ptグリッド）

### 4.1 基本単位

すべての余白・パディング・マージンは **8ptの倍数** を基本とする。

| トークン | 値(pt) | 用途 |
|---------|--------|------|
| `space-xxs` | 2 | アイコン内パディング |
| `space-xs` | 4 | テキスト間の微小ギャップ |
| `space-sm` | 8 | リスト項目間、ラベルとフィールドの間 |
| `space-md` | 16 | パネル内パディング、カード間ギャップ |
| `space-lg` | 24 | セクション間、画面端とコンテンツの最小距離 |
| `space-xl` | 32 | 画面タイトルと本文の間 |
| `space-xxl` | 48 | 大セクション区切り |

### 4.2 角丸

| トークン | 値(pt) | 用途 |
|---------|--------|------|
| `radius-sm` | 4 | チップ、バッジ |
| `radius-md` | 8 | カード、入力フィールド |
| `radius-lg` | 12 | パネル、モーダル |
| `radius-xl` | 16 | ボトムシート |
| `radius-full` | 9999 | 丸ボタン、アバター |

### 4.3 シャドウ

| トークン | 値 | 用途 |
|---------|------|------|
| `shadow-sm` | `0 1px 3px rgba(0,0,0,0.3)` | カード、ボタン |
| `shadow-md` | `0 4px 12px rgba(0,0,0,0.4)` | パネル |
| `shadow-lg` | `0 8px 24px rgba(0,0,0,0.5)` | モーダル、ポップアップ |
| `shadow-glow` | `0 0px 12px {aspectColor}40` | 願相グロー（カード選択時） |

## 5. 画面背景

- フラット単色は避ける
- 霧、雲、灰、魔力、光の筋などの重なりで奥行きを出す
- 背景は派手すぎず、前面のパネルと文字を邪魔しないこと
- パフォーマンス考慮: 背景は静止画 + 軽量パーティクルで構成。フルアニメーション背景はバトル画面のみ許可

---

## 6. パネル

- 半透明の暗色パネル（`bg-secondary` + 不透明度 85〜95%）を基本とする
- 背景から情報を切り分けるため、`border-default` の 1px 縁取りか内側シャドウを付ける
- 角丸は `radius-lg`（12pt）を基本とする
- パネル内パディングは `space-md`（16pt）

### 6.1a HP ゲージ表示（v0.5.3）

- **HP ゲージ形状**: 水平バー（0～100）
- **カラー**: デフォルト `#4CAF50`（緑）→ 低 HP 時に `#FF9800`（橙）→ 危機的 HP 時（≤15）に `#F44336`（赤）
- **表示位置**: 願主ポートレート上部、相手ユーザー情報近傍
- **テキスト**: 現在 HP / 最大 HP（例："75 / 100"）
- **アニメーション**: ダメージ時に滑らかにスライド減少（EaseInOutSine 0.6s）
- **閾値マーカー**: HP 85/70/55/40/25/10 に微小な分割ライン（WishTrigger 対応）

### 6.1b 願主スキルUI（v0.5.4）

- **スキルアイコン配置**: 願主ポートレート下部に2つ並列表示（左=Lv2スキル、右=Lv3スキル）
- **アイコンサイズ**: 36×36pt（タップターゲットは44×44pt以上確保）
- **状態表示**:
  - **未解禁（レベル未到達）**: アイコン暗転＋ロックマーク、タップ不可
  - **使用可能**: アイコン発光＋微振動アニメーション（pulse 2s infinite）
  - **使用済み**: グレーアウト `opacity: 0.35`＋「×」マーク
- **カラー**: スキルアイコン枠はLv2=`#FFD700`（金）、Lv3=`#FF4500`（赤橙）
- **ツールチップ**: 長押しでスキル名・効果テキスト・使用回数を表示（吹き出し形式、最大3行）
- **相手スキル表示**: 相手の願主スキル使用状況も同様にアイコン表示（公開情報）

> 願主スキルの詳細仕様は `GAME_DESIGN.md` §7.4 を参照。

### 6.1c 情相（EmotionTag）表示（v0.5.4）

- **表示位置**: カード詳細ビュー内、願相アイコンの右隣に小アイコンで表示
- **アイコンサイズ**: 20×20pt
- **カラー定義**:
  - 渇望（Thirst）: `#E53935`（紅） — 炎のような赤
  - 贖罪（Atonement）: `#7B1FA2`（紫） — 沈鬱な紫
  - 慈愛（Grace）: `#43A047`（翠） — 穏やかな緑
  - 執着（Cling）: `#F57F17`（琥珀） — 粘り気のある橙
  - 諦観（Resign）: `#546E7A`（鈍色） — 灰がかった青
  - 怨念（Grudge）: `#B71C1C`（焦紅） — 暗い深紅
  - None（無相）: 表示なし
- **盤面表示**: カード上の情相アイコンは小サイズ（16×16pt）で右下に配置
- **フィルタUI**: デッキ構築画面で情相フィルタを願相フィルタの隣に配置

### 6.1d 界律セット状態表示（v0.5.4）

- **裏向き界律**: カード裏面＋封印紋様（FX_LAW_SEAL）を界律枠に表示
- **裏向きラベル**: 「セット中」テキストを枠下部に表示（両プレイヤーに見える）
- **表返し可能表示**: 自分のメインフェイズ中、裏向き界律の枠が脈動（pulse）して「タップでオープン」のヒント表示

> 界律のセット/オープン仕様は `GAME_DESIGN.md` §5.4.1 を参照。

## 8. カードUIルール

### 8.1 カード枠サイズ

| コンテキスト | 幅(pt) | 高さ(pt) | 用途 |
|------------|--------|---------|------|
| バトルフィールド | 72 | 100 | 盤面配置時 |
| 手札 | 64 | 90 | 手札表示 |
| コレクション一覧 | 80 | 112 | カード一覧グリッド |
| カード詳細 | 240 | 336 | 拡大表示 |
| デッキ構築 | 56 | 78 | デッキリスト内 |

### 8.2 カード枠構造

枠の形は全カード共通。上部にイラスト領域、下部にテキスト領域。

| 領域 | 配置 | 内容 |
|------|------|------|
| コストバッジ | 左上 | 願相カラー背景 + 白文字(`card-cost`) + 黒 1px 縁 |
| 願相チップ | 右上 | 願相アイコン + 願相カラードット（二重識別） |
| イラスト | 中央上部 | `illustrationId` の画像。願相カラーの薄いオーラを外周に |
| 名前帯 | イラスト下 | `bg-secondary` 帯。願相カラーの左端アクセントライン |
| テキスト領域 | 下部 | 効果テキスト（`body-sm`）。キーワードは太字 |
| 戦力/願撃 | 右下 | 戦力(`card-power`) / 願撃(`card-power`) |

### 8.3 レアリティ装飾

| レアリティ | 枠装飾 | 名前帯 |
|-----------|--------|--------|
| C | 鉄色細枠 | 装飾なし |
| R | 銀色枠 + 微光 | 銀のアクセントライン |
| SR | 金色枠 + グロー | 金のアクセントライン + 微粒子 |
| SSR | 虹色枠 + 強グロー | 虹のアクセントライン + パーティクル |

## 10. Safe Area・画面サイズ適応

### 10.1 Safe Area 対応

| 端末 | 上部 | 下部 | 対応方法 |
|------|------|------|---------|
| iPhone 15 Pro Max | 59pt | 34pt | Unity `Screen.safeArea` を使用 |
| iPhone SE (3rd) | 20pt | 0pt | 同上 |
| iPad (10th) | 24pt | 20pt | 同上 |

- すべてのインタラクティブ要素は Safe Area 内に配置する
- 背景・装飾要素は Safe Area 外まで拡張してよい（edge-to-edge デザイン）
- 下部タブバーは Safe Area の下端に合わせ、背景のみ画面端まで伸ばす

### 10.2 画面サイズ適応

| デバイスカテゴリ | 画面幅(pt) | レイアウト調整 |
|----------------|-----------|-------------|
| iPhone SE / mini | 320〜375 | カード一覧: 3列、手札: 重なり表示 |
| iPhone 標準 | 376〜414 | カード一覧: 4列、手札: 標準表示 |
| iPhone Plus / Max | 415〜430 | カード一覧: 4列、手札: ゆとり表示 |
| iPad | 744〜1024 | カード一覧: 6列、バトル画面は横持ち対応 |

### 10.3 回転（Orientation）

- iPhone: **縦持ち固定**（Portrait Lock）
- iPad: 縦横両対応（MVP後 Phase 1 で対応）。MVP時は縦持ち固定

## 11. ダークモード / ライトモード

### 11.1 MVP方針

- MVPは **ダークモードのみ**（上記カラーパレットがダークモード）
- TCGの世界観（異時代ファンタジー）に合致し、カードイラストが映える
- 競合参考: Master Duel, MTG Arena もダーク基調

### 11.2 将来のライトモード対応方針（POST_MVP Phase 1）

ライトモード追加時は以下のトークンを差し替える:

| トークン | ダーク | ライト（将来） |
|---------|--------|-------------|
| `bg-primary` | `#0D0D14` | `#F5F5F0` |
| `bg-secondary` | `#1A1A2E` | `#FFFFFF` |
| `text-primary` | `#FFFFFF` | `#1A1A2E` |
| `text-secondary` | `#B0B0CC` | `#6E6E8A` |
| `border-default` | `#3A3A5C` | `#D0D0E0` |

- 願相カラーは両モードで同一HEX値を使用（コントラスト比を維持）
- 背景画像はモード別に2種用意する
- 実装時は Unity の ScriptableObject でトークンを管理し、モード切替で差し替える

## 12. アイコンと装飾

### 12.1 願相アイコン

- 各願相は **色＋形状** の二重識別（`ACCESSIBILITY_SPEC.md §1`）
- アイコンサイズ: 最小 20×20pt、推奨 24×24pt
- 線幅: 2pt（視認性確保）

| 願相 | アイコン形状 | 色 |
|------|----------|------|
| 曙 | 剣（上向き三角） | `#FF5A36` |
| 空 | 目（楕円＋円） | `#4DA3FF` |
| 穏 | 葉（二重曲線） | `#59C36A` |
| 妖 | 渦（螺旋） | `#9A5BFF` |
| 遊 | 星（五芒星） | `#F4C542` |
| 玄 | 鎖（連環） | `#6E6E8A` |

### 12.2 装飾ルール

- 光・粒子・オーラは補助的に使う。情報を邪魔しないこと
- レアリティ装飾（§8.3）以外のパーティクルは最小限に
- アニメーション低減モード（`ACCESSIBILITY_SPEC.md §4`）では全パーティクルを停止する

## 13. ハプティクスフィードバック

### 13.1 対応デバイス

- iOS: UIFeedbackGenerator（Taptic Engine）
- 非対応端末: ハプティクスをスキップ（エラーにしない）
- 設定画面でオン/オフ切替可能（デフォルト: オン）

### 13.2 ハプティクスマッピング

| アクション | フィードバック種別 | 強度 |
|-----------|----------------|------|
| カードプレイ（手札→フィールド） | `UIImpactFeedbackGenerator` | `.medium` |
| 顕現攻撃宣言 | `UIImpactFeedbackGenerator` | `.heavy` |
| HPゲージ変動 | `UIImpactFeedbackGenerator` | `.light` |
| パック開封（封入カード出現） | `UINotificationFeedbackGenerator` | `.success` |
| SSRカード出現 | `UINotificationFeedbackGenerator` | `.success` × 3連打 |
| ターン開始通知 | `UISelectionFeedbackGenerator` | — |
| ボタンタップ | `UISelectionFeedbackGenerator` | — |
| エラー（CP不足等） | `UINotificationFeedbackGenerator` | `.error` |
| 勝利 | `UINotificationFeedbackGenerator` | `.success` + カスタムパターン |
| 敗北 | `UIImpactFeedbackGenerator` | `.heavy` (1回) |

> `ANIMATION_SPEC.md` のアニメーションタイミングと同期すること。

## 14. トランジション・マイクロインタラクション

### 14.1 画面遷移

| 遷移種別 | アニメーション | 時間 |
|---------|-------------|------|
| タブ切替 | クロスフェード | 200ms |
| Push（画面階層下降） | 右からスライドイン | 300ms |
| Pop（画面階層上昇） | 左へスライドアウト | 250ms |
| モーダル表示 | 下からスライドアップ + フェードイン | 350ms |
| モーダル非表示 | 下へスライドダウン + フェードアウト | 250ms |

### 14.2 マイクロインタラクション

| 要素 | 効果 | 時間 |
|------|------|------|
| ボタン押下 | scale(0.97) | 100ms |
| ボタン離す | scale(1.0) | 150ms (ease-out) |
| カードホバー/長押し | scale(1.05) + shadow-lg | 200ms |
| トースト通知表示 | 上からスライドイン | 300ms |
| トースト通知非表示 | フェードアウト | 200ms (3秒後自動) |

### 14.3 イージング

| 用途 | イージング関数 |
|------|-------------|
| 画面遷移 | `ease-in-out` (cubic-bezier 0.42, 0, 0.58, 1) |
| 要素出現 | `ease-out` (cubic-bezier 0, 0, 0.2, 1) |
| 要素消滅 | `ease-in` (cubic-bezier 0.4, 0, 1, 1) |
| バウンス（パック開封等） | `cubic-bezier(0.34, 1.56, 0.64, 1)` |
