# 万願果（ばんがんか）— オリジナルTCG 企画書

**企画・設計・実装**: 内山 茂樹
**作成日**: 2026年3月
**プラットフォーム**: Nintendo Switch / PS5 / PC (Steam) / iOS / Android（基本プレイ無料）

---

## コンセプト — このゲームで得られる体験と感情

> **「願いは、追い詰められるほど強くなる」——追い詰められた願主が覚醒し、試合をひっくり返す1v1対戦TCG**

万願果では、HPが削られるほど願主が成長する。HPが閾値を踏むたびに願力カードが発動し、同時に願主の成長ゲージが溜まる。追い詰められた先に必殺のリーダースキルが解放され、試合の流れをひっくり返す——この**「追い詰められるほど願いが力に変わる」**体験こそ、万願果で一番届けたい感情。

デュエル・マスターズのシールドトリガーも逆転メカニクスだが、あれは「ランダムに埋まったカードが捲れる運の逆転」。万願果の逆転は「願主が成長し、スキルを切るタイミングを自分で判断して掴み取る逆転」。**運ではなく、自分の判断で願いを通す**。

すべてのゲームデザイン（ルール・演出・サウンド・UI）は、この「追い詰められるほど強くなる」体験を最大化するための手段として設計している。

この核となる感情を支える2つの体験：
- **読み合いの快感** — 界律の上書き、願主スキルのタイミング、奇襲。CPを「カードに使うか、願成チャージに使うか」のリソース判断。読みが当たったから逆転できた、という因果関係が逆転を「実力」にする
- **願いの物語への没入** — 6人の願主がそれぞれの願いを賭けて戦う。願主が追い詰められて覚醒する体験は、ストーリーモードで語られる「願いを諦めない物語」と重なる

## 既存TCGとの差別化

### 既存TCGとの差別化

| 要素 | 万願果の設計 | 既存TCGとの違い |
|------|------------|----------------|
| 勝利条件 | HP削り＋直撃（Final追撃で即勝利） | 「常に攻める動機」＋「最後まで逆転可能」の両立 |
| フィールドルール | 界律（共有1枠・上書き式） | 設置者ボーナスがある非対称な共有ルール |
| リーダー | 願主（試合中に成長＋固有スキル） | ストーリーと直結した成長システム |
| HP表示 | 数値ではなくゲージ表示 | リーサル判断に経験と読みが介在する |
| ダメージ | %制（固定%/現在%の2種） | HP残量による非線形なダメージ計算 |

## 1. テーマ・ターゲットと体験設計

### テーマ — このゲームで誰に何を届けるか

| 要素 | 内容 |
|------|------|
| **どんなプレイヤーに** | TCG経験者＋対戦ゲームに興味があるライト層（15〜35歳） |
| **どんな世界に向けて** | 既存TCGの環境固定化やカードインフレに疲れたプレイヤー、そしてTCGの読み合いの面白さをまだ知らない人たちに |
| **どこに変化を起こすか** | 「システムが健全に回り続ける対戦環境」と「キャラクターの願いに感情移入しながら戦う体験」を届ける |

### ターゲット

- **コア**: 15〜35歳、TCG/対戦ゲーム経験者。「構築と読み合いの楽しさ」を知っている層
- **ライト**: TCG未経験だが対戦ゲームに興味がある層。プリセットデッキとチュートリアル12ステップで導入

### プレイヤーの体験フロー

```
初回起動 → チュートリアル（灯凪の章、HP30短縮戦）→ AI対戦で練習
   → ランクマッチ参戦 → パック開封でカード収集
   → デッキ構築の試行錯誤 → ストーリーモードで世界観を知る
   → 界律・情相の段階アンロック → ドラフトモードで構築力を試す
```

**敷居と天井の分離**:

TCGの最大の参入障壁はデッキ構築にある。カードを知らない初心者に「34枚選べ」と言っても何をすればいいかわからない。SF6のモダン操作が「最初から同じ対戦を体験できる入口」であるように、万願果では**プリセットデッキ**を用意し、構築を省略してすぐに対戦の面白さを体験できるようにした。

- 初心者はチュートリアルで顕現と詠術だけを使ってプレイ開始（操作の敷居を下げる）
- 界律（共有フィールドルール）と情相（感情タグ）シナジーはプレイを重ねるにつれ段階的にアンロック（戦略の天井を維持する）
- HP30の短縮戦で基本を学び、HP100のフル対戦で奥深さに到達

敷居（プリセットデッキで即対戦）と天井（6色×3タイプ×情相タグの構築自由度）は独立に設計可能であり、どちらかを犠牲にする必要はない。

## 2. ゲームルール

以下のルールはすべて「逆転のドラマ」を構造的に生み出すための手段として設計している。

### 準備

- 各プレイヤーはデッキ **34枚**（同名3枚上限）を持つ
- 先攻/後攻をランダム決定。先攻5枚、**後攻6枚**の初期手札
- デッキ上6枚を**願力ゾーン**に伏せて配置（閾値: 85%/70%/55%/40%/25%/10%）

### バトル画面の全体像（DQライバルズ型グリッドUI）

```
┌─────────────────────────────────────────────┐
│  相手の願主 [Lv2] ██████████░░ HP:62%       │
│                                             │
│  ┌──────┐ ┌──────┐ ┌──────┐  ← 相手後列    │
│  │ 顕現 │ │ 顕現 │ │  空  │                │
│  └──────┘ └──────┘ └──────┘                │
│  ┌──────┐ ┌──────┐ ┌──────┐  ← 相手前列    │
│  │ 顕現 │ │  空  │ │ 顕現 │                │
│  └──────┘ └──────┘ └──────┘                │
│           ┌────────────┐                    │
│           │  界律カード  │  ← 盤面中央      │
│           │ 《突撃陣》   │    両者に影響     │
│           └────────────┘                    │
│  ┌──────┐ ┌──────┐ ┌──────┐  ← 自分前列    │
│  │ 顕現 │ │ 顕現 │ │  空  │                │
│  └──────┘ └──────┘ └──────┘                │
│  ┌──────┐ ┌──────┐ ┌──────┐  ← 自分後列    │
│  │  空  │ │ 顕現 │ │  空  │                │
│  └──────┘ └──────┘ └──────┘                │
│                                             │
│  自分の願主 [Lv1] ████████████████░░ HP:85% │
│  CP: 4/5    手札: ▮▮▮▮▯                    │
│  [願力ゲージ ██████░85%|70%|55%|40%|25%|10%]│
└─────────────────────────────────────────────┘
```

**画面の設計意図**:
- 界律カードが**盤面の中央**に大きく表示 → 「今どのルールが適用されているか」が一目でわかる。万願果の最大の独自メカニクスを画面の最も目立つ場所に置く
- 願力ゲージに**閾値マーカー**を表示 → 「次の閾値まであとどれくらいか」が目測でわかる。逆転のドラマの緊張感を視覚化
- HPは数値ではなく**ゲージ表示** → リーサル判断に経験と読みが介在し、「ギリギリ届かなかった」「思ったより削れた」のドラマが生まれる

### ターンの流れ

```
1. スタート   … 自軍ユニット全員待機状態に回復
2. ドロー     … CP上限+1（最大10）、CPフル回復、1枚ドロー
3. メイン     … カードプレイ（顕現/詠術/界律）、願主スキル使用
4. コンバット … 攻撃宣言 → ブロック選択 → 戦力比較 → 願力ダメージ解決
5. エンド     … ターン終了、相手のターンへ
```

### 勝利条件（2層構造）

| 条件 | 名前 | 説明 |
|------|------|------|
| **即時勝利** | 直撃勝利 | 相手HPが0（Final状態）の時にもう一度直接攻撃を通す |
| **時間切れ** | 判定勝利 | 24ターン終了時、HP残量が多い方が勝利 |

**この2層構造が「追い詰められるほど強くなる」を加速する**：
- Final状態に追い詰められた側は、願力カードの閾値を全て踏んでいる → **願成ゲージが大量に溜まり、願主スキルが解放済み**。最も追い詰められた状態が、最も強い状態でもある
- HP1%でも残っていれば、覚醒した願主のスキルで逆転のチャンスがある
- 攻める側は「追い詰めるほど相手の願主が成長する」リスクを抱える → 攻めるリスクがコンセプトから自然に生まれる

**「攻守のジレンマ」が構造から保証されている**：デュエル・マスターズのシールドトリガーは「攻めた側にランダムなリスクがある」構造だが、万願果の願力カード閾値はゲージ上のマーカーとして位置が見える。ただしHPはゲージ表示（数値非表示）のため、「この攻撃で踏むかどうか」はSF6の体力ゲージと同じ「だいたい読めるが確定ではない」不確実性になる。直撃勝利と判定勝利の2層構造が「攻めないと勝てない」動機を保証し、願力カード閾値が「攻めすぎると逆転される」リスクを保証する。攻撃のリスクと攻撃の必要性が構造的に両立している。

### カードタイプ（3種）

#### 顕現（Manifestation）— ユニット

盤面に配置する戦闘ユニット。攻撃・ブロックに参加する。

```
┌─────────────────────┐
│ CP: 3    曙(赤)       │
│                       │
│   [イラスト]          │
│                       │
│ 《焔の騎士アルヴァ》   │
│ 戦力: 5000            │
│ 願力: 5%(固定)         │
│ [Rush]                │
│ 「誓いの焔は、絶えず  │
│  己を焦がし続ける。」  │
└─────────────────────┘
```

- **戦力**: 戦闘の勝敗を決める数値。高い方が勝つ
- **願力ダメージ**: 直接攻撃時に相手HPを削る値（%制）
- **キーワード**: Rush（即攻撃可能）、Blocker（ブロック可能）、GuardBreak（ブロック無視）、Ambush（相手ターン割り込み）

#### 詠術（Incantation）— スペル

手札から使い捨て。即時効果を発動する。

```
┌─────────────────────┐
│ CP: 2    穏(緑)       │
│                       │
│   [イラスト]          │
│                       │
│ 《大地の縫合》         │
│ 味方1体の戦力+2000    │
│                       │
│ 「壊れた世界の境を    │
│  繕い、守りを固める。」│
└─────────────────────┘
```

#### 界律（Algorithm）— 共有フィールドルール

**万願果の最大の独自メカニクス。** 盤面に1枚だけ存在できる共有ルール。後から出した界律が前の界律を上書きする。

```
┌─────────────────────────────────────┐
│ CP: 3    曙(赤)                      │
│                                      │
│ 《突撃陣》                           │
│                                      │
│ 【共有ルール】CP2以下の顕現にRush付与 │
│ 【設置者ボーナス】CP2以下の願力ダメ+1 │
│                                      │
│ 「誓いは速さに変わり、               │
│  境界を駆け抜ける。」               │
└─────────────────────────────────────┘
```

**駆け引きのポイント**:
- 共有ルールは**両者に適用**される → 自分有利な界律でも、相手にも恩恵がある
- **設置者ボーナス**は設置者だけに適用 → 同じ界律でも自分が張った方が有利
- **裏向きセット**可能 → 次ターンにオープン。相手に「何が来るか」を読ませる駆け引き
- 1枠しかないため、**いつ上書きするか**が戦略の鍵

### 願主成長システム（進化ゲージ）— 「願いは、追い詰められるほど強くなる」

願主は試合中に成長する。成長手段は2つ：

| 成長手段 | 内容 | コンセプトとの関係 |
|---------|------|-----------------|
| **願成チャージ** | CP1消費で願成+1（メインフェーズ中、何回でも） | 能動的な成長。CPをカードに使うか成長に使うかの判断 |
| **閾値踏み** | HPが願力カード閾値を下回ると願成+1 | **追い詰められるほど成長する**。コンセプトの核 |

```
Lv1 ──[3pt]──> Lv2 ──[4pt]──> Lv3
              スキル1解放    スキル2解放
              戦力+1000     戦力+1000
```

- リーダースキルは**CP不要**、**各レベル1ゲーム1回**
- **CPを使って能動的に成長を急ぐか、カードプレイにCPを回すか**のリソース判断
- **HPが減ると願成も溜まる** → 追い詰められた側がスキル解放に近づく → 逆転の芽が構造的に生まれる

### 奇襲（Ambush）— 相手ターンの割り込み

手札から特定の顕現を相手ターン中にプレイできる（追加コスト: 手札1枚捨て）。

- **防衛型**: 相手の直接攻撃宣言時に発動 → ブロッカーとして登場
- **報復型**: 味方ユニット退場時に発動 → 即座に反撃態勢

「相手の手札にAmbush持ちがいるかもしれない」という可能性が、攻撃の判断を複雑にする。

## 3. 願主（リーダー）と世界観

### 世界設定

「交界」と呼ばれる時空の交差点に、あらゆる時代・世界から6人の「願主」が集まる。彼らは「万願果」——あらゆる願いを叶える果実を求めて戦う。プレイヤーは「果求者」として願主の力を借り、対戦に挑む。

### 6人の願主

| 願主 | 願相 | 色 | 願い | アーキタイプ |
|------|------|-----|------|------------|
| **Aldric** | 曙 | 赤 | 滅ぼした民を蘇らせたい（贖罪） | 速攻・波状攻撃 |
| **Vael** | 空 | 青 | 自分がここにいた証を残したい（存在証明） | 妨害・制圧 |
| **灯凪** | 穏 | 緑 | 自分の物語を誰かに届けたい（盲目の語り部） | 堅実・全体バフ |
| **Amara** | 妖 | 紫 | 自分が作った世界を消したい（破壊と解放） | 自壊・墓地蓄積爆発 |
| **Rahim** | 遊 | 黄 | 亡くした姉を蘇らせたい（喪失と成長） | 再演・リソース循環 |
| **崔鋒** | 玄 | 暗 | 敗者の歴史を記録に残したい（記録） | 耐久・殉陣 |

**設計意図**: 各願主の「願い」がゲームメカニクスに直結している。

- Aldricの贖罪 → 味方全員を奮い立たせる速攻（Lv3: 全軍Rush+GuardBreak）
- Amaraの破壊衝動 → 自軍を解体して爆発（Lv3: 全自壊→墓地枚数×2%ダメージ）
- 崔鋒の殉陣 → 自分が犠牲になっても記録を残す（Lv3: 自爆→戦力10%固定ダメージ）

「この願主はなぜこの戦い方をするのか」が物語で理解でき、キャラクターへの愛着がデッキ選択の動機になる。

**設計意図**: アスペクトを跨いだシナジーデッキが成立する。例えば「渇望」タグのカードを集めれば、赤+黄の攻撃的な混色デッキが組める。

## 5. バランス設計の思想

### コストカーブ — 数式でバランスの基準線を定義

```
バニラ顕現の基準戦力 = CP × 2000 + 1000

例: CP2 → 5000, CP3 → 7000, CP5 → 11000
```

キーワード・効果を持つカードは、この基準から**ペナルティ**を引いて戦力を決定する：

| キーワード/効果 | ペナルティ |
|----------------|-----------|
| Rush | -500 |
| Blocker | -500 |
| GuardBreak | -500 |
| 登場時ドロー | -1000 |
| 登場時除去 | -1500 |

このルールにより：
- 全カードの強さが一貫した基準で設計される
- 新カード追加時にパワーインフレを防げる
- 「このカードは基準より強い/弱い」が明確に判断できる

### 構造的課題はシステムで解消する

TCGには構造的な課題がある。万願果ではこれらを個別のカード調整ではなくシステムレベルで解消する設計を採った。

**環境固定化の回避 → 界律の「共有1枠・上書き式」**：ある界律が環境を支配しても、それを上書きする界律がメタとして必ず機能する。界律は両者に影響するため、相手の界律をそのまま利用する戦術も成立する。個別のカードをナーフして環境を動かすのではなく、ゲームの基本ルール自体が環境を循環させる。

**終盤の手札枯れ回避 → CP制**：デュエマのマナ制では手札がリソースを兼ねるため、終盤に手札が枯れてトップ勝負になりやすい。万願果ではCP（毎ターン自動累積＋フル回復）を採用し、手札がリソースを兼ねない設計にした。終盤でも選択肢が残り、最後まで判断が面白い状態を維持できる。

**先攻/後攻バランス → 手札枚数補正**：特定のカードを強化して後手を救うのではなく、手札枚数というシステムレベルで補正する（先攻5枚、後攻6枚）。目標は先攻勝率45〜55%。108,000戦の自動バランスシミュレーションで検証済み。

### レアリティ ≠ 強さ

- Common（C）でも構築の核になるカードがある
- レアリティは**入手難易度のみ**を表す
- SSRが「強い」のではなく「効果がユニーク」な設計

## 6. コンセプト「逆転のドラマ」を実現するための演出設計

ルールだけでなく、グラフィック・サウンド・UIのすべてがコンセプトの感情を増幅する手段。

### 「読み合いの快感」を実現する演出

- **界律カードが盤面中央に大きく表示** — DQライバルズ型グリッドUI。今どのルールが適用されているかが一目でわかり、「この界律を上書きすべきか」の判断が画面から直接読み取れる
- **願主の進化演出** — レベルアップ時にカットイン演出＋専用SE。「ここからスキルが使える」という戦略的転換点を感情的に盛り上げる
- **奇襲（Ambush）発動演出** — 相手ターン中に画面が揺れ、伏せカードがめくれる。「待ってました！」の快感を演出で増幅

### 「願いの物語への没入」を実現する演出

- **願主ごとの固有BGM** — 6人の願主にそれぞれテーマ曲。自分のデッキの願主の曲が流れることで「この願主と一緒に戦っている」感覚
- **フレーバーテキスト** — 全162種のカードに世界観を伝えるテキスト。カードを集めるほど世界の全体像が見えてくる
- **パック開封演出** — カードが光りながら現れるエフェクト。レアリティに応じた演出の豪華さ。「何が出るか」のワクワク感を最大化（おもちゃ的遊びの要素）

### 「追い詰められるほど強くなる」を実現する演出

- **願力ゲージの減少演出** — HPが閾値に近づくとゲージが赤く点滅。「次の攻撃で閾値を踏む＝願主が成長する」の緊張と期待が同時に生まれる
- **願力カード発動＋願成ゲージ上昇の同時演出** — 閾値を下回った瞬間、伏せカードがめくれると同時に願成ゲージが光って上昇。「追い詰められたが、願いが強くなった」を視覚的に表現
- **願主覚醒（レベルアップ）演出** — 閾値踏みでレベルアップした場合、特別なカットイン。「ここから逆転が始まる」を最も盛り上がる演出で表現
- **直撃勝利の演出** — Final状態からのトドメは特別なフィニッシュ演出。覚醒した願主のスキルでトドメを刺した場合はさらに豪華に

## 7. マネタイズ設計

### 設計思想 — Pay-to-Winにしない

- 課金による差は**カード入手速度のみ**。レアリティ=強さではない
- 無課金でも全カードフルコンプ可能（約8〜10ヶ月）
- 毎日のプレイ報酬で約1パック/日のペースで開封可能

### 収益構造

| 収益源 | 内容 | ターゲット |
|--------|------|-----------|
| パック | 通常(5枚500G) / プレミアム(10枚900G) | 全プレイヤー |
| バトルパス | 月額500願晶。広告なし+限定コスメ+SSR確定チケット | 中課金 |
| コスメ | カードスリーブ、願主スキン、盤面エフェクト | 全課金層 |
| スターターバンドル | 初回限定セット | 新規 |
| リワード広告 | 1日3回まで。報酬ブースト | 無課金 |

### 天井（保証システム）

- SSRは**50パックに1枚保証**（シリーズ別カウント）
- 4枚目以降の重複はゴールド/生成素材に自動変換
- カード生成（クラフト）で任意のカードを直接生成可能

## 8. 設計の振り返りとアピール

### うまくいった設計判断

**界律（Algorithm）の「共有1枠・上書き式」**

ポケモンカードのスタジアムカードから着想を得た。スタジアムは両者に影響するフィールドカードだが、万願果では「1枠しかない」「設置者にボーナスがある」という2つの制約を加えた。これにより：
- 「いつ上書きするか」の駆け引きが生まれた
- 設置者ボーナスにより「同じ界律でも自分が張った方が有利」という非対称性が生まれた
- 盤面に常に1つだけルールが存在するため、初心者にも理解しやすい

**願力カード閾値による「読める逆転」**

デュエル・マスターズのシールドトリガーから着想を得た。「攻めた側にリスクがある」構造を取り入れつつ、閾値の位置をゲージ上のマーカーとして示すことで「どこで発動するか」はわかるようにした。ただしHP自体は数値ではなくゲージ表示のため、「この攻撃で踏むかどうか」は目測と経験による判断になる。完全なランダムでもなく完全な計算でもない、「だいたい読めるが確定ではない」不確実性が駆け引きを生んでいる。

**HPのゲージ表示**

ストリートファイター6の体力ゲージから着想を得た。数値ではなくゲージで表示することで、リーサル判断に経験と読みが介在する。正確に読めるプレイヤーと読めないプレイヤーの間に、運ではなく実力による差が生まれる。

### 試して戻した設計判断

**前列/後列＋界律中央配置の採用**

盤面を前列3枠＋後列3枠の計6枠とし、中央に界律カードを配置するDQライバルズ型グリッドUIを採用している。

開発途中で一度シャドウバース型の横並びレイアウトに変更して比較した。しかし界律は万願果の最大の独自メカニクスであり、盤面中央に大きく目立つ形で表示すべきと判断し、グリッド型に戻した。

- 前列/後列の配置判断が戦略の幅を広げる（Blockerを前列に置いて後列を守る等）
- 界律カードが盤面中央にあることで「今どのルールが適用されているか」が視覚的に明確
- グリッド構造が界律の存在感を最大化し、万願果らしさを盤面で表現している

**学び**: 一度変えて比較した上で選び直すプロセスが、設計の確信を深める。「試さずに悩む」より「試して比較する」方が正しい判断に辿り着ける。
