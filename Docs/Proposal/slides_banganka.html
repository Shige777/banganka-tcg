駆け引きの構造

  • 盤面に1枚だけ存在できる共有ルール
  • 後から出した界律が前の界律を上書き
  • 共有ルールは両者に適用される
  • 設置者ボーナスは設置者だけに適用
  • 裏向きセット可能 → 次ターンにオープン
  • 1枠しかないためいつ上書きするかが戦略の鍵

★ ポケモンカードのスタジアムから着想。「1枠」「設置者ボーナス」の2制約を加えることで、上書きタイミングの駆け引きと非対称性を実現した。

成長手段 (2種)

願成チャージ — CP1消費で願成+1。何回でも可能。CPをカードに使うか成長に使うかの判断。

閾値踏み — HPが閾値を下回ると願成+1。追い詰められるほど成長 = コンセプトの核。

攻める側のジレンマ

追い詰めるほど相手の願主が成長。攻めないと勝てないが、攻めると覚醒される。

鯱鉾勝利(追撃で即勝) + 塗り勝利(24T後HP比較) → 攻撃の必要性とリスクが両立。

攻守のジレンマは構造で保証する

デュエマのシールドトリガーから「攻めた側にリスクがある」構造を学び、願力カード閾値+鯱鉾/塗り勝利の2層で「攻めないと勝てない × 攻めるとリスク」を構造的に保証。

構造的課題はシステムで解消する

界律の「共有1枠・上書き式」が環境を自然に循環させる。個別カードのナーフに頼らない設計。CP制（自動累積+フル回復）で終盤の手札枯れを構造的に回避。

敷居と天井は独立に設計できる

プリセットデッキ+HP30短縮戦で敷居を下げ、界律・情相・ドラフトで戦略の天井を維持。SF6のモダン操作と同じ設計思想。

試して戻した判断

前列/後列をシャドバ型に変更して比較。界律の存在感を最大化するためグリッド型に戻した。「試さずに悩む」より「試して比較する」。
